* 演化故事：从"怎么把模型压扁"开始

** 第 0 层：OBS / OBD —— 不能瞎删，要看二阶导

这事其实 1989/1993 年就开始了，是给神经网络剪枝用的。
当时的问题朴素得不行：模型太大跑不动，能不能把一些权重直接归零？
朴素答案：把绝对值小的删掉。
LeCun 他们说不行——你只看权重大小，没看这权重一旦动了会让 loss 抖多少。
真正该删的是"动了它，loss 几乎不变"的那些。

那怎么知道动了它 loss 会不会抖？把 loss 在当前权重处展开，泰勒二阶项里有个 Hessian。
Hessian 告诉你每个权重的"敏感度"，对角线上数值越大越敏感、越不能动。
*这套二阶导决定权重重要性的思路*，就是后面所有低比特量化方法的祖宗。

留下的问题：Hessian 太大了，一个层就要 O(d^2) 的矩阵。能不能用？

** 第 1 层：GPTQ —— 把祖宗的招在 LLM 上跑通

到 2022 年 LLM 起来了，GPT-3 级别的模型有上百 B 参数。
全精度跑不动，要量化到 4bit 甚至更低。
当时主流是 round-to-nearest（RTN）：每个权重独立量化到最近的格子点。
朴素是朴素，但量化误差会一点点累积，跑分掉得厉害。

GPTQ 看到的问题：你逐个权重独立量化，等于假设权重之间不相关。可是层内权重当然相关。
*改进*：把每一层看成一个"逐列重建"的问题——量化第 j 列时，前面已经量化好的列固定下来，后面没量化的列允许补偿，目标是最小化整层输出的均方误差。
而"补偿哪些列、补偿多少"——用什么算？用 Hessian。
本质上就是把 OBS 的二阶展开搬到 LLM 的层级，做出来一个一次过、逐层完成的 PTQ 方案。

GPTQ 解决了 4bit 能用的问题。但留下了一个新麻烦：再往下走呢？2bit？1bit？
量化误差会爆炸，因为格子点之间的间距变得跟权重本身一样大了。

** 第 2 层：PB-LLM —— 别都压到 1bit，留个口子

2023 年开始有人尝试 1bit 量化。
难点很直接：1bit 只有两个可选值 ±α，绝大部分权重的"真值"被压扁成两个数，信息损失太大。
模型直接坏掉。

PB-LLM（Partially-Binarized LLM）看到的问题：其实不是所有权重都"同等重要"。
GPTQ 那个 Hessian 已经告诉过你——有些权重一动 loss 就抖，有些动了无所谓。
*改进*：把权重分两堆。重要的（salient）那一小部分继续用 4bit 或 8bit；剩下大头（non-salient）才一刀切到 1bit。
所以叫"部分二值化"。这样平均比特数还是很低，但关键权重得到了保护。

留下的问题：这本质上是个混精度方案，1bit 比例越高越省内存但越掉点。能不能真正全部都 1bit，还不掉太多？

** 第 3 层：BiLLM —— 把"分两堆"做彻底

2024 年初的 BiLLM 把这事推到 *纯二值化*（所有权重都 1bit）的边界。
它看到的 PB-LLM 的问题：保留一部分高比特，本质上是"承认搞不定"，最终内存还是没省到极致。

*改进*：BiLLM 也分类，但分类后两类都用 1bit，只是给它们不同的 scale 和不同的二值化策略：
- 显著权重（重要的少数）：用更精细的"残差二值化"——先 1bit 拟合一次，剩下的残差再 1bit 拟合一次（等于 2 个 1bit codebook 叠加，但存储上还是按 1bit 看）
- 非显著权重：直接 group 内 1bit + scale

判断谁重要，仍然继承 GPTQ 的 Hessian 重要性。
本质上 BiLLM 是把 OBS→GPTQ→PB-LLM 这条线推到了"全 1bit 但还能用"的最低点。

留下的问题：BiLLM 是给稠密模型设计的。所有的"显著/非显著"判断都是按单个权重矩阵做。
碰上 MoE 怎么办？

** 第 4 层：ARB-LLM 系 —— 把误差再迭代精化一遍

2024 后半到 2025 这阵子的工作（ARB-LLM、BinaryMoS、I-LLM 等）大致都是在 BiLLM 框架上做"误差细化"：
量化一次后，把残差当成新问题再量化一次；或者交替更新 scale 和 binary code，让两边一起逼近最优。
本质是同一个思想：1bit 表达能力太弱，那就用多步骤的"逼近-修正"逼出更小的误差。

留下的问题：所有这些方法的视角都还在一个 dense 的矩阵上，看到的是"权重重要性"和"重建误差"。
MoE 模型多出两件东西：第一，多个专家之间有冗余（你不需要把每个专家都独立压一遍）；
第二，专家是被 router 选的——量化稍微动一点权重，router 就可能换人，激活分布就崩了。
这两件事 dense 量化体系完全没考虑过。

** 第 5 层（目标论文）：MoBiE —— 给 MoE 量身改三处

MoBiE 看到的问题，摘要里说得很清楚，正对应上面那两个 dense 时代不存在的麻烦：

*问题 1：专家间冗余*
MoE 一层动辄 8/64/128 个专家，每个专家是个独立的 FFN。
按 dense 的玩法，每个专家都被独立量化一次——其实它们之间共享相当一部分低频结构。
*MoBiE 的招*：把同层所有专家堆起来做 *联合 SVD*，把低秩共享部分单独拎出来（保留较高精度），剩下的高秩个性化残差再做 1bit。
等于先共享、后个性化，总比特数没增加但表达力强了。

*问题 2：重要性估计是 task-agnostic 的*
GPTQ/BiLLM 那个 Hessian 是从校准数据的"层输入"算出来的，是局部的、和最终任务 loss 无关。
对 dense 模型还能凑合，对 MoE 不行：MoE 的不同专家在不同 token 上被激活，每个专家对最终 loss 的贡献天差地别，纯靠局部 Hessian 评不准。
*MoBiE 的招*：在 Hessian 重要性里灌进 *全局 loss 梯度*——既看局部敏感度，也看这个权重对端到端 loss 实际有多大影响。

*问题 3：量化让 router 飘了*
MoE 的 router 是一个小线性层 + softmax，决定 token 去哪个专家。
专家权重一被量化，专家输出就变了，router 的"信号"就变了——同一个 token 量化前进专家 3，量化后可能进专家 7。
这种"路由漂移"不是简单的重建误差能 capture 的。
*MoBiE 的招*：给量化误差加一个 *输入 null space 方向上的约束*——说白了，要求量化误差尽量落在"router 看不见的方向"上。
路由实际依赖的是某些主方向；只要把误差挤到 null space 里，router 的输出就近似不变。

* 压缩总览

  问题/方法                        | dense 时代                | MoBiE (MoE)
  --------------------------------+--------------------------+----------------------------
  谁重要 (importance)             | 局部 Hessian (GPTQ)       | 局部 Hessian + 全局 loss 梯度
  怎么 1bit (binarization)        | 显著/非显著分两堆 (BiLLM)  | 联合 SVD 抽共享 + 残差 1bit
  量化误差怎么管 (error control)   | 最小化层输出 MSE (GPTQ 系) | MSE + 输入 null space 约束
  专家间关系 (cross-expert)        | 不存在                    | 联合 SVD 砍冗余
  路由稳定性 (routing)             | 不存在                    | null space 约束保护 router

  一句话：dense 量化关心"权重重建有多准"；
          MoBiE 多关心"专家间能不能省共享部分"和"router 会不会被你改飘"。

* 演化的真正主线

表面看，这条线是"压缩比一步步逼近 1bit"。
但更深一层的演化其实是 *"信息论视角不断换坐标系"*：

- OBS 时代：坐标系是"单个权重"——这个权重重要不重要
- GPTQ 时代：坐标系是"一层的输出空间"——这一层的输出误差是多少
- PB-LLM / BiLLM 时代：坐标系是"权重的二分类"——哪些权重该被特殊照顾
- ARB 时代：坐标系是"残差空间"——误差能不能再迭代地剥一层
- MoBiE 时代：坐标系换成了 *"router 关心的子空间 vs router 不关心的 null space"*

每次跨越，*重要的不是新方法本身，而是它把"重要性"重新定义在了哪个空间上*。
MoBiE 真正新的东西，不在于"用了 SVD"或"加了梯度"——这些都是工具。
新的是它第一次承认：*在 MoE 里，量化的目标不能只是"权重 / 输出像不像"，而要保住"路由像不像"*。

* 往下一步会去哪

如果上面这条主线成立，下一步合理的方向：

1. *Activation 也得这么搞* —— MoBiE 摘要只谈了 weight binarization。MoE 的 activation 同样有"路由相关"和"路由无关"两个子空间。Activation quant 应该也走 null space 路线。
2. *Router 本身参与量化目标* —— 当前是"量化完别让 router 飘"。下一步应该是"量化时让 router 一起优化"，把 router 的可微替代物（比如 Gumbel-Softmax 形式）直接放进 loss。
3. *专家级别的混精度* —— 联合 SVD 暗示了"专家之间有冗余"。再往下应该出现"按专家激活频次和重要性分配不同比特数"的方案——高频专家保 2bit，长尾专家压 1bit 甚至更低。

* 诚实标注

- *确认*：MoBiE 摘要原文给出"对 dense LLM 的现有 binary 方法存在 MoE-specific 三个问题"，以及三个对应解法的名称。这部分是论文自述，不是推测。
- *确认*：OBS→GPTQ→PB-LLM→BiLLM→ARB-LLM 这条 LLM 二值化主线是该领域的公开常识，可在各方法的 related work 里互相印证。
- *推测*：MoBiE 摘要没有列出具体引用的 baseline 名字，本文挑 BiLLM/PB-LLM/ARB-LLM 作为"被改进对象"是基于"现有的 dense LLM 二值化方法 = 这一串"的领域判断，未在原文中直接核对。
- *不可靠*：原论文被作者明确撤稿，并自承"NGES 部分推导错误"。NGES 很可能对应摘要中的第 2 项（"global loss gradients into local Hessian"）。因此第 2 项的内部数学可能站不住，这里的描述只能按"作者声称"理解，不能当结论。
- *找不到后续*：因为论文 2026 年 4 月才上线、4 月底就撤稿，没有明确的"批判 MoBiE"的后续工作。前沿延伸部分是基于演化逻辑的合理外推，不是引用关系。
- *递归层数*：往前找到 5 层（OBS / GPTQ / PB-LLM / BiLLM / ARB 系 → MoBiE），到 OBS 已是该领域的奠基性工作，可以停。
