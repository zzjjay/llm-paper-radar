* 问题之河

LLM 学会"思考"——做数学题、写代码、一步步推理——这两年靠的几乎都是强化学习。给模型一个题，让它生成几十条解题路径，对的奖励、错的惩罚，反复迭代。所有最强的推理模型——DeepSeek-R1、OpenAI o1、Qwen3——都是这么训出来的。

可是有件事一直说不通：RL 真的"教会了"模型新的推理招式吗？还是它只是把模型已经会的招式，挑出来用？

这个问题如果答错了，整个领域的算力就在烧错地方。一次 RL 训练要烧几千到几十万美元，几万张 GPU 小时。如果 RL 其实只是在"挑"而不是在"教"，那这笔钱大半是冤枉的。

* 演化叙事

** 2017 | AlphaZero: RL 真的能从零发明新策略

*** 它看到了什么问题

那年之前的下棋 AI，几乎都是"先学人类棋谱、再用 RL 微调"。监督学习先把人类的招式灌进去，RL 只是锦上添花。这导致一个隐含假设：RL 没法独立发明高质量策略，得靠人类先教。

*** 它的解法

DeepMind 干脆把人类棋谱全扔了。AlphaZero 从随机初始化开始，只靠自我对弈 + RL，下围棋、国象、将棋全到超人水平。围棋里它还发明了人类几千年没见过的招式（比如 5 路侵入、肩冲）。

*** 为什么这招管用（以及哪里埋了新坑）

棋类游戏的搜索空间巨大但规则封闭、奖励信号干净（赢/输二值）。RL 配上蒙特卡洛树搜索，能在指数级搜索空间里探出人类没探过的角落。

这埋的坑很大：后人把"RL 能发明新策略"当成普适真理。LLM 圈一看到 RLVR（RL with verifiable rewards，数学题答案是可验证的），立刻类比："那不就是 LLM 版的 AlphaZero 吗？RL 应该也能教 LLM 发明新的推理路径。" 这个类比是否成立，多年没人正面验证。

** 2024-2025 | GRPO / DeepSeek-R1: 把 AlphaZero 套路搬到 LLM 推理

*** 它看到了什么问题

LLM 在简单题上靠 next-token prediction 已经够用，但碰到需要多步推理的数学/代码题，pretrain 给的能力封顶。怎么把模型推上更高的台阶？

*** 它的解法

RLVR：让模型对每道题生成 G 条解答（rollout），用程序自动判对错（二值奖励），然后用 GRPO / PPO 这类策略梯度更新参数。DeepSeek-R1 把这套规模化，做出第一个开源的"会长推理"的模型，性能直追 OpenAI o1。

*** 为什么这招管用（以及哪里埋了新坑）

确实有效——pass@1 涨得很猛。整个领域立刻 all-in，几千万美元的算力往里砸。

但有件事悄悄出问题：所有的 RL 都对每条 rollout 的每个 token 平等地施加梯度。如果只有少数 token 真的需要被纠正，剩下的全是噪声，那就是在用大锤敲钉子。更深的问题：大家假设"RL 像 AlphaZero 一样在发明新招式"，但没人去验证 LLM 这边类比是不是成立。

** 2025 | Yue et al.: Pass@k 反证——base 早就会了

*** 它看到了什么问题

如果 RL 真的"教会"模型新的推理招式，那应该在 base model 完全做不出的题上看到突破——而不只是在 base model 偶尔做对的题上提高成功率。

*** 它的解法

他们换了个度量：不只看 pass@1（采样一次的正确率），还看 pass@k（采样 k 次至少有一次对的概率）。结果反转——k 大了之后，base model 的 pass@k 反而高过 RL model。

*** 为什么这招管用（以及哪里埋了新坑）

这说明：正确解题路径 *早就在 base model 的采样分布里*，RL 只是把这些路径的概率提上来，把其他路径的概率压下去。RL 像是"在已有答案里选"，不是"在没有的地方造"。AlphaZero 那个发明新策略的浪漫故事，在 LLM 这边不成立。

但这只是行为层证据。一个棘手的反驳是："也许 RL 还是在教新东西，只不过新东西碰巧也在 base 的 long tail 里。" 需要更硬的证据。

** 2025 | Davis-Recht & Zhang: 理论与边界——RL 只是 sharpen

*** 它看到了什么问题

Yue 的结果还能被解释成统计偶然。需要的是：从理论上 *证明* RL 在这个设定下只能做"概率重排"，以及从实验上找到 *什么时候* RL 有用 / 什么时候没用。

*** 它的解法

- Davis & Recht 证明：所有用二值奖励的主流 RL 算法（GRPO / PPO / REINFORCE），数学上都等价于在 "P(答对)" 的某个单调变换上做随机梯度上升。换句话说，它们的目标全是"把对的概率推高"。而这种优化 *只有在 base model 已经会一点点时才有意义*——base 从来没做对过的题，没有梯度信号可用。
- Zhang 在实验里印证：RL 提升只在"能力边缘"的题上发生——base 偶尔会、偶尔不会的那种。太简单（全对）或太难（全错）都没用。

*** 为什么这招管用（以及哪里埋了新坑）

到这一步，"RL 不教新东西，只是 sharpen 已有分布"基本被钉死了。但还有个具体问题没解决：RL 到底 sharpen 在 *哪里*？是把每个 token 的概率都微调一点？还是只在某些位置发力？如果是后者，是哪些位置？

** 2025 | Wang et al. (Forking Tokens): RL 的力其实集中在分叉口

*** 它看到了什么问题

如果 RL 真的只 sharpen 一部分位置，那对所有 token 平等地施加梯度就是浪费——大部分梯度是噪声。需要找到"真正起作用的那些 token"。

*** 它的解法

把生成时的每个 token 按 base model 的熵排序。高熵 token 意味着模型在那一步"不知道该走哪条路"——这就是 *分叉口*（forking token）。Wang et al. 把 GRPO 的梯度限制在 top 20% 熵最高的 token 上，结果发现：性能等同于对所有 token 做 RL。

*** 为什么这招管用（以及哪里埋了新坑）

直觉清晰：模型在低熵位置（"the answer is..."后面跟 "5"）已经很确定，没什么可学；高熵位置（要决定先用代数还是几何）才是真正决定解题路径的地方。把梯度信号集中在那里，效率提升 5 倍，效果不掉。

但他们留了个尴尬的局限：*他们依然在 RL 框架里*。只是把 RL 做得更省。如果 RL 的有用信号真的就这么稀疏、这么可定位，那为什么还要 RL 的 rollout-奖励-策略梯度这一整套昂贵循环？没人敢往下问。

** 2026 | ReasonMaxxer (本文): 既然信号这么简单, RL 就别要了

*** 它看到了什么问题

把前面这条链拼起来：
- Yue: RL 不创造新答案，base 都有
- Davis-Recht: 数学上 RL 就是在 P(对) 上做梯度上升
- Wang: 信号集中在 1-4% 的高熵 token 上

每一篇都在围着同一个真相打转，但都不敢迈出最后一步——*把 RL 本身扔掉*。这篇就是迈那一步。

*** 它的解法

三件事拼起来：

1. *机制刻画*：用 oracle 介入实验证明，RL 模型 vs base 模型的差异，*只在 1-3% 的 token 上发生*，且这些 token 永远在 base 的 top-5 候选里。在这些位置上换成 RL 偏好的 token，能精确复现 RL 的全部性能；随机换则不行。
2. *无 teacher 定位*：base model 自己的 token 熵，就能定位这些位置——根本不需要训 RL 模型来告诉你。
3. *ReasonMaxxer*：
   - 从 base 采样几百条 rollout（offline，不用 online RL）
   - 用 base 的熵筛出高熵位置
   - 在这些位置上跑对比学习（正确 rollout 的 token 拉高、错误的压低）
   - 其余位置用 KL 锚回 base，防止瞎飘
   - 只训一个 rank-32 的 LoRA adapter

*** 为什么这招管用（以及哪里埋了新坑）

结果惊人：跨 3 个模型族、6 种规模、6 个数学 benchmark，*50 道题、几百条 rollout、几分钟单卡训练，性能匹配甚至超过烧了 $1,200 - $103,000 的 RL 基线*。训练成本压低三个数量级。

埋的坑：这是数学推理任务。代码、长链 agent 任务、多轮对话——这些任务里"信号是否也这么稀疏"还没验证。如果换个任务里 RL 真的在教新东西，这套方法就不灵了。

* 前沿延伸

ReasonMaxxer 是 2026-05 的最新进展。同时期相邻的工作可以作为旁证或补充：

- *TINA (Wang et al., 2025b)*：从另一个角度逼到同一个结论——用 LoRA 约束下的 RL，性能能匹配全参数 RL。说明 RL 的修正本身就是低维的、能塞进 1% 参数里。这是 ReasonMaxxer "用 LoRA 装下 RL 全部修正"的同期证据。
- *Karan & Du (2025) - Reasoning with Sampling*：完全不动模型，只在推理时改采样策略，也能提升推理。和 ReasonMaxxer 互补：一个证明"训练时不用 RL"，一个证明"连训练都不用，光改解码就行"。
- *Agarwal et al. (2025) - Entropy Minimization*：无标签做熵最小化也能提升推理。和"高熵 token 才是关键"的发现互相印证。

这条线接下来最可能的方向：
1. *把 ReasonMaxxer 推到代码、agent、长链推理*——验证"sparse policy selection"是不是 RL 的普遍规律，还是只在数学题上成立。
2. *甚至连 LoRA 都省掉*：既然 RL 修正是 0.3-0.5% 的参数，能不能直接做 inference-time activation patching？
3. *倒回去重新审视 RLHF*：人类偏好的 RL 训练（不是可验证奖励），是不是也是同样的稀疏结构？如果是，那 RLHF 的算力也大半是冤枉的。

* 洞见

读完这条线，真正在发生的不是"RL 方法的迭代"，是 *领域对自己核心信念的解体过程*。

最初的信念："RL 像 AlphaZero 一样，能教 LLM 学会新的推理招式。"

这个信念支撑着整个行业的算力投资——几千万美元、几万张 GPU 卡、几万个工程师小时。

然后一篇一篇论文，从不同角度往这个信念上敲：
1. 行为层敲：base 的 pass@k 反而高（Yue）
2. 理论层敲：RL 数学上就是个概率 sharpener（Davis-Recht）
3. 边界层敲：只在能力边缘有用（Zhang）
4. token 层敲：信号集中在 1-4% 的位置（Wang）
5. 参数层敲：修正能塞进 0.3% 的参数（TINA / 本文）
6. 算法层敲：不用 RL 也能拿到这些（ReasonMaxxer）

到 ReasonMaxxer 这里，那个最初的信念基本碎了。

更深层的转变是 *研究方法论* 的转变：从"造更强的训练算法"转向"先搞清楚算法到底在做什么"。前者是工程式 scale-up，后者是科学式的机制研究。ReasonMaxxer 整个第 3、4 节都是 mechanistic interpretability 风格的实验——先做 oracle intervention 证因果，再做 LoRA distillation 证低维。把"算法做了什么"搞清楚之后，新的高效方法是自然推出来的。

下一步最可能往哪走？两条：

- *把 sparse policy selection 当成假说去检验其他设定*：RLHF、code RL、agentic RL 是不是也都是这个结构？如果是，整个 post-training 的成本都能压一两个数量级。
- *机制研究先于方法创新成为常态*：以后做 post-training 论文，"先做 mechanistic analysis，再设计方法"会成为标配。"我训了个更大的 RL"的论文会越来越没竞争力。

* 启发

这套读法对"怎么做研究"的启发：

1. *最贵的假设最值得验证一次*。这个领域几千万美元烧在"RL 能教新东西"这个假设上，*却几年没人正面验证*。如果你工作的领域有个"大家默认成立、但其实没人证明过"的假设，那就是金矿。
2. *别在 RL 框架内做局部优化, 跳出框架看*。Wang et al. 发现 80-20 之后，停在了"怎么让 RL 更高效"。本文的关键一步是问"既然信号这么稀疏，那 RL 本身还要吗？" *把"省 X" 升级成 "不要 X"*，往往是真正的突破。
3. *机制研究就是方法创新的源头*。这篇先做了两节硬核的 mechanistic analysis（sparse + low-dim），然后整个方法（ReasonMaxxer）几乎是从机制推论自然落出来的。机制清楚了，方法不用拍脑袋。
4. *小样本 + 单卡 + 几分钟* 这个数字本身是个杠杆。它把"RL 是必需的"这个信念彻底打穿——任何还在说"我们必须用 RL 因为 X"的论文，都得回答："那为什么 ReasonMaxxer 用 $5 也能做到？"
5. 对你具体能用：如果你在用 RLVR / GRPO / PPO 做推理任务的 post-training，*先试 ReasonMaxxer 这套基线*。成本几乎为零，万一 work，省下的预算可以投到真正缺数据/缺机制理解的地方。
