* 问题之河

大模型推理时, 上下文每多一个 token, KV cache 就长一截。长上下文场景里, 显存是先被这堆 cache 撑爆的, 不是被算力卡住的。

于是过去三年里, 一整条研究线只在做一件事: *把 KV cache 砍掉一大半, 让模型还能答对*。

砍 cache 有两个子问题:

1. *每个注意力头分多少预算*? (budget allocation) ---- 全部头平分? 浅层多深层少? 还是看着办?
2. *预算之内, 留哪几个 token*? (token selection) ---- 看注意力分数? 看 key 向量? 看历史累计?

这两个问题三年里被各种方法各种 *启发式规则* 来回试。每篇都涨一点点, 但每篇都拍脑袋 ---- 凭什么浅层就该多? 凭什么"前几个 token 是 attention sink 必须留"? 凭什么用最近 32 个 token 当观察窗?

LKV 的论断很狠: *这些 heuristic 全是绕路。直接让任务损失反向传播, 自动学出"哪个头分多少预算、留哪些 token", 一行手工规则都不要*。

听起来理所当然 ---- 这不就是 deep learning 的标准做法吗? 但过去三年没人这样做, 是因为 "选 top-K" 这个操作 *不可微*, 端到端学不了。LKV 解开这道结的关键是一个叫 *Soft-TopK* 的可微算子, 加上一个能从 KV 向量直接打分、不需要 attention 矩阵的选择器。

整条故事就是: 大家怎么一步步从"全部均分"走到"看着办"走到"学着办"的。

* 演化叙事

** 2023.09 | StreamingLLM: 第一次承认"必须留点什么"

*** 它看到了什么问题

GPT-3.5 刚出来时大家就发现一个怪事: 你用 sliding window 跑长上下文, 每次只保留最近 1024 个 token, 模型很快就"傻掉" ---- 输出胡言乱语。

按理说, 只要最近的 1024 个 token 在, 模型至少能续写得通顺。为什么会崩?

*** 它的解法

Xiao 等人调出 attention 热图, 看到一个反常的细节: *第一个 token (BOS 或者别的) 永远获得异常高的注意力分数*, 不管语义多无关。

他们管这个叫 *attention sink* ---- 像水槽, softmax 必须把概率分到某处, 头几个 token 就成了那个"垃圾桶"。一旦把它扔了, softmax 分布失衡, 整个推理就坏。

解法极简: *永远留前 4 个 token + 一个滑动窗口*。能跑百万长度, 性能不掉。

*** 留下的新问题

这招把"显存爆炸"压住了, 但 *漏掉了所有远古的关键信息*。如果用户在 100K token 之前埋了一个事实, 模型答不出。

更深的问题是: StreamingLLM 默认 *远古信息 = 无关信息*。但显然不是 ---- 真正的长上下文应用 (代码库分析、多文档 QA) 里, 关键信息可能在任何位置。

球传给了 H2O。

** 2023.10 | H2O: 把"留谁"从位置规则换成分数规则

*** 它看到了什么问题

H2O 团队从 StreamingLLM 那里继承一个观察: attention 是 *稀疏* 的 ---- 大部分 token 在大部分时间不被关注, 少数几个反复被关注。

那为什么要按位置 (最近的留下) 来决定保留? 应该按 *是否被反复关注* 来决定。

*** 它的解法

给每个 token 记一笔账: *它历史上累计被 attend 了多少分数*。每次要驱逐时, 留分数最高的前 B 个 ("Heavy Hitters"), 其余扔掉。

预算分配上不做花样: 所有层、所有头, 平均分配总预算。简单。

*** 为什么有效, 哪里埋了坑

有效是因为它确实抓住了 attention 的稀疏结构 ---- 删低分 token 几乎不损害输出。

坑有两个, 都被后面的论文一一捅破:

- 第一个坑: *累计分数反应慢*。prompt 末尾刚出现一个关键指令 ("用中文回答"), 它的累计分数还没涨起来, 就被当成不重要扔了。这个坑 SnapKV 来填。
- 第二个坑: *全部头均分预算太傻*。注意力头的角色差异巨大 ---- 有些头管语法, 有些头管远距离检索, 有些头管局部模式。给它们等分预算, 等于让饿的人和饱的人吃同样多。这个坑 PyramidKV、Ada-KV 一路接力填。

** 2024.04 | SnapKV: 看末尾窗口, 别看整个历史

*** 它看到了什么问题

SnapKV 发现一个更精细的现象: *prompt 处理完的瞬间, 模型最后几个 token 的 attention pattern, 已经"暴露"了后面要 attend 谁*。

H2O 用全历史累计, 把这个最关键的"末窗口信号"和远古噪声平均掉了。

*** 它的解法

只看 prompt 最后一个固定窗口 (比如最后 32 个 token) 的 attention, 算每个历史 token 被这个窗口 attend 的频率。频率高的留下。

一句话: *把"它历来重要吗"换成"它对马上要问的问题重要吗"*。

预算分配仍然全部头均分 ---- 这条线 SnapKV 没动。

*** 留下的新问题

涨点很猛, 但暴露出一个核心矛盾: *选预算之内留谁的策略变聪明了, 但"每个头分多少预算"还停留在 H2O 时代*。

接下来三年, 主战场转移到 budget allocation。

** 2024.05 | PyramidKV: 浅层多、深层少, 写个固定曲线

*** 它看到了什么问题

PyramidKV 团队跑了大量 attention 可视化, 发现一个规律: *浅层的 attention 分布更"散" (entropy 高), 深层更"集中" (entropy 低)*。

物理意义: 浅层在做信息聚合, 需要看很多 token; 深层在做特定推理, 只看少数关键 token。

那为什么浅层和深层分一样多的预算? 应该 *给浅层多分、深层少分*。

*** 它的解法

写一个金字塔型的线性衰减: 第 1 层预算最大, 第 L 层预算最小, 中间按等差递减。具体衰减斜率是个手工超参。

*** 为什么有效, 哪里埋了坑

有效, 是因为它确实抓住了"层间差异"这个真实信号。同等总预算下, 比 H2O/SnapKV 涨一个点左右。

坑也立刻被下一篇论文指出来:

- *金字塔太死*。现实里, 有些"检索头"恰恰在中间层或深层 ---- LongBench 上一类问题 (haystack 类) 的关键头就在 16-20 层。强行让深层预算低, 把这些头饿死了。
- *没考虑头内差异*。同一层里, 不同头干不同事, 都用同一个层预算又等于回到 H2O 的均分模式 (只是层间不均分了)。

球传给 Ada-KV 和 DuoAttention。

** 2024.10 - 2025.02 | DuoAttention 和 Ada-KV: 第一次让"学"进来

*** 它们看到了什么问题

PyramidKV 用固定衰减曲线, 这就是个 *写死的 prior*。无论任务多需要深层预算, 都不肯给。

这两篇论文同时意识到: *应该用学习, 而不是写死规则*。但走的是两条不同路。

*** DuoAttention 的解法 (Xiao et al., 2024.10)

每个头训一个 *门控 0/1 分数*, 通过 distillation 学出来 ---- 接近 1 的头被分类为 *Retrieval Head* (检索头, 必须全保留 KV), 接近 0 的头被分类为 *Streaming Head* (流式头, 只保留 attention sink + 近窗)。

简洁、可解释、效果好。但 *只学二分类*。

*** Ada-KV 的解法 (Feng et al., 2025.02)

跑推理时, 用当前 prompt 算每层的 attention 分布, 按 *attention 集中度* 自适应地把层预算切分给不同头 ---- 集中度高的头说明"很专一", 分多一点; 散的头说明"不挑食", 分少一点。

不需要训练, 推理时动态自适应。但 *依赖算 attention 矩阵*。

*** 为什么有效, 哪里埋了新坑

DuoAttention 第一次把"学"引入这个问题, 证明了端到端学的潜力。Ada-KV 第一次做到了"每个头自己的预算"细粒度自适应。两条线都涨点。

坑还在:

- *DuoAttention 的二分类太粗*。头的真实角色远不止"检索 vs 流式" ---- 有局部头、归纳头、复制头、ICL 头、归纳偏置头, 一个 binary 切不下来。预算粒度其实只有两档。
- *Ada-KV 把自己困回了 O(t^2)*。它要算 attention 来分预算, 而我们做 KV 驱逐的根本目的之一就是 *避开 O(t^2) 的 attention 计算*。这是一个 *循环依赖*: 为了省内存得驱逐, 为了驱逐得算 attention, 算 attention 又费时间。
- *都需要兜底规则*。Ada-KV 强加最小预算阈值 (防止某些头被饿死), DuoAttention 强加 sink + 近窗 (保流畅性)。这些都是 "*学是学了, 但还得手工兜底*" 的表现 ---- 说明学的不彻底。

球传给 LKV。

** 2026.04 | LKV: 把所有手工规则一次性拿掉

*** 它看到了什么问题

读完前面五年, LKV 团队的判断很直接: *这个领域一直在做"半监督式"的优化 ---- 学一点, 兜一点, 学一点, 兜一点*。

为什么不直接 *端到端反向传播, 让任务损失自己决定每个头分多少、留哪些 token*?

之所以以前没人这样做, 是因为 "选 top-K" 是个 *离散* 操作 ---- 不可微, 梯度传不回去。所有"学习"成分到了 top-K 这一步都断链。

LKV 的核心是: *找一个可微的 top-K 代用品*, 把整个 pipeline 串成一根可微的链。

*** 它的解法

三件事:

*1. Soft-TopK 算子 (核心数学工具)*

把硬 top-K 换成一个用拉格朗日乘子 lambda 控制的可微 sigmoid 选择: 每个候选 token 得到一个介于 0 和 1 之间的"保留概率", 总和约束为 K。lambda 由一个闭式方程解出。训练时这个算子有梯度, 推理时直接 hard top-K (因为概率几乎都是 0 或 1)。

这个工具把整个问题从"离散组合优化"变成"连续可微优化"。所有后续设计都建立在它上面。

*2. LKV-H: 全局打平的预算竞争*

不再按"层内分配"做。把所有层所有头 *打平成一个长队列*, 给每个头配一个可学的 embedding, 算一个分数, 然后用 Soft-TopK 在 *全局* 选谁分多少预算。

这就把 PyramidKV 的金字塔、DuoAttention 的二分类、Ada-KV 的层内自适应 *全部解构了* ---- 哪个头需要多少, 任务损失说了算, 没有层级 prior、没有 binary、没有最小阈值。

*3. LKV-T: 不算 attention 矩阵的 token 选择器*

这是 Ada-KV 那个 O(t^2) 循环依赖的解药。给每个头加一个轻量 MLP: 输入 KV 向量本身, 输出一个 token 重要性分数。然后 Soft-TopK 选预算之内的 token。

*关键: 完全不需要算 attention 分数*。复杂度从 O(t^2) 降到 O(t)。同时也避免了"为了选 token 必须知道未来 query"的循环困境。

*** 为什么有效, 论文给了什么新发现

实验上 LKV 在 LongBench、RULER 等基准上 SOTA, 15% cache 留存就近无损。这个数字本身没什么稀奇 ---- 每篇 KV 驱逐论文都说自己 SOTA。

真正有意思的发现在 *ablation* 里: 作者把 LKV-H (学预算) 和 LKV-T (学选择) 分开消融, 发现 *性能增益主要来自 LKV-H*, 而不是 LKV-T。

翻译一下: *过去三年大家把功夫主要花在"选哪些 token"上, 但真正卡住性能的是"每个头分多少预算"*。SnapKV、KeyDiff、CriticalKV 这些 token 选择上的迭代, 边际收益其实在递减; 而 budget allocation 从均分到 PyramidKV 到 Ada-KV 到 LKV-H, 才是性能跃迁的主战场。

这个 ablation 结论, 比 LKV 本身的方法还有价值。

*** 留下的新坑

- *Soft-TopK 训练需要标注数据 (self-distillation)*。这要求有一个未压缩的 teacher 模型, 训练成本不低。
- *LKV-T 的 MLP 是 per-head 的*, 参数随头数线性涨。对超大模型 (MoE 之类) 不一定 scale。
- *端到端可微的代价是失去 inference-time adaptability*。Ada-KV 还能根据当前 prompt 动态调整, LKV 训完就固定了 ---- 不同任务可能需要不同的预算分布, 但 LKV 学的是数据集平均最优。
- *Soft-TopK 的近似有多紧?* 训练时是 soft 的, 推理时切换到 hard, 这个 gap 有多大? 论文有理论分析但没在所有 setting 里严格量化。

* 前沿延伸

LKV 2026 年 4 月才发布, 太新, 还没明确的后续论文。但顺着这条河往下看, 几个方向是必然的:

- *Inference-time adaptive 学习预算*。 LKV-H 学的是 "对数据集而言最优的全局预算分配", 但不同 prompt 需要的预算分布是不同的。下一步是 "学一个 hyper-network, 根据 prompt 动态输出预算分配", 兼有 Ada-KV 的灵活性和 LKV 的端到端性。
- *跨任务的预算 routing*。 既然 LKV-H 是全局学的, 那很自然的扩展: 不同 task 类型 (代码 / QA / summarization) 走不同的预算路径。这其实是 MoE-of-budgets。
- *LKV-T + quantization 联合优化*。 LKV 只压 cardinality (留多少 token), 不动 precision (每个 token 几 bit)。同一可微框架天然能容纳 bit-width 维度 ---- KIVI、AQUA-KV 这一族能合并进来。
- *Soft-TopK 推广到其他离散选择*。 这个算子本身是个通用工具, MoE 的 expert routing、speculative decoding 的 draft token 选择、retrieval 的 chunk selection, 都可以套同一套可微化思路。

更远一点: LKV 实际上提出了一个 *框架级的转变* ---- "凡是 KV cache 管理里能写成 top-K 选择的地方, 都可以端到端学"。如果这条思路扩散开, 那么过去三年所有"加 heuristic 加 prior"的论文都会变成一种过渡形态。

* 洞见

读完这条线, 表层是"KV cache 压缩越做越精细", 骨子里有三层更深的转变。

*第一层: 从"写规则"到"学规则"的迟到觉醒*。 整个深度学习自 2012 年以来的核心信条就是 "学比写好"。但 KV 驱逐从 2023 年到 2025 年, 大家都在堆 heuristic、写 prior、加兜底, 几乎没人尝试端到端学。原因是技术性的 (top-K 不可微), 但更深的原因是 *惯性* ---- 系统优化领域出身的人习惯写规则, 学界的范式没传过去。LKV 不是发明了什么新技术 (Soft-TopK 在别的领域早有), 它只是把 *深度学习的标准做法* 严肃地用到了这个问题上。下次进入一个"由系统/工程师主导的领域"时, 问自己一句: *这里有没有人还在写 heuristic? 端到端学的范式过来了吗?*

*第二层: 真正卡住性能的瓶颈, 经常不是大家在卷的那个*。 这条线上, 从 2024 到 2026 年, *绝大多数论文在做"更好的 token 选择"* (KeyDiff、CriticalKV、CapKV、EntmaxKV、Conf-KV...)。LKV 的 ablation 说: 真正的瓶颈是预算分配, 不是 token 选择。这种"领域共识下其实有错误的优先级"的情况, 在工程领域反复出现 ---- 量化领域, 大家卷激活量化精度, 卡住的其实是 outlier handling; 检索领域, 大家卷 embedding 模型, 卡住的其实是 chunk 切分。 *选战场比选战术更重要*。当一个领域有十个团队都在做同一件事时, 真正的 alpha 往往在它们都没盯的那个角落。

*第三层: 可微化作为一种"领域升维"工具*。 LKV 的本质是 *把一个离散组合优化问题 (从 N 个 token 选 K 个) 变成连续可微优化*。这个套路在 ML 里反复出现: Gumbel-Softmax 把离散采样变可微 (推动了 VAE + categorical 后续工作), DiffSort 把排序变可微 (推动了 differentiable retrieval), 现在 Soft-TopK 把 top-K 变可微 (推动了端到端 KV 驱逐)。*每一次"离散 -> 可微"的转化, 都把一个领域从手工规则推到端到端学*。如果你看到一个领域里到处是 "argmax + heuristic", 想想: 能不能把那个 argmax 用一个可微算子替掉? 那可能就是下一个 LKV。

* 启发

对做研究和做工程的人, 这条线有几条具体启发:

*一、惯性思维比技术难度更难突破*。 端到端学 KV 驱逐, 技术上 Soft-TopK 在 2018 年的 differentiable sorting 论文里就有雏形, 但 KV 驱逐领域到 2026 年才有人这样做。中间隔了八年。原因不是技术不存在, 是 *系统优化领域的研究者还没把它捡起来*。 如果你跨界 (从 ML 进系统、从系统进 ML), 你的最大 alpha 不是发明新东西, 而是 *把另一边的标准工具搬过来, 用第一性原理重做一遍*。LKV 就是把"端到端学习"这个 ML 标准工具搬进了"系统优化"问题。

*二、Ablation 是论文里最值钱的一节, 而且通常被读者跳过*。 LKV 的方法描述里 LKV-H 和 LKV-T 并列, 看起来同等重要; 但 ablation 揭示 LKV-H 才是真正驱动性能的。 *任何一篇带 ablation 的论文, 先读 ablation 再读 method, 你会发现作者自己讲的故事和数据讲的故事经常不一样*。这是判断"作者真知道为什么 work"还是"作者凑了一堆 trick 都试了一遍"的最快方法。

*三、当一个领域里到处是手工兜底规则, 那就是端到端学的入口*。 Ada-KV 要最小预算阈值, DuoAttention 要 sink + 近窗, PyramidKV 要金字塔斜率。每个兜底规则都是一个 *"我们其实不知道这里该怎么办, 凭经验拍一个"* 的信号。如果你能数出五个以上这种兜底, 那基本可以断定: 端到端学一个版本能赢。 *找入口的最快方法, 不是看哪里 SOTA 没人超越, 而是看哪里 hyperparameter 还在拍脑袋*。

最后一句: *倒读法的真正价值, 是让你看见一个领域的"工程惯性曲线"*。LKV 看起来是一个"又一个 KV 驱逐方法", 倒读到 StreamingLLM, 才看见这个领域用了三年才走完"从写规则到学规则"这条 ML 里早已铺好的路。下次进任何一个新领域, 先问: *它的工程惯性是什么? 哪一步还没走?* 那一步, 就是下一个 LKV 的位置。
