* 一句话

把 GPTQ 那套"用 Hessian 把量化误差传到邻居身上"的精巧机器拆掉，只留对角线，发现在 2-bit 这种极端低位宽下反而稳了 7 个百分点。

* 五层倒读：每一篇看到了什么

** 层 5（1990）OBD：第一次承认"不是所有权重都一样重要"

* 问题 *：神经网络训完很大，要剪掉一些权重。可是剪谁？最朴素的做法是"剪绝对值最小的"，但这忽略了权重在 loss 曲面上的"位置"——一个小权重可能正好处在很陡的方向上，剪了就完蛋。

* 解法 *：用二阶导数（Hessian 对角线）打分。直觉是：loss 在权重 w_j 附近的二次近似，权重变化带来的 loss 增量大约是 (1/2) * H_jj * (Δw_j)^2。所以 H_jj 大的权重一动就疼，H_jj 小的权重可以放心剪。

* 留下的问题 *：只看对角线，等于假设权重之间互不相干。但 Hessian 的非对角项其实在说"如果你剪了 w_j，可以同时调一调 w_k 来补偿损失"——OBD 放弃了这个补偿。

** 层 4（1993）OBS：把对角线扩成全 Hessian，剪一个补一片

* 看到了 OBD 什么问题 *：OBD 剪完就走，不利用"剩下的权重还可以微调"这个信息。Hassibi 想：既然有全 Hessian，那剪一个权重 w_j 之后，剩下所有权重的最优微调量也能解析地写出来——一个拉格朗日乘子问题。

* 解法 *：保留整个 Hessian H。剪 w_j 时，同时把其他权重沿着 H^{-1} 的第 j 行（也就是非对角项告诉我们的"耦合方向"）做一个补偿位移。剪一个、补一片，loss 增量比 OBD 小一个数量级。

* 留下的问题 *：要存 d×d 的 Hessian 和它的逆，d 是层的输入维度。1990 年代的小网络勉强能算，到几十亿参数的 LLM 上每层 d=4096 甚至更大，这事变得不可能直接做。

** 层 3（2022a）OBQ / OBC：把 OBS 从"剪枝"搬到"量化"

* 看到了 OBS 什么问题 *：OBS 本来是为剪枝设计的——把权重设为 0。但同样的二次近似框架可以用来描述"把权重设到最近的量化格点"。Frantar & Alistarh 重写：原来"剪 w_j"换成"把 w_j 量化到 q_j"，剩下的补偿公式照搬。这就是 Optimal Brain Quantization。

* 解法 *：层一层做（layer-wise reconstruction），每层目标变成 min ||WX - Ŵ X||²。这正好可以用 OBS 那套二阶式来解，每个权重依次被量化，剩余权重按 H^{-1} 补偿。

* 留下的问题 *：每量化一个权重就要更新一次 H^{-1}（按 OBS 公式做秩-1 更新），还要重新选下一个最不疼的权重——总复杂度 O(d_col^3)，对 LLM 来说几小时跑不完一层。

** 层 2（2022b）GPTQ：让 OBQ 在 LLM 规模跑得动

* 看到了 OBQ 什么问题 *：OBQ 每步都要找"当前最不疼的列"，导致每一步都不一样的顺序，缓存不友好，O(d^3) 太慢。Frantar 等人发现：在 LLM 上，列的处理顺序其实差不多——干脆固定从左到右处理。

* 解法 *：
1. 列的处理顺序固定（一次性预定）
2. H^{-1} 用 Cholesky 分解预先算好，避免反复求逆
3. 按块批量处理，能在 GPU 上几小时量化完一个 175B 模型

这让"用 Hessian 信息做 LLM PTQ"第一次变得实用，GPTQ 几乎成了 LLM 量化的事实基线。

* 留下的问题 *：GPTQ 假设 calibration set（通常 128~512 条句子）算出来的 H = X X^T 已经足够准。但低位宽（2-3 bit）下，量化误差很大，需要"把误差精确传播"，对 H 的准确性要求一下变高了。这时人们开始隐约感觉：GPTQ 的 perplexity 看起来还行，downstream 任务却掉得离谱（Lin 2024、Lee 2024 都报告过这点）。但没人说清楚为什么。

** 层 1（2026）DASH-Q：原来非对角项是噪声

* 看到了 GPTQ 什么问题 *：作者做了一个很朴素的实验——拿两批不同的 128 条 calibration，分别算 H，比一下。结果对角线很稳，非对角项漂得到处都是。换句话说，GPTQ 那套精巧的"非对角补偿"其实在补偿采样噪声，不是在补偿真正的曲率。

更狠的是：H 有 d^2 个非对角项、只有 d 个对角项，前者数量级压倒后者，所以 H 整体的估计误差几乎全来自非对角项。在 2-bit 这种极端场景下，量化误差需要靠 H^{-1} 来传播——传播的方向本身就是噪声，量化误差被放大不是被消除。

* 解法 *：把 H 砍成对角阵 D = diag(h_jj)，其中 h_jj = Σ x_jk^2 就是输入特征第 j 个通道的能量。这一刀下去：
- 不同输入通道的优化彻底解耦，变成 d 个独立的一维加权最小二乘
- 每组（group）的 scale s 和 zero-point z 有解析最优解
- 加一个 ridge 项 λs² 防止稀疏组里 s 退化

听起来朴素到不像 2026 年的工作——就是"weighted-LS with weights = activation power, solved per group, in closed form"。但实验数据说话：Llama-3.1-8B 上 2-bit/32 group，GPTQ 平均 zero-shot 掉到 38%，DASH-Q 直接拉回 50%+。

* 留下的问题 *：DASH-Q 显式承认这是"用偏差换方差"——它给了一个偏的 H 估计，但稳定。这意味着在 calibration set 足够大（几万条以上）的时候，全 Hessian 方法应该会反超。论文没在这个边界上做对比。另外，DASH-Q 和那些用正交旋转把 outlier 摊平的方法（QuaRot, SpinQuant）是正交的——两个方向能不能叠加，是开放问题。

* 前沿延伸：DASH-Q 之后

** QuaRot / SpinQuant / QuIP# 这条线（2024–2026）

它们不质疑 GPTQ 的 Hessian，而是从另一头攻：先用一个正交矩阵把激活值的 outlier 摊平到所有通道（旋转后 X 的方差更均匀），再让 GPTQ 去量化。本质是"让 H 变得更友好"，而不是"换一个 H"。

DASH-Q 论文承认这条线 complementary——意味着你可以先 QuaRot 旋转，再用 DASH-Q 做对角加权 LS，理论上能叠加。但论文没做。

** 这条河往哪流

整条线的演化轨迹其实是一个有趣的来回：

| 阶段 | 对 Hessian 的态度 |
|------+-------------------|
| OBD  | 只敢用对角线 |
| OBS / OBQ / GPTQ | 全 Hessian，越精细越好 |
| DASH-Q | 又退回对角线，但理由完全不一样了 |

OBD 用对角是因为"算不动全 Hessian"。DASH-Q 用对角是因为"全 Hessian 估出来是噪声"。同一个公式，三十年绕了一圈，理由变了——前者是工程妥协，后者是统计洞察。

* 内核：这条线背后真正在发生什么

表面上是 LLM 量化技术的迭代。深一层是 *"什么时候该相信你算出来的数"* 这个问题反复被重新问。

OBS/GPTQ 假定：拿到的 Hessian 估计是可信的，所以精细利用它是最优的。这在 1990 年代小网络、海量训练数据的情况下成立。但 LLM PTQ 把场景倒了过来——模型超大（H 维度爆炸），calibration data 超小（百量级），统计可信度被压到极限。

DASH-Q 实质上是把统计学里的 *偏差-方差权衡* 引入到 PTQ 设计。它的关键不是"对角 Hessian 这个方法"，而是"先证明非对角不可信，再放弃它"。这个分析框架（Hessian shrinkage estimator, batch-to-batch stability metric）比方法本身更值得记。

下一步最可能的方向：
1. *自适应 shrinkage*：根据 calibration 数据量自动决定 ρ（非对角项保留多少），而不是 0 或 1 二选一
2. *对角 + 正交转换的叠加*：QuaRot/SpinQuant 把激活摊平，让对角假设更准确，叠加在一起
3. *把同样的分析挪到 KV cache / activation 量化*：这两个场景 calibration 同样稀缺
