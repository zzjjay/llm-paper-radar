* 一句话
推理模型变得越来越啰嗦——OpenAI o1、DeepSeek-R1 这一类会"出声思考"，
一道简单题也要吐几千个 token；这条线讲的是过去两年里大家是怎么一步步
学会"让模型自己学会闭嘴"的，从硬塞 token 上限，到看难度发力，
最后落到 CRISP 这一招：让模型拿"简洁版的自己"当老师，再自己拜自己。

* 主角是谁
真正的主角不是 CRISP，是一个问题：
*怎么让会思考的模型，少说点废话，又不变笨。*

CRISP 只是这条线上当前最干净的一个答案。

* 演化叙事: 问题怎么长出来的

** 第 0 层: 推理模型学会了"出声思考",但也学会了啰嗦

2024 年底,OpenAI 放出 o1,接着 DeepSeek-R1、Qwen3、Gemini 2.5、s1 一窝蜂跟上。
这批模型的共同点:答题前会先吐一大段"内心独白"——枚举情况、自我怀疑、
重新计算。这个东西真的有用,benchmark 分数蹭蹭涨。

但代价也来了。简单题它也照样唠叨。问"1+1 等于几",
它会先说"让我想想,加法的本质是......",几百个 token 下去才给答案。
推理 token 直接决定推理成本,这事不能忍。

研究者管这个叫 *overthinking*。问题立住了:
*能不能让模型只在该深思的时候深思,平时干脆利落?*

** 第 1 层 (路 A): 硬上 RL,给它定个 token 上限

最直觉的办法:训练时直接告诉模型"超过 N 个 token 扣分"。

- *L1* (2025) 在 GRPO 训练循环里加 token 数惩罚。
- *ThinkPrune* (2025) 在 RL 里同时约束长度和正确率。
- *DLER* (2025) 嫌一刀切太粗暴,加了课程学习——先松后紧。
- *Leash* (2025) 用 sigmoid 形状的奖励,让惩罚平滑些。

这条路有效。但它踩了三个雷:

1. *离不开标准答案*——RL 要算正确率,正确率要对答案。没标注的领域用不了。
2. *会把模型逼僵*。Xu 等人 (2026) 发现,RL 长期下去会让模型"过度自信",
   该探索的不探索了,推理多样性掉得厉害。
3. *上限是死的*。难题简单题一视同仁,简单题压不够,难题被腰斩。

第 3 个问题最致命。下一层就是冲着它去的。

** 第 1 层 (路 B): 不一刀切,看题目难度发力

既然简单题该多压、难题该少压,那就让算法自己分。

- *DiPO* (2026) 用模型自己的 rollout 通过率当难度信号:通过率高的题压狠点。
- *DIET* (2025) 用同样的思路,给每道题单独定 token 预算。
- *LiteCoT / DAP* (2025) 换条路——直接从更强的模型那里蒸馏简洁解法。

这条路比 A 聪明,但又多了两个新麻烦:

1. *要算难度*。算难度本身就贵——你得让模型把每道题做好几遍才能估通过率。
2. *LiteCoT 这种又引入了 distribution shift*。学生学的是老师的口吻,
   不是自己的口吻。在 RL 圈子里这个问题反复出现:用别人家模型的
   reasoning trace 训自己的模型,经常学了形式忘了内核。

到这里,大家心里浮起一个问题:
*难度调节这件事,能不能不显式地算,让它自己冒出来?*

** 第 1 层 (路 C): 干脆让模型给自己当老师——自蒸馏

自蒸馏 (self-distillation) 是另一条更早就有的脉络。
基本思路:同一个模型,两个角色——一个当老师生成目标,一个当学生学。

- *SDFT* (2026, Shenfeld 等) 早就证明:自蒸馏的 SFT 比标准 SFT 不容易遗忘老技能。
- *OPSD* (2026, Zhao 等) 把这招带到推理压缩:老师写示范,学生学。
  *但老师有标准答案*——又把"要 ground-truth"这个原罪带回来了。
- *SDPO* (2026, Hübotter 等) 用密集反馈做信用分配,也要监督。
- *PACED* (2026) 按学生能力加权,但能力的评估又要正误信号。
- *OPCD* (2026, Ye 等) 蒸馏 system prompt 行为——离 CRISP 已经很近,
  但没考虑难度自适应。

自蒸馏避开了 distribution shift——师生同源。但只要老师那一端要"答案、
难度、奖励"中的任何一个,就还在路 A、路 B 的坑里打转。

到这一步,所有牌都摊在桌上了。剩下的问题是:
*能不能找一种自蒸馏的姿势,什么外部信号都不要?*

** 第 2 层: CRISP 的关键一招

CRISP 的答案出乎意料地朴素:

#+begin_quote
老师就是给自己加了一句 "be concise" 的同一个模型。
#+end_quote

具体姿势:

1. 学生模型按正常 prompt 生成一段 rollout (一串思考 token)。
2. 同一个模型,加上 "请简洁" 这条指令,在 *学生写出来的同一串 token 上*
   重新算一遍每个位置的 logits——这就是"老师在这里会怎么想"。
3. 拿 *逐 token 的 reverse KL* 当 loss,让学生往老师那边靠。
4. 每隔一段时间用新学生刷新老师 (老师权重 = 当前学生权重)。

这里有几个微妙的设计:

- *没有标准答案*。老师不知道这题正确解是什么,只知道"该简洁"。
- *没有难度估计器*。但效果上简单题被压得多,难题被压得少——
  因为对简单题,"简洁版的我" 和 "啰嗦版的我" 分歧大,KL 信号强,压力大;
  对难题,两个版本的 token 分布本来就接近 (都得老老实实推),KL 小,压力小。
  *难度自适应是从 KL 几何里自动冒出来的,不是设计出来的。*
- *Reverse KL,不是 forward KL*。reverse KL 是 mode-seeking,
  会让学生集中在老师认可的高概率路径上,不会去模仿老师的整个长尾。
  这一选择直接影响输出的"果断度"。
- *On-policy*。蒸馏发生在学生自己生成的序列上,不是老师生成的。
  彻底切断 distribution shift。

整套东西没有 RL、没有奖励、没有外部模型、没有标注。
只有一个 "be concise" 的指令,和一个 KL 损失。

** 效果

- Qwen3-8B/14B、R1-Distill-Llama-8B 上跑。
- MATH-500: token 砍 57-59%,准确率 *涨* 9-16 个点。
- AIME 2024: 14B 模型上 token 砍 41%,准确率涨 10 个点。
- DeepPlanning (多步 agent 规划): token 砍 42-51%。

"压缩 + 涨点" 这个组合很反直觉。CRISP 给出的解释是:
*被砍掉的那些 token 本来就在干扰决策——自我怀疑、重复枚举、
迂回试错。把它们删了,反而让真正的推理浮出来。*

** 这条线还会往哪走?

CRISP 把外部信号都干掉了,但有几个口子还开着:

1. *prompt 这一维*。现在只用了 "be concise"。如果换成 "be rigorous",
   "show work for X but skip Y",同样的 KL 蒸馏框架就变成可控风格塑造的通用工具。
2. *agent 场景*。论文已经在 DeepPlanning 上验证了,
   但 agent 多步规划里"该 verbose 还是该 silent"的判断比数学题复杂得多,
   这一招的边界在哪还不清楚。
3. *和 RL 混搭*。完全去 RL 化是 CRISP 的卖点,但有些任务里
   ground-truth 反馈是免费的 (代码能跑就行)。
   未来很可能是 KL 蒸馏当主干、稀疏 RL 信号当点缀的混合配方。

* 内核 (整条线在发生什么)

表面看是"压缩 chain-of-thought 的技术迭代"。
往深里看,真正在发生的是 *外部信号的剥离*:

| 时期 | 训练信号 |
|------+----------|
| RL 时代 (L1/DLER) | 标准答案 + 长度奖励 (双外部信号) |
| 难度时代 (DiPO/DIET) | 标准答案 + 难度估计器 (依然外部) |
| 早期自蒸馏 (OPSD/SDPO) | 标准答案 + 自蒸馏 (一半外部) |
| CRISP | 一条 prompt 指令 (零外部信号) |

这条线读下来,你会发现 LLM 后训练领域有一个隐隐的方向:
*让模型用自己已经会的东西教自己新东西。*

CRISP 的意义不只在"压缩 CoT" 这个具体任务,
而在于它演示了一种范式: *能力的不同维度——简洁性、严格性、风格——
也许都可以通过 "prompt 一个版本,KL 蒸馏另一个版本" 这样的姿势内化进权重。*
这条路如果走通,RLHF 那种重型的"外部 reward model"框架可能会被大幅瘦身。

* 下一步最可能往哪走

1. *Prompt-conditioned distillation 通用化*。 不只蒸馏简洁性,
   蒸馏任何能被 prompt 描述的目标。
2. *少量 RL 信号兜底*。 KL 蒸馏处理风格 / 长度,
   稀疏 RL 处理正确率,组合拳。
3. *Agent 长 horizon 场景*。 真正的考验在多步、多模态的 agent
   推理上,数学题只是开胃菜。
