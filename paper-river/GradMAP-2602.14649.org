* 这条河的主角是谁

不是 GradMAP 这篇论文。

主角是一个十年没解决的问题：*怎么把一个大模型变小，又不把它变蠢*。

这个问题从 2015 年的图像分类网络一直问到 2026 年的大语言模型。每隔几年，有人给出一个答案，结果用着用着发现新坑。坑变深，新人再来填。GradMAP 是这条河最近的一段。

下面这条线，从最老的源头一路往下走，看每个人到底看到了前人的什么死穴。

主干这条线是： *权重 -> 神经元 -> 整层* 的"剪刀越剪越大"，加上 *测重要性* 和 *剪完补救* 两个永恒难题。

* 一、源头：剪掉小权重就好了吗（Han 2015）

2015 年，深度学习还在搞图像分类。模型已经开始嫌大。Han 等人发现一个非常朴素的事实：神经网络里很多权重几乎是零，留着也没用。

*他们的办法*：把所有权重按绝对值排序，最小的那 90% 直接置零。模型还能跑，精度掉一点点。

听起来很美。但用了几年大家发现两个坑：

1. *硬件不买账*：把权重随机置零得到的是稀疏矩阵。GPU 算稀疏矩阵不快，除非有专门的稀疏内核。算了半天，模型小了，但实际推理速度没快多少。
2. *选哪些权重不对*：绝对值小不代表不重要。一个权重可能很小，但它接在一个非常活跃的输入上，乘起来贡献巨大。光看权重本身，看走眼。

第二个坑在 LLM 时代被 Wanda 修正——它把权重大小乘上输入激活的大小，作为新的重要性分数。但硬件那个坑没解。

于是有人想：与其剪散乱的权重，不如剪 *整块* 结构——一整个神经元、一整个注意力头。剪完之后矩阵就是真小一圈，GPU 直接受益。

这是结构化剪枝。代表作就是 LLM-Pruner。

* 二、第二代：剪整组神经元（LLM-Pruner，2023）

LLM-Pruner 给 LLaMA 这种几十亿参数的模型做了第一次像样的结构化剪枝。它把彼此依赖的一组权重（比如某个 FFN 神经元的输入权重和输出权重）当成一个整体，要剪一起剪。

*它看到了 Han 那条线的问题*：单独剪权重得到的是稀疏矩阵，不实用。把整组神经元一起剪，剪完矩阵真小了一圈，推理真快了。

*它自己留下了什么新坑*：

1. *精度掉得厉害*：神经元级别的剪枝粒度其实还是细。但你剪了之后，模型内部各层之间的协调被打乱了，必须再 fine-tune 一段才能拉回来。fine-tune 一个 7B 模型不便宜。
2. *压缩比有天花板*：你能剪掉 20% 的神经元，但想剪 40% 就基本崩了。
3. *结构变化太碎*：剪完之后每一层的形状都变了，部署起来麻烦。

这时候有人提出一个更激进的想法：*为什么不直接剪掉整个 Transformer 层？*

一层就是一整块。剪掉它，前后层直接相连，模型架构完全不变，只是变浅了。代码几乎不用改，推理直接快。

这就是层剪枝（layer pruning）的开端。我们这条河进入第三段。

* 三、第三代：直接砍整层（ShortGPT / Laco，2024-2025）

ShortGPT 提出的问题极简：*哪些 Transformer 层根本没在干活？*

它的办法叫 Block Influence（BI）：测一层的输入和输出有多像。如果一层的输出几乎等于输入（余弦相似度接近 1），说明这层啥也没改，等于个恒等函数，剪掉无所谓。

*它看到了 LLM-Pruner 的问题*：剪到神经元粒度不够爽，剪一整层才省事，剪完模型也不用 fine-tune 多久，部署也方便。

*它自己留下的坑*：

1. *相似不等于没用*：一层的输入输出长得像，不代表它没贡献。它可能在做一个非常关键的微调——把某个 token 的表示往正确的方向轻轻一推。这点轻推决定了下游 task 的答案。光看相似度，看不到这个"轻推"的价值。这是 GradMAP 后来狠狠批评 ShortGPT 的那点。
2. *剪完没人收拾*：ShortGPT 剪完就走，不做任何补救。精度掉了就掉了。

针对第一个坑，有人提出 SLEB。

* 四、第三代续：用损失变化测重要性（SLEB / BlockPruner，2024）

SLEB 不信余弦相似度。它说："要测一层重不重要，最直接的办法是把它真剪掉，看模型的 loss 涨多少。"

于是它的流程是：依次把每一层临时屏蔽掉，跑一遍前向，记一下 loss。loss 涨得最少的那层就是最该剪的。然后剪掉，剩下的层再来一轮。

*它看到了 ShortGPT 的问题*：表层相似度只是表象，要看一层对最终任务的影响。

*它自己留下了什么新坑*：

1. *慢得要命*：每剪一层要把所有剩余层都试一遍前向。剪 K 层就是 O(L*K) 次前向。一个 7B 模型这么玩，时间从分钟级飙升到小时级。这是 GradMAP 接下来要解决的第一个痛点。
2. *仍然没补救*：跟 ShortGPT 一样，剪完就走。

* 五、第三代再续：剪完得修（LLM-Streamline / UIDL，2024）

到 2024 年下半年，大家意识到一个共同问题： *无论怎么挑层，剪完精度都会掉。光挑得准没用，得有办法把掉下来的精度拉回来。*

LLM-Streamline 的办法：在剪掉的那段位置插入一个轻量级网络（一个小 MLP 或一个小层），用蒸馏的方式让它模仿被剪掉的部分。

UIDL（Gromov 等）的办法更直接：剪完之后用 QLoRA 做一轮快速 fine-tune。

*它们看到了 ShortGPT / SLEB 的问题*：只剪不补，精度损失太大，特别是剪到 40% 以上的时候模型几乎崩盘。

*它们留下的新坑*：

1. *修补本身又慢又贵*：训练一个小网络或者跑 QLoRA，都需要大量的校准数据和 GPU 时间。整个流程的成本回到了 LLM-Pruner 时代。
2. *校准数据敏感*：训练补救网络对校准数据分布很敏感，泛化性存疑。

到这里，问题已经非常清楚了：

- *测重要性*：要么不准（余弦相似度），要么慢（loss 屏蔽）
- *剪后补救*：要么没有，要么贵

谁能同时把这两件事做快做好，谁就赢了。

GradMAP 来了。

* 六、当下：GradMAP 的两手棋（2026）

GradMAP 看到这条河里所有人的坑，提出两件事，各自针对一个痛点。

** 第一手：用梯度大小测重要性

不用余弦相似度，不用 loss 屏蔽。它用 *梯度*。

直觉是：对一个 batch 的样本做一次前向+反向，每层会得到一组梯度。如果一层的梯度普遍很小，说明 loss 对这层的参数不敏感，剪掉它对 loss 影响小。

具体做法：把每层所有参数的梯度做 L2 范数加起来，得到这层的"重要性分数"。分数低的剪掉。

*它和 SLEB 的差异*：SLEB 是"把这层剪掉真跑一遍前向"，O(L) 次前向才能挑一层。GradMAP 是"做一次反向，所有层的梯度一次拿到"，O(1) 次反向选一层。剪 K 层就是 K 次反向，不是 K*L 次前向。

*它和 ShortGPT 的差异*：ShortGPT 只看一层的输入输出像不像，看的是 *表征空间的相似度*。GradMAP 看的是 *这层对 loss 的偏导数*，直接捕捉这层对任务的贡献。所以那种"输入输出很像但对任务关键"的层，ShortGPT 会冤杀，GradMAP 不会。

这一手解决了"快"和"准"的矛盾。

** 第二手：投影补偿矩阵

剪完层之后，剩下层的输出分布会发生漂移——某一层突然要接到一个原本不该接的层，统计性质就变了。

LLM-Streamline 的办法是 *在剪掉的位置插一个新网络*。GradMAP 的办法是 *在剩下的某一层里改一个权重矩阵*。

具体怎么做：

1. 找出剪枝后输出"漂移"最大的那一层（一般只有一两层漂得厉害，其他层影响很小）。
2. 在这层的 down-projection 权重矩阵前面，乘上一个学出来的小矩阵 W'。
3. 训练 W' 让这层剪枝后的输出尽量贴近剪枝前。最后把 W' 吸收进原权重，部署时不增加任何参数。

*它和 LLM-Streamline 的差异*：

- LLM-Streamline 训一整个小网络去模仿被剪掉的层。GradMAP 训一个矩阵去对齐分布的均值漂移。后者参数量少几个数量级。
- LLM-Streamline 改变了模型结构（多了一段）。GradMAP 不改变结构（吸收进原权重）。
- LLM-Streamline 要训到收敛，时间长。GradMAP 训一个矩阵，几分钟。

*它和 UIDL 的差异*：UIDL 是全模型 QLoRA fine-tune，影响所有层。GradMAP 只动一层、只动一个矩阵。

这一手解决了"修"必须"贵"的旧账。

** 合起来的效果

论文报的数字是 4 倍剪枝速度，同时精度比 SLEB / ShortGPT / MKA 全面好。压缩到 40% 的极端比例下，别人崩了，它还稳。

* 这条河往下流向哪里

把整条线拉直看，发生了什么？

表面上是 *剪枝粒度* 在变大：权重 -> 神经元 -> 整层。技术越来越粗。

但更深一层在变化的是 *研究者对"重要性"的理解*：

- 2015：重要性 = 参数本身的大小（看 *存在感*）
- 2023：重要性 = 参数与上下游的依赖（看 *结构位置*）
- 2024 早：重要性 = 这层输出和输入的相似度（看 *表征变化*）
- 2024 中：重要性 = 屏蔽它对 loss 的影响（看 *任务影响*）
- 2026：重要性 = loss 对这层参数的梯度（看 *任务敏感度*）

每一代都在把"重要性"这个概念往 *离任务更近的地方* 拉。从静态的参数特征，到动态的任务响应。

第二条暗线是 *补救方式* 在变轻：

- 早期：不补救，硬扛
- 中期：训练一整段网络补救（重）
- 现在：训练一个矩阵，吸收进原权重（轻）

补救从"加东西"变成"调东西"。从结构改动变成参数微调。这背后的认知是：剪枝引起的损伤，大部分是 *分布漂移*，而分布漂移可以用一次线性变换矫正。不需要重新学功能，只需要对齐。

下一步可能往哪走？两个猜测：

1. *梯度还能更精细*：GradMAP 用的是参数梯度的 L2 范数。但梯度本身有方向和结构。例如可以看不同任务上的梯度方向是否一致，来判断一层是"任务通用"还是"特定任务"。剪掉任务特定层未必影响通用能力。
2. *补偿矩阵会变成通用范式*：投影补偿这种"一矩阵搞定漂移"的思路，可能从层剪枝外溢到量化、蒸馏、模型合并等所有需要"对齐分布"的场景。它本质上是个轻量级的分布对齐器，应用面比层剪枝大得多。

整条河的下一段，大概率不是再发明一种剪枝方法，而是 *把这套"用梯度选 + 用投影补"的范式搬到其他压缩任务上*。
