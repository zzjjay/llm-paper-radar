* 问题之河

一句话：让模型一步步"想"一道难题时，它每吐一个字都要把刚才的思考存进 KV 缓存这个小本子里，思维链越长本子越厚，内存就爆了——能不能边想边把本子上不重要的字直接*擦掉*（eviction，驱逐），还不让它想糊涂？

这个问题听起来跟"压缩本子"是一回事，其实差了一个量级。压缩（量化）是把字写小，字还在；驱逐是干脆把字划掉，划掉就再也找不回来了。划掉省内存最狠（本子大小永远固定），可一旦划错一个关键的字，模型就可能彻底崩——不是答错一道题，而是陷进"我再想想……不对再想想……"的死循环，吐出一堆不成句的乱码。

整条研究线，就是一群人围着同一个问题死磕：*到底该擦掉哪个字？* 一开始大家都盯着"哪个字被注意得多"（注意力分数），擦掉没人看的。这个直觉统治了三年。直到有人发现：有一小撮字，注意力分数平平，可它的"数值幅度"大得吓人——擦掉它，天塌。判断标准从"谁被看得多"，悄悄换成了"谁的分量重"。

* 演化叙事

** 2023.06 | H2O: 注意力分数里藏着"重击者"，留它们就够了

*** 它看到了什么问题

最朴素的省内存办法是"滑动窗口"——只留最近 N 个 token，老的全扔。问题是模型经常要回头看很早以前的某个关键信息，你把它扔了，它就瞎了。那留哪些老 token 呢？当时没人有靠谱的判据。

*** 它的解法

H2O 注意到一件事：如果你把每个 token 历史上收到的注意力分数累加起来，这个分布是*幂律*的——极少数 token 吸走了绝大部分注意力，像几个"重击者"（heavy hitter）。于是规则很简单：留住这些累计注意力最高的，再留一截最近的，剩下的一律驱逐。它甚至证明了这等价于一个"动态子模优化"问题，有理论保证。

*** 为什么这招管用（以及哪里埋了新坑）

管用，是因为"被很多后续 token 反复注意的 token"，确实往往携带了关键信息。但这里埋了一颗雷：H2O 全程只盯着*注意力分数*，而注意力分数是 query 和 key 算出来的——它从头到尾*没看过 value 长什么样*。可注意力的输出，是 value 的加权和。一个 token 即使没人特别注意它，只要它的 value 数值巨大，它对最终输出的贡献也可能是决定性的。H2O 对这种 token 视而不见。这颗雷，三年后才被 VaSE 引爆。

** 2024.04 | SnapKV: 别把连成片的上下文选得稀碎

*** 它看到了什么问题

H2O 式的"按单点分数选 top-k"有个副作用：被选中的 token 东一个西一个，把原本语义连贯的一段上下文切得支离破碎。在"大海捞针"这类检索任务上，模型拿到一堆碎片，经常只能拼出半个正确答案。

*** 它的解法

SnapKV 做两件事。第一，用 prompt 末尾一小段"观察窗口"里的 query 来投票，看它们最关心 prompt 的哪些位置——这比用全部历史更聚焦。第二，也是关键：选中某个关键 token 时，沿着序列方向做一次 *max-pooling*，把它的左右邻居一起捞进来。这样留下的不是孤立的点，而是一小簇连贯的上下文。

*** 为什么这招管用（以及哪里埋了新坑）

pooling 保住了上下文的完整性，检索精度立刻回来了。但 SnapKV 的世界观还是"一次性预填充"——把一整篇长文档塞进去、压一次、然后生成。它不太适配"边解码边压缩"的推理场景。更重要的是，它的打分依然只用 key、依然是*确定性*的 top-k：分数低于阈值线的 token，概率 100% 被处决，没有任何翻盘机会。这个"确定性"后来也成了 VaSE 要改的点。

** 2025.05 | R-KV: 推理链里全是"重复验算"，光看重要性会被骗

*** 它看到了什么问题

推理模型（会写长篇思维链的那种）暴露了前人方法的一个新软肋。它的思维链里塞满了"让我再验算一遍……""等等我重新想想……"这种自我重复的段落。这些重复段落有个阴险特性：*它们会给自己刷出很高的注意力分数*。于是 H2O、SnapKV 一看分数高，全留下了——结果缓存里塞满了内容几乎一样的冗余 token，真正多样的信息反被挤掉。

*** 它的解法

R-KV 在"重要性"之外加了第二个维度：*冗余惩罚*。它实时比较各个 token 的 key 向量，如果两个 token 长得太像（高度冗余），就不要把它们都留下。最终的去留是"重要 + 不冗余"的联合打分。这让它在推理任务上成了当时最强的驱逐法，10%~34% 的缓存就能追平甚至偶尔超过满缓存。

*** 为什么这招管用（以及哪里埋了新坑）

"逼缓存多样化"这个思路是对的，VaSE 后来也继承了"多样性"这个目标。但 R-KV 的多样性是拿 *key* 算的——它衡量的是"token 之间像不像"，量的还是 key 空间。从 H2O 到 SnapKV 到 R-KV，三代人换着花样打磨"怎么用注意力和 key 选 token"，却始终没有一个人，把 *value 的数值幅度* 纳入去留判据。这棵树上的果子摘完了。要破局，得换一棵树。

** 2023.09 / 2024.02 | 暗线: 有些"分量"动不得

*** 它看到了什么问题（StreamingLLM）

几乎同时，另一拨人在啃一个看似无关的问题：模型为什么没法处理超长流式文本？StreamingLLM 发现一个怪现象——开头那几个 token 是"注意力黑洞"（attention sink），它们语义上啥也不是，可你一旦把它们扔了，整个注意力分布瞬间崩盘。结论：*有些 token 的价值不体现在语义上，而体现在它对系统稳定性的支撑上*。

*** 它的解法与升级（Massive Activations）

Sun et al. 把这件事挖到了底。他们发现模型里存在"巨型激活"（massive activation）：极少数（常常不到 10 个）激活值，幅度比周围大到 10 万倍。这些值几乎不随输入变化，本质是模型学出来的*隐式偏置项*。做干预实验：把它们置零，模型灾难性崩溃；把它们固定成均值，反而毫发无伤。

*** 为什么这条暗线是 VaSE 的钥匙

这两篇合起来，给出了一个前面三代驱逐法完全没消化的教训：*判断一个数值能不能扔，"幅度"是一个独立于"注意力"的维度*。一个东西可以注意力很低，却因为幅度巨大而绝对动不得。主线那帮人一直在"注意力/key"里打转，钥匙其实早就躺在隔壁房间了。VaSE 做的，就是把这把钥匙捡起来，插进 KV 驱逐的锁里。

** 2026.06 | VaSE: 给"大分量 value"留专座，再让驱逐"抽签"

*** 它看到了什么问题

VaSE 先做了一个让人后背发凉的实验：故意去驱逐那些 value 幅度（它用 Range(v)，即向量最大项减最小项来衡量）最大的 KV 对。结果 GSM8K 准确率从正常水平崩到 *14.3%*——比闭着眼睛*随机*驱逐还要低 38.9%。模型陷入"无尽的自我反思循环"，开始吐不成句的乱码。这正是 massive activations 那条暗线预言的灾难，只不过这次发生在 value 缓存上：一小撮 value 幅度极大，因为输出是 value 的加权和，它们的影响力不成比例地大，而过去三代驱逐法对它们*毫无防备*。

*** 它的解法

两步，对应两个被忽略的维度。

第一步，*留专座*。从总预算 K 里划出一块固定名额 Nv（实践中 Nv = K/4），专门给 Range(v) 最大的那批 KV 对。它们被无条件保留，根本不进入"按注意力竞争去留"的环节。注意这和前人 Guo et al. 的做法不同——前人是把幅度*乘进*注意力分数里去微调排名，VaSE 是直接*隔离出一块保护区*，干净利落。

第二步，*抽签代替硬选*。剩下的名额，不再像 SnapKV 那样按注意力分数 top-k 硬切，而是把分数当权重去*抽样*。这么改的理由很微妙：top-k 下，分数低于线的 token 被驱逐的概率是 0——它永无翻身之日；而抽签下，每个 token 的保留概率都严格大于 0，于是"在全部 T 步里至少活下来一次"成为可能，缓存的多样性和覆盖面都上去了。（它的变体 VaSE-DKV 则是每步重新抽一个高斯投影矩阵 G，避免某个 token 被一次性的投影永久打入冷宫。）

*** 为什么这招管用（以及哪里埋了新坑）

拆开看收益：光是加上 value 打分，准确率最多回升 16.2%；再加上抽签的随机性，额外再涨 4.7%。合起来，在约 4 倍压缩、六个推理任务上，VaSE-AttnV 比前 SOTA 驱逐法 R-KV 高 4.4%（Qwen3-4B，59.09% vs 54.69%）和 4.9%（Qwen3-14B，65.81% vs 60.90%）；对 CurDKV 更是高出 7.7%/9.2%。

最漂亮的是和"选择式"方法 SeerAttention-R 的对比：后者准确率相当（甚至略低于 VaSE），但它*留着全部 KV 缓存*、只是每步挑一部分来读，内存随序列长度线性增长；VaSE 用的是*固定大小*的缓存。同样的精度，一个本子越写越厚，一个本子永远那么薄。

新坑也留着：Nv = K/4 是个拍出来的固定比例，不同层、不同任务的"大分量 value"占比未必一样；Range(v) 这个判据虽然和 L2、方差正相关，但它本质是个静态启发式，未必抓得住"哪些大分量在当前这一步真的有用"。

* 前沿延伸

这条线在 2025 末到 2026 还在快速分叉，VaSE 是其中一支：

- *选择式 vs 驱逐式的拉锯还没分胜负*。SeerAttention-R（用训练出来的注意力门控）代表"留全本只读一部分"，精度高但内存线性增长，且 VaSE 指出它的学习式门控"在领域漂移下泛化吃力"（在代码任务上掉点）。VaSE 代表"直接擦、内存恒定"。两条路线各有死穴。
- *把"贡献"算得更准*。CurDKV 用 CUR 分解的杠杆分数来同时考虑 key 和 value，但 VaSE 批评它"不促进多样性"且"用固定的 G 导致 token 被确定性驱逐"。ForesightKV、ThinKV 等则尝试用"长期贡献""思维自适应"等更前瞻的信号来替代当下的注意力快照。

截至目前（2026.06），"value 幅度是一个独立于注意力的、不可忽视的去留维度"这个洞见刚被 VaSE 摆上台面，后续大概率会有人把"留专座"的固定比例做成自适应的、把"抽签"和"长期贡献预测"结合起来。

* 洞见

这条线表面是"KV 驱逐越做越好"，底下真正发生的，是*判据的重心在迁移*。

H2O 到 R-KV 这三代，全部在回答同一个问句的不同变体：*"哪个 token 被注意得多/被注意得有没有重复"*。它们把"注意力分数"这个信号榨干了——加历史累计、加 pooling、加冗余惩罚。但它们共享一个未被审视的前提：*一个 token 值不值得留，由它在注意力里的地位决定*。

VaSE 的贡献不是又发明了一个更精巧的打分函数，而是*指出了这个前提是片面的*。注意力的输出是 value 的加权和，所以"权重"（注意力）和"被加的东西"（value 的幅度）是两个正交的维度。三代人只优化了前者。当 massive activations 那条暗线证明"幅度本身就能决定生死"之后，把这个维度补进来几乎是必然——VaSE 只是第一个动手的人。

还有一个更隐蔽的认知转变：从*确定性*到*随机性*。从 H2O 到 R-KV，"选最好的 K 个"被默认是天经地义的。VaSE 指出，在一个要走 T 步、每步都在删东西的过程里，确定性 top-k 等于给落选者判了死刑——而保留一点随机性，反而让整体覆盖更好。这是一个从"贪心最优"到"探索保多样"的小型范式转移，和强化学习里探索-利用的张力同源。

下一步往哪走，基本能猜到：*把静态启发式换成自适应/预测式*。Nv=K/4 的固定专座比例、Range(v) 的静态判据，都是"拍出来的"。一旦有人把"哪些大幅度 value 在当前步真正有用"用一个轻量预测器算出来，再把它和抽签的随机探索结合，这条线就会从"基于幅度的启发式"进化到"基于贡献预测的驱逐"。

* 启发

这条线对"怎么找问题"最直接的启发：*当一个领域里所有人都在打磨同一个信号时，去问那个没人审视的前提*。

H2O、SnapKV、R-KV 都是好工作，但它们卷在"注意力打分"这一个维度里越来越深。真正的突破来自一个外行式的追问——"等等，注意力的输出明明是 value 的加权和，你们怎么从来不看 value？" 这个问题不需要更高的数学，只需要拒绝接受"大家都这么做"这个隐含前提。

第二个启发：*跨房间找钥匙*。VaSE 的核心武器（大幅度的东西动不得）不是它自己发明的，是 massive activations / attention sink 那条"可解释性"暗线早就摆好的结论。主线那帮做"效率"的人，和暗线那帮做"理解模型"的人，几年里基本不串门。谁先把隔壁房间的钥匙拿过来插进自己的锁，谁就破局。

落点在"能用"：这意味着你在读一条研究线时，除了顺着它往下读，更该问——
（1）这群人共享的、却从没人证明过的前提是什么？把它写下来，挨个怀疑。
（2）有没有另一个领域，已经把我卡住的这个东西研究透了？我要做的可能不是发明，而是搬运。
