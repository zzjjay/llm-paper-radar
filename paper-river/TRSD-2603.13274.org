* 一句话

让推理模型"想到一半就回答也能答对"——不是教它想得更短，而是教它在任何被打断的位置都能交卷。

* 问题怎么演化的——一条线讲下来

** 第一站: 2022, CoT 让模型"边想边答"

故事从一个简单观察开始: 让模型在回答之前把思考过程写出来——"让我们一步一步想"——它的数学题正确率突然上去了。这就是 Wei 等人的 chain-of-thought (思维链) 提示。

它带来的好处显而易见: 大模型从"猜答案"变成"推理出答案"。

但它也带来一个看似无害的副作用: 模型要先写一大段"思考"，才肯交答案。一道小学算术题，原本几十个 token 就能答完，现在动辄写几百几千个 token。

那时候没人觉得这是问题——能答对就行，多写点字而已。

** 第二站: 2025.01, DeepSeek-R1 把"长思考"训进了模型骨子里

三年过去，事情变本加厉。DeepSeek-R1 用强化学习训练模型，让它*自发*地写出极长的推理链——一道题动辄想几千上万个 token，反复自我怀疑、自我修正、"等等让我再想想"。

效果: 数学奥赛级别的难题被攻克了。

代价: 推理时间和 token 成本爆炸。更糟糕的是，R1 还启动了一条"蒸馏"流水线——用 R1 当老师，教小模型也学着这样想。结果学生不光学到了推理能力，也*继承了老师的啰嗦*——简单题也要长篇大论。

这时候大家才意识到: 推理模型有"过度思考" (overthinking) 的毛病。

** 第三站: 2025.01, s1 的暴力实验——直接把思考切断会怎样

s1 这篇论文做了个干脆的事: 既然模型想太多，那我*推理时*直接拦下来——叫这个 "budget forcing" (预算强制)。

具体怎么做? 两招:
- *想短一点*: 到达指定 token 数就强制塞一个"思考结束"标记，逼模型立刻交答案
- *想长一点*: 模型想停的时候，偷偷把结束符换成 "Wait"，逼它再想想

第二招居然有用——模型被强迫"等一下"之后，经常能修正错误。但*第一招的代价很重*: 在该想的题目上提前切断，准确率直接断崖式下跌。

这里出现了第一个清晰的问题: *训练时想得很长的模型，推理时根本不能被打断*。它没有"半路交卷"的能力。

** 第四站: 2025.02, TokenSkip 想得更聪明——挑出不重要的词跳过

s1 那种"在某个位置一刀切"太粗暴。TokenSkip 换了个思路: 既然 CoT 里很多词是废话，那我*学会哪些词可以跳过*不就行了?

它的做法:
1. 用一个"重要度评分器" (借鉴 LLMLingua-2) 给思考链里每个 token 打分
2. 按压缩比例 γ 砍掉低分的 token，得到一个"精简版"思考链
3. 用这个精简版去 fine-tune 模型，让它学会自己生成精简链

结果不错: 砍掉 40% 的 token，准确率只掉 0.4%。

但这里又埋了新问题: *你怎么知道哪些 token 真的"不重要"*? 重要度评分本质上是一个外挂的判断器，它的判断对不对，没人能验证。同期的 LightThinker、CoT-Valve、O1-Pruner 也都在这条"挑词/控长度"的路上走——每个方法都在回答同一个问题: 哪些字该留，哪些字该砍?

** 第五站: 2026.02, TRSD 翻桌——不挑了，挑本身就是错的

TRSD 这篇论文的真正洞见，是看穿了上一站那个隐含假设的错误。

它问: 为什么我们要假设"某些 token 比别的更重要"? 如果模型真的学会了推理，那么*它在任何一个被打断的位置，都应该能基于已经想到的内容给出最佳答案*。

换句话说: 问题不在思考链里，问题在模型身上——它没被训练过"半路交卷"。

TRSD 的做法朴素到让人吃惊:
1. 拿一个冻结的老师模型，让它*完整*想完一道题，记下它最终的答案分布
2. 训练学生模型 (同架构、同初始化), 让它在*只看到思考链前缀*的情况下，匹配老师那个"完整思考后"的答案分布
3. 关键: 截断的位置是*随机*的——不挑、不选、不打分

这个训练目标在说什么? *无论你被打断在哪里，都要给出和"想完了"一样的答案*。

效果:
- 在任何 token 预算下，截断都不再让准确率崩盘
- *没有任何长度惩罚*，但训出来的模型自发地生成更短的思考链——因为它学会了"早点想清楚"

** 这条线背后真正在发生什么

从 CoT 到 TRSD，表面是"思考链怎么变短"的技术迭代，深层是一个认知转变:

| 阶段        | 隐含假设                          |
|-------------+-----------------------------------|
| CoT         | 多写点字总没坏处                  |
| DeepSeek-R1 | 训练时想得越长，能力越强          |
| s1          | 推理时可以人为干预长度            |
| TokenSkip   | 思考链里有信息密度差异，可以压缩  |
| TRSD        | 信息密度差异是个伪命题——          |
|             | 真正的问题是模型不会"中途收尾"    |

每一步都在剥掉前一步的一个隐含假设。TRSD 剥掉的是"token 有重要度"这个看似无害的假设——它说，与其费心去判断哪些 token 重要，不如直接训练模型在任何位置都能优雅地结束。

* 接下来可能去哪

如果 TRSD 这条思路成立，那"哪些 token 重要"这类研究会逐渐失去地基。值得期待的下一步:

- *把"任意截断鲁棒性"做成新的预训练目标*, 而不是后训练补丁
- *和推测解码 (speculative decoding) 结合*: 推测解码本来就在"猜下一段"，如果模型本身就鲁棒于中断，推测的容错空间会大很多
- *自适应预算*: 模型自己判断"我已经想清楚了, 现在就交卷"——TRSD 已经隐隐做到, 但还没显式建模
