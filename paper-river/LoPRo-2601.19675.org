* 一句话定位

把一个几千亿参数的大模型压到 2 比特还不崩，靠的不是单一招数，而是十年里
一条问题演化线的合流——*怎么压一个不均匀的矩阵*。LoPRo 是这条线上最近的
一个结。

* 演化叙事：从 1993 到 2026

** 一切从一个问题开始：删掉一个权重，剩下的怎么办？

1993 年，Hassibi 和 Stork 在做神经网络剪枝。他们的问题很基本：一个训练
好的网络，你想去掉一些权重让它变小，删哪个？

朴素答案：删绝对值最小的那个。

他们的答案不一样。他们说：删一个权重之后，整个网络的输出会变。但你可以
*调整剩下的权重去补偿这个变化*。所以真正该问的是——"删哪一个、然后怎么
调整其他人，能让整体损失最小？"

这是个二阶问题。他们用 Hessian 矩阵（损失对权重的二阶导）给出了闭式解。
这个方法叫 OBS（Optimal Brain Surgeon）。

要记住的就一句话：*权重之间是相关的，动一个就要补偿其他人，Hessian 告诉
你怎么补*。

** GPTQ：把这个老办法搬上大模型

时间跳到 2022 年。Frantar 团队发现：量化（把 fp16 权重压成 4 bit）和剪枝
（把权重设成 0）其实是同一类问题——都是"扰动一个权重，怎么调整其他人"。

他们把 OBS 改造成 GPTQ：一列一列地量化权重矩阵，每量完一列，就用 Hessian
告诉你剩下未量化的列应该怎么微调来补偿误差。

这一招让 175B 的 OPT 模型量到 4 bit 还能用。整个量化社区从此有了一个能跑的
起点。

但是——往下压到 3 bit、2 bit 的时候，GPTQ 顶不住了。

为什么顶不住？因为权重里有 *异常值*。一个矩阵里大多数权重在 [-0.1, 0.1]，
但有少数几个在 [-3, 3]。量化网格如果照顾异常值，就太粗，挤掉了大多数权重的
精度；如果照顾大多数，异常值就被截掉了。补偿再聪明，也补不回截断的信息。

** QuIP 的洞见：那就把异常值打散

2023 年，Cornell 的 Chee 等人提出 QuIP。他们的想法很怪但很漂亮：

如果异常值集中在某几列、某几个位置，导致分布不均匀——那 *把矩阵旋转一下*，
让异常值平均分摊到所有位置上呢？

具体操作：左右各乘一个正交矩阵 U 和 V^T，得到 U·W·V^T。模型的输入输出
同步反向旋转，整体功能完全等价。但量化的对象 W 变了，分布更均匀了。

这个性质他们叫 *incoherence*——矩阵在某种意义上"不集中"。

代价：每次推理都要在线做这两次旋转，挺慢。

** QuaRot：让旋转免费

2024 年，Ashkboos 等人发现：如果你用 *Walsh-Hadamard 矩阵* 作为旋转矩阵
（一种全是 ±1/√n 的特殊正交矩阵），它有两个好处：

1. 它可以被融进相邻的 RMSNorm 或者线性层里，运行时不需要额外计算
2. 它有理论保证能压制异常值

这一下旋转的代价从"每次推理多算一次"降到接近零。QuaRot 一时成为最强的免
fine-tune 方案之一。

但 QuaRot 也留了个口子：*它把所有列等同对待*。旋转之后你已经分不清原来哪
些列是"重要的"了，因为信息被打散了。这在标准量化场景没问题，但下面这个分支
里就出问题。

** SVD-Quant / LQER：与其压全部，不如分两层

差不多同一时间，2024 年另一条线在走完全不同的路。

Li 等人（SVD-Quant）和 Zhang 等人（LQER）说：*量化之所以损失大，是因为
权重矩阵里有它"压不进低 bit"的成分*。那不如把矩阵分成两层：

#+begin_src text
        W   =   W_r   +   R
        |       |         |
       全部    低秩部分    残差
       fp16   保留高精度  量化到低 bit
#+end_src

W_r 通过 SVD 取前几个奇异值得到——它捕捉了 W 里最"主成分"的部分。剩下的
残差 R 相对均匀，更适合低 bit 量化。

这个分解给 2-bit 量化带来了显著的精度回收。但代价也清楚：W_r 是高精度，要
存、要算，秩越高负担越大。论文里提到 *256 ranks 让推理吞吐量降低 45%*。

更微妙的问题是：*残差 R 还是不均匀的*。SVD 提取了最大的几个主成分，但留下
来的 R 里依然有异常值的二阶残留。

那能不能对 R 也做旋转？这就是 LoPRo 要回答的问题。

** LoPRo 看到的死结

把前面所有招数摊在桌上，LoPRo 团队看到了一个尴尬：

- 旋转（QuaRot）能让矩阵均匀，但 *会破坏列结构*
- 重要列保护（AWQ 风格）需要 *识别哪些列重要*
- 低秩分解（SVD-Quant）需要量化残差，残差还是有异常值

直接把 QuaRot 套到残差 R 上？*会让你失去对重要列的保护能力*。论文给了数
据：LLaMA2-7B 上对残差做全 Hadamard 旋转，困惑度从 8.4 涨到 9.49——比不
旋转还差。

这是个死结：旋转让分布均匀（好），但旋转后所有列长得一样，你想保护的重要
列就保不住了（坏）。

** LoPRo 的解法：旋转一部分就够了

LoPRo 的核心是一个不复杂但很巧的设计：

*第一步：按重要性给列排序*。用 Hessian 对角线（这一列对损失的敏感度）除以
该列的平均幅值，得到每列的"量化难度"。按这个分数排序，把最难量化（最重要）
的列排到最前面。

*第二步：分块旋转，前面那块不动*。把排好序的矩阵切成几块。第一块——最重要
的那几列——*用单位矩阵*，也就是 *不旋转*。剩下的块各自用 Walsh-Hadamard
旋转。

#+begin_src text
        Q =  [ I       0    ...  0    ]
             [ 0    H_wal   ...  0    ]
             [ :       :    \    :    ]
             [ 0       0    ... H_wal ]

       重要列：保持原样
       次要列：分块打散
#+end_src

*第三步：和低秩拼起来*。这套部分旋转可以无缝套在 SVD-Quant 那一层之上。
W_r 是高精度低秩，R 经过排序+分块旋转后量化。两套机制不再打架。

*额外：R1SVD（rank-1 sketch）*。常规 SVD 在大矩阵上很慢，他们用一个 rank-1
草图近似把开销砍半。让 8x7B 的 Mixtral 也能在 2.5 小时内量化完成。

** 结果

- LLaMA2/3 在 2-bit、3-bit 上超过所有 fine-tuning-free baseline
- 在某些设置上甚至超过需要 fine-tune 的方法
- 推理时 4× 加速、内存额外开销 < 2%
- MoE 模型 Mixtral 困惑度反而下降 0.4，准确率上升 8%

* 真正在发生的认知转变

如果只看技术演化，是 OBS → GPTQ → QuIP → QuaRot → SVD-Quant → LoPRo。
但底下有一条更深的线：

*第一阶段（OBS、GPTQ）：把权重当独立的数字*。压一个、补偿另一个，单元是
单个权重。

*第二阶段（QuIP、QuaRot）：把权重矩阵当一个整体的分布*。整体不均匀就整体
变换。单元从权重升到矩阵。

*第三阶段（SVD-Quant、LoPRo）：把权重矩阵当多个层次的叠加*。一部分天然
适合高精度，一部分天然适合低精度。不同部分用不同手段。单元从矩阵升到
"矩阵的子空间"。

每一步都是 *把对象切得更细，针对性更强*。LoPRo 的"分块"和"按列重要性排序"
继续了这条线——它甚至在矩阵内部都分出"该保护的列"和"该打散的列"。

* 下一步可能往哪走

按这条认知线，*再细一层* 大概是下一步：

- 不只是分"重要列/次要列"两类，而是给每列一个连续的"该多旋转多少"分数
- 不只是 SVD 取前 r 阶，而是按通道、按行做自适应的低秩
- 把这个分解过程做成可学习的，让模型自己决定哪里该旋哪里该留

学习化版本的部分已经在 SpinQuant（Liu et al. 2025）出现——它把旋转矩阵也
变成可学习的。可以预期 LoPRo 类方法的"分块"和"重要性切分"也会被学习化。

另一个方向是 *MoE 友好*：LoPRo 在 Mixtral 上意外地不掉点反而涨点，暗示
MoE 的稀疏激活和"重要列保护"天然契合。这条线值得追。
