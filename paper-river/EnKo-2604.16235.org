* 一句话定位

这条线在追问：当我们想让一个全球通用的大模型只服务一种语言，模型里那一大堆"别的语言"是负担还是底子？

目标论文给出 2026 年的答卷：把多余语言的 token 砍掉，韩语任务不仅没退化，机器翻译还更稳了——尤其是模型不再"说着说着串成英文/中文"。

* 正向叙事：问题怎么一层层长出来的

** 起点：一个模型装下所有语言（2019, mBERT）

2019 年 Google 想做一件事：与其每种语言训一个模型，不如让一个 BERT 把 104 种语言都学了。把所有语言的文本拌一起喂进去，共享同一个 vocab、同一份参数。这就是 mBERT。

效果出人意料地好——它居然能跨语言迁移：用英语标注训练，泰米尔语也能跑得不差。这给整个领域定了调：*多语言是一种"免费的午餐"*，参数共享会让小语种沾大语种的光。

但有个隐患没人当回事：这 104 种语言共用一个有限的 vocab（mBERT 是 11 万 token）。塞下韩语的 Hangul、中文的 Hanzi、阿拉伯文、印地文……每种语言能分到的"格子"其实很少。当时大家不在乎，因为"反正能跑"。

** 转折：规模救不了所有事（2020, XLM-R）

一年后 Facebook 推 XLM-R，把训练数据从 Wikipedia 扩到 2.5TB CommonCrawl，模型从 1.7 亿涨到 5.5 亿，vocab 扩到 25 万。性能确实又涨了。

但 XLM-R 论文里抛出了一个让人不舒服的发现：*固定模型容量下，往里塞更多种语言，单语言性能会掉*。他们起名叫 *curse of multilinguality*（多语言诅咒）——你以为是免费的午餐，其实是大家分一锅饭，多一张嘴每人就少一口。

他们给的解法是简单粗暴的：把模型做得更大。容量够大就能装下所有语言，咒就解了。

这答案在 2020 年说得通。但它把一个更尖锐的问题留给了部署的人——*我只服务韩语用户，为什么要付那 90% 用不到的参数的电费？*

** 拐弯：与其养肥模型, 不如砍掉用不到的部分（2020, Load What You Need）

同年 Abdaoui 等人提了个完全反方向的招：既然我的服务只用法语和阿拉伯语，那 mBERT 里那 102 种语言的 token embedding 留着干嘛？

他们的做法朴素到不像研究：拿一份目标语料，统计哪些 token 出现过，没出现的 token 直接从 vocab 里删掉，对应的 embedding 行也删掉。删完模型小了一半，下游任务性能几乎没掉。

这是 *词表剪枝（vocabulary pruning）* 的雏形。但这个工作有两个尾巴：
- 实验只在 mBERT 这种小模型上做，2024 年的大语言模型还没出现
- 评估指标基本是分类准确率，对"生成质量"这种更微妙的东西没说话

后人接下来要处理的，就是这两条尾巴。

** 收紧：推理时按需缩, 不是训练时一刀切（2024, Lee & Kim）

四年过去，大语言模型时代来了。生成式模型比分类模型更脆——输出端是一个 softmax over vocab 的概率分布，vocab 越大，softmax 越贵，候选越分散。

Lee & Kim 2024 注意到一件事：哪怕你最终只生成韩语，模型每一步预测下一个 token 时，仍然要在全部 15 万词上算 softmax。中间那十几万个"绝不会被选中的"中文、日文、希伯来文 token，每一步都在浪费 FLOPS。

他们提了 *动态词表剪枝*：推理时根据上下文动态裁剪 softmax 候选集，只在可能用到的那部分上算概率。

但这也只是省了算力，*没回答另一个更要命的问题——为什么模型会"想去"那些 token？*

** 旁路：模型会自己说着说着串语（2024, Marchisio et al.）

几乎同时，另一条线在拷打多语模型。Marchisio 等人正式提出了 *language confusion* 这个指标：让模型用韩语回答问题，统计它的输出里出现了多少非韩语词。

结果很难看。哪怕是 Llama-3、Qwen 这种 SOTA，用韩语 prompt 也常常会突然蹦出几个英文词，甚至整段切到英文。这不是 bug，是模型在多语共享 vocab + 多语共享 embedding 空间下的天然行为——韩语 token 和英语 token 在同一个语义空间里挤着，距离一近就会"搭错车"。

他们提出了 Word-level Pass Rate (WPR) 这个评估量：输出中目标语言词占的比例。这个指标后来被本文直接拿去用了。

但 Marchisio 这条线只做了诊断，*没给治疗方案*。他们把锅留在了桌子上：你们想想办法。

** 落地：把两条线接上, 在韩语上做体检（2026, 本文 EnKo）

2026 年 4 月，Hoyeol Kim 和 Hyeonwoo Kim 把上面三条线收到一起：
- 从 Abdaoui 那条线，他们继承"按语言剪 vocab"的思路
- 从 Lee & Kim 那条线，他们继承"vocab 剪掉之后 softmax 也跟着瘦"
- 从 Marchisio 那条线，他们借来 WPR 指标去验证一件事：*剪掉非目标语 token，能不能从根上治串语？*

他们设计了三种词表配置：
- Original：原模型不动
- EnKo：只保留英语 + 韩语 token，其他全砍
- EnKoZh：保留英语 + 韩语 + 中文 token（因为 Qwen 这类模型在中文上预训练很重，可能依赖中文的潜在表示）

然后在 14 个开源模型上（Qwen3 0.6B~14B、Gemma-3 270M~12B、Llama-3、Tri、Aya-23 等）跑四类基准：通识（KMMLU）、文化（HAERAE/CLIcK）、指令遵循（LogicKor/KoMTBench）、机器翻译（WMT24++）。

结果分三块说：

*第一块：通识和文化, 几乎没掉*。比如 Qwen3 系列三种配置间差距 <0.01，Gemma-3-12B 的语言学评测甚至涨了一点。这一刀砍下去（vocab 减 36%），韩语本身的能力没受伤——印证了 Abdaoui 当年的判断：多余 token 真的就是多余的。

*第二块：指令遵循变化看架构*。Qwen3 这种深度中文预训练的模型，砍掉中文 token（EnKo）时性能掉，留着中文（EnKoZh）就好——说明它的推理能力部分*寄生在中文 token 的潜在表示上*。Llama-3.1-8B 反过来，砍得越狠表现越好。这是个新发现：剪枝对推理的影响不是单调的，取决于模型预训练时的语言占比。

*第三块, 也是最有戏剧性的：机器翻译普遍变好*。Llama-3.1-8B 的 COMET 从 0.5879 涨到 0.6342，Aya-expanse-8B 从 0.6957 涨到 0.7496。砍掉无关语言反而让翻译变准——因为输出分布不再被那些"不该出现的语言 token"分散注意力。

而 *WPR 是真正的 smoking gun*：Qwen3-4B 的 WPR 从 0.8882 直接拉到 0.9923，提升 0.1041。Marchisio 抛出的"串语"问题，在 token pruning 这个意外的手术下被治掉了大半。

最后一个略尴尬的发现：vocab 砍了 36%，*推理延迟只快了 0.89%*。embedding 不是真正的瓶颈，attention 才是。这条路是省内存的路，不是省算力的路。

* 提炼洞见

** 这条线背后真正在变的是什么

表面上是"剪枝技术在进化"，本质是 *对"多语共享"这件事的认知翻转*。

2019-2020 那批人相信 *多语言是资产*——共享能让小语种沾光。2024-2026 的发现把这个故事的另一面挖出来了：在生成式模型下，*多语言也是污染源*。多余的语言 token 不只是占内存，它们的 embedding 在语义空间里和目标语 token 混居，会主动"勾"模型走错路——这就是串语的微观机制。

所以 EnKo 这条线真正在做的事是：*把"通用模型 + 单语部署"这种半吊子方案，往"通用预训练 + 单语术后切除"这种新形态推*。模型仍然在所有语言上预训练（拿到知识、对齐、推理能力），但部署前做一次定向"剪除"。这跟传统的 "monolingual from scratch" 和 "multilingual all-the-way" 都不一样，是第三条路。

** 下一步最可能往哪走

三个口子是开着的：

1. *attention 层的语言剪枝*。本文最后一句话泄了底——vocab 剪了 36% 但延迟只省 0.89%，瓶颈在 attention。如果能识别 attention head 里"专门处理某种语言"的子结构并把它剪掉，速度才会真涨。可参考 Voita 等人 2019 在 NMT 上对 head 的研究。

2. *动态语言探测*。EnKo 现在是部署前一次性切除，但用户偶尔会发一句英文夹中文的混合 prompt。下一步会有人做"在线判断输入语言、临时启用对应 token 集"的混合方案——把 Lee & Kim 的动态裁剪从"按候选概率"扩展到"按语言族"。

3. *剪枝即正则化*的理论化。本文翻译变好是个意外。如果有人能证明"剪掉无关语言 token 等价于在输出分布上施加 L1 正则"，那 token pruning 就不只是部署技巧，而是训练阶段的一个正式工具。

* 验收复盘

- *问题是主角*：是。整条线读完，记住的是"多语共享 → 单语部署 → 串语 → 切除"这条问题链
- *因果不断*：是。每一篇都是被前一篇没解决的问题推出来的
- *差异清晰*：是。每篇用一句话讲了它和前篇的差异
- *外行能跟*：基本能。Token / vocab / softmax 等少数术语在场景中带出，没用前不解释
