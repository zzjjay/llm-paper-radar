* 问题之河

一个推理模型在解一道难题，可能会想 32000 个 token。这一长串想法，是一条平的、无分段、无层次的流——模型自己也不知道哪段是关键、哪段是岔路、哪段已经可以扔掉。每一个旧 token 都还得呆在注意力窗口里，吃显存、吃算力，到死都不能下场。

为什么难？因为 LLM 训出来就是一个 token 一个 token 往后写。它没学过"想到一半，把刚才那段压成一句话，然后只看那句话往下推"。这是人脑做得很自然的事情——你解一道数学题，写到中间会停下来说"OK 到此为止，我得到了 b+7 整除 56，下面去找 56 的因子"，然后把前面的草稿翻过去不看了。但 LLM 没有"翻过去"这个动作。

一代一代人在补这个缺口：有人在 prompt 里加 hack；有人造特殊 token 当压缩槽；有人把推理切成块、靠 RL 学习块间衔接；MEMENTO 走到这一步，问的是——能不能让模型自己生成"备忘录"，并且在物理层面把旧块从 KV cache 里抹掉，但又不丢掉这块里隐含的信息？

* 演化叙事

** 2022 | CoT: 让 LLM 把推理写出来

*** 它看到了什么问题

GPT-3 之前的范式是 prompt → answer，一步到位。结果就是难一点的算术题、需要多步推理的题，模型经常错——因为它在内部没有足够的"计算空间"把中间结果存下来。

*** 它的解法

把答案前的推理过程显式地写在输出里。"Let's think step by step." 让模型把脑子里的草稿打到 token 流上。这一下子，模型有了一个外部的工作记忆——就是它自己刚写的那串字。

*** 为什么这招管用（以及哪里埋了新坑）

Transformer 的注意力机制让后面的 token 可以看到前面所有的 token。所以一旦中间推理被写出来，后面就能引用、就能基于它再推。等于把"工作记忆"从模型的隐藏态搬到了 token 序列上——隐藏态有限，token 序列可以很长。

新坑：=token 序列可以很长= 这件事，本身就是下一个十年的麻烦。

** 2023 | Gist Tokens: 把信息压进特殊 token 的 KV

*** 它看到了什么问题

prompt 越来越长。每次 inference 都得把完整 prompt 重新编码一遍，KV cache 越来越大，重复算力越来越多。

*** 它的解法

训练若干个特殊的 "gist" token，让模型学会把 prompt 的语义压缩到这几个 token 的 KV 表征里。下次只需要把这几个 gist token 的 KV 缓存下来，prompt 本身可以扔。

*** 为什么这招管用（以及哪里埋了新坑）

把"信息"从"显式 token"转移到"KV 张量"。本质上是用 KV cache 当作压缩存储器。

新坑：压进去的东西人看不懂了。如果模型出错，你没法看 gist token 来知道它压了什么进去——可解释性整个被牺牲掉。

** 2024-2025 | o1 / DeepSeek-R1: 让模型想很久

*** 它看到了什么问题

CoT 写一两段不够。AIME、IMO 这种题，往往要 2 万、3 万 token 的反复尝试、回溯、自我纠错才能解出来。

*** 它的解法

用 RL（PPO/GRPO/...）训练模型，奖励"答对"而不是"步数少"。模型自己学会要"想很久"。DeepSeek-R1 等开源出来后，几万 token 思考链成了标配。

*** 为什么这招管用（以及哪里埋了新坑）

测试时算力可以换准确率——你给模型更多 token 的空间，它就能解更难的题。

新坑：KV cache 爆了。32K token 的思考链全部要呆在注意力窗口里——每个旧 token 都跟新 token 等价竞争注意力，但其实绝大部分旧 token 早就用完了（一个错误的中间假设，一次失败的代数变形）。模型自己没有"把这段擦掉"的能力。

** 2025 | InftyThink: 切块 + 文字总结

*** 它看到了什么问题

承接上面：思考链是平的、扁的、不可压缩的。但事实上人类做数学题不是这样想的——你解到中间会停下来说"OK，这一段的结论是 X，前面的草稿我不再看了"。

*** 它的解法

把思考链切成若干块。每块末尾让模型生成一个 summary。开始下一块时，prompt = 原始问题 + 之前所有块的 summary，而不是 + 之前所有块的原文。用 SFT 训练模型学会这种"块结束、写总结、从总结接着想"的模式。

*** 为什么这招管用（以及哪里埋了新坑）

把推理变成马尔可夫式的：新块只依赖上一块的 summary，不依赖全部历史。KV cache 不再随推理变长而线性涨——它在每个块结束时被"重置"到 summary 长度。

新坑（两个）：
1. 块间衔接靠 SFT 学，不够稳。模型有时会写出无法继续的 summary，或者下一块和 summary 对不上。
2. =丢掉原文= 这件事可能太狠了——原始 thinking 里有一些"未明确说出来但有用"的细节，被 summary 完全抹掉。

** 2025 | The Markovian Thinker: 用 RL 学块到块

*** 它看到了什么问题

InftyThink 的块间衔接是 SFT 教出来的，但 SFT 看不到"接不上"的反馈。模型可能在 summary 里漏一个关键中间值，下一块就直接死。

*** 它的解法

把"块结束 → 写 summary → 下一块" 整个流程套上 RL。最终奖励是"答对题"，模型自己会学到：summary 里要把下一块真正需要的信息留下来，不重要的可以扔。这就是 "Markovian" ——保证状态转移完整。

*** 为什么这招管用（以及哪里埋了新坑）

RL 给了模型反馈：哪些 summary 让下游成功了，哪些没有。比 SFT 的"模仿一个正确 summary"信号强得多。

新坑：依然是 =文本级= 的 carry-over。summary 是几句话，几句话能编码的信息有限。原始 thinking 块里隐藏着的"非语言可表达"的信息（比如某些 token 之间的几何关系、注意力模式中的隐含联系）全部丢失。

** 2026 Apr | MEMENTO: 双信息流

*** 它看到了什么问题

回头看 InftyThink/Markovian Thinker 这条线，它们都犯了同一个错误：=把旧块的 KV cache 整个扔了，只留下文本 summary= 。问题是，旧块的 KV 里包含了 summary 文字之外的信息——那个块的模型内部表征。

MEMENTO 的关键实验：他们手动把 memento 的 KV 重新计算（在没有原始块作为上下文的情况下），结果 AIME24 准确率掉了 15 个百分点。说明 memento 自己的 KV 张量里，真的"偷偷"携带了原始块的信息。

*** 它的解法

不是物理删除旧块，而是 =注意力 mask 掉旧块= 。流程：

1. 模型生成思考块 T_i（占用 KV cache）
2. 模型紧接着生成 memento M_i（一段压缩文本，几句话）
3. =关键= ：M_i 的 KV 在 T_i 仍在 cache 里时被计算——所以 M_i 的 KV 看过了完整的 T_i
4. 然后把 T_i 的 KV 物理删除，但保留 M_i 的 KV
5. 后面的 token 通过 attention mask 只能看到所有 M_i 和当前块，看不到任何 T_i

加上数据集 OpenMementos（228K 条标注好的 trace + summary 对），用两阶段 SFT。再用他们改的 vLLM fork 做 RL，进一步关闭和 baseline 的精度差距。

*** 为什么这招管用（以及哪里埋了新坑）

=双信息流= ：每个块的信息其实通过两条通道传给后面——(a) memento 的文字本身；(b) memento token 的 KV 张量（在它生成时见过完整的原始块）。这两条通道互补：文字保可解释性，KV 保隐式信息。

省的资源是真的省：~2.5x KV 峰值下降，~1.75x 吞吐提升。精度损失很小（AIME26 在 Qwen3-32B 上只掉 2.6 pp）。

新坑（MEMENTO 自己留下的）：
1. 还是需要 SFT 一轮——OpenMementos 数据集 22.8 万条，离 frontier 训练规模还小
2. memento KV 是"隐式信息"——MEMENTO 论文自己证明它存在，但没说清里面到底是什么。可解释性退步了一点
3. 块边界是用 GPT-5.x 离线打分 + 算法切的，online 切块（让模型自己决定何时停）还没解
4. 评估只到 32B，scaling 行为更高规模未知

* 前沿延伸

截至 2026-04，MEMENTO 是这条线的最新公开论文。但它指出了几个邻接方向值得追踪：

- *PENCIL* (Yang et al. 2025): 小模型 (25M) 用 reduction rule 学习"擦掉"中间推理，在 3-SAT 之类的合成任务上展示了"超越固定 context 长度问题求解"的潜力。MEMENTO 把它列为未来工作——能否把 PENCIL 的"erase"思想搬到 frontier 规模和真实任务上？
- *KV cache 修剪* (ThinKV, R-KV, LazyEviction, Reasoning Path Compression): inference-time 直接基于注意力模式裁剪 KV，正交于 MEMENTO，可以叠加
- *Sliding-window attention* (Olmo-3): 架构层面限制注意力跨度。MEMENTO 论文证明 Olmo-3-7B 可以和 MEMENTO 组合，提示这是未来的一种叠加方式
- *MemAgent / MEM1*: 训练模型管理外部 memory（不是自己的 thinking），主要用于工具调用、检索场景。和 MEMENTO 是同一类"教模型自管 context"的思想，但目标域不同（agent 而非数学推理）

* 洞见

读完整条线，看到一个深层的认知迁移：

*认知 1：从"想出来" → "想得起来"。* CoT 解决了"模型能不能想"，但代价是它什么都想着，扔不掉。后面这十年其实在补一个能力—— =选择性遗忘= 。人脑非常擅长做这件事；LLM 一直不会做。MEMENTO 这条线的核心突破，就是给模型装上了"擦黑板"的能力。

*认知 2：压缩的载体在迁移。* 最早是 token（CoT 写显式步骤）。然后是 KV 张量（gist token 把信息压进表征）。然后是文本 summary（InftyThink）。再然后是 RL-trained summary（Markovian Thinker）。MEMENTO 走到 =文本 + KV 同时携带= ——这其实是承认了一个事实：人类的语言不足以无损压缩 LLM 自己的思维状态。需要一条隐式通道来补充。

*认知 3：可解释性和效率的张力。* Gist token 走到极端：纯 KV，最快，但不可读。InftyThink 走另一极：纯文本，可读，但效率不够 / 信息可能丢。MEMENTO 找到一个折中——主体可读，但留一条暗流。问题是：这条暗流里到底是什么？MEMENTO 自己也答不出来。下一篇论文很可能是来探这件事的。

*下一步最可能往哪走：*
1. =自适应 block 边界= ——模型自己决定何时停下、何时写 memento（去掉离线切块依赖）
2. =memento KV 的可解释性研究= ——那里面到底装了什么？能不能控制它装什么？
3. =跨任务 transfer= ——MEMENTO 是在数学/科学/编程上训出来的，能否泛化到 agent、long-document、多模态？
4. =RL on memento policy= ——MEMENTO 的 RL 现在还只是 close 精度差距，没有奖励"memento 写得好"。如果显式奖励 "memento 让下游成功"，会怎样？

* 启发

这条河给"做研究"的启示：

*启示 1：注意范式之间的"残值"。* InftyThink/Markovian Thinker 扔掉旧块时，扔掉的是 KV 张量——但他们没意识到 KV 张量本身就是一种压缩。MEMENTO 的洞察来自：=别人扔掉的东西里，还有可用的信号= 。这是一种"考古"式的发现，在已有方案里找未被利用的副产品。

*启示 2：物理动作 ≠ 信息动作。* InftyThink 的"丢弃"是物理（文本被替换为 summary），但信息丢失也是真实的。MEMENTO 的"丢弃"是物理（KV 被 free），但 attention mask 之前 memento 已经看过了原始块——所以信息没丢。同一个"丢弃"动作，序列上的微小差异决定了是否丢失信息。

*启示 3：可解释性可以"分层"。* 不一定要选"全部可读"或"全部不可读"。MEMENTO 的双流方案——显式层人可读、隐式层为效率服务——是一个值得在其它地方复用的设计模式。比如：医疗 AI 的诊断推理（显式部分给医生看，隐式部分供模型自己用）。

落点在 *能用* ：

- 如果你在做长 CoT 推理产品，立刻可以试 MEMENTO——~2.5x KV 节省、~1.75x 吞吐提升，是真金白银的成本下降
- 如果你在做 agent、长文档摘要、多轮对话，这套"块 + memento + KV mask"范式可以借鉴——核心是 attention mask 的设计模式，不依赖具体任务
- 如果你在做 LLM serving 框架，应该立刻关注 vLLM 是否会 merge MEMENTO 这种 "in-place KV masking" 能力——这可能成为下一代 inference engine 的标准特性
- 如果你在做 LLM 训练，OpenMementos 数据集已开源（228K traces）——这是少有的"带 segment + summary 标注"的开源数据，对训练长推理模型有直接价值
