Evaluation is comprehensive across six transformer families and multiple cache budgets using LongBench F1. Practicality is high: the method is simple (reserve 10% at boundaries), requires no retraining or calibration, and integrates into existing KV managers. The inference perf axis is weak—no wall-clock speedup numbers—but the quality-vs-memory trade-off is clearly useful for resource-constrained serving. The work is secondary to primary compression (quantization / pruning) but solidly in-scope as a KV layout optimization.

#### 摘要
研究在全局共享 KV cache 容量约束（globally capped decode-time）下对七种 KV eviction 策略的系统性评估。核心发现是所有策略（LRU、H2O、SnapKV、StreamingLLM、Ada-KV、QUEST、Random）在 prompt boundary 处存在共同漏洞：缺乏结构保护时，F1 在六个 pure-transformer 模型上崩溃至接近零（≤0.064）。在每个 boundary 处预留 10% cache 作为结构保护后，C=256（13% retention）下可恢复 C=2,048 参考天花板质量的 69–90%，且跨七个 LongBench 模型成立。进一步发现：结构保护到位后，各 scoring 策略的差异退居次要——简化变体与 LRU 在 K=32 时 TOST 等价（Δ=0.02），per-head allocation 仅带来额外的小幅增益。

- ⚠️ 七种 eviction 策略在无 boundary 保护时 F1 全部崩溃至 ≤0.064
- 🎯 在每个 prompt boundary 预留 10% cache，C=256 下恢复 69–98% 参考质量
- 📊 保护到位后简化 scoring 变体与 LRU TOST 等价（K=32，Δ=0.02）；K=8 时 attention 策略超 LRU 0.011–0.021 F1
- 💡 position-0 sink 持有 ~75% prefix attention mass，是 structural protection 关键原因

#### 摘要

- 📊 结果：在 Qwen3-30B-A3B 和 GLM-4.7-Flash 上分别超越最强 dynamic MoE baseline 6.1 和 4.0 分

### 10. Prune, Update and Trim: Robust Structured Pruning for Large Language Models (8/10) 🔁
**arxiv** · `2605.18331` · 2026-05-18
👥 Diego Coello de Portugal Mecke, Tom Hanika, Lars Schmidth-Thieme · 🏷 cs.LG
🔗 [arXiv](https://arxiv.org/abs/2605.18331) · [PDF](https://arxiv.org/pdf/2605.18331v1)

🧪 pruning_distill · structured pruning with sequential FFN updates and attention-head removal · cal: 未明确说明；后训练剪枝通常成本较低 · perf: 未报告具体端到端性能数据

#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 4/5 · practicality 4/5 · composite 8/10 · 这是一项结构化剪枝方法，针对FFN层和注意力头的移除进行了创新（顺序更新、权重补偿），并扩展到Grouped-Query Attention。方法相对简洁，代码开源。但缺陷在于摘要未报告具体的准确性基准（MMLU、GSM8K等）、最大模型规模、端到端推理加速和校准成本等关键指标，导致难以完整评估实用性和准确性保留。评分考虑：(1)算法简洁性强、结构化剪枝易部署，(2)缺乏速度数据、(3)PTP通常低成本，(4)但缺乏完整的准确性和性能报告。

#### 摘要
现有 post-training pruning（PTP）方法在剪除 FFN hidden nodes 和整层 attention 时未补偿引入的误差，导致极端稀疏率下性能急剧下降。Putri 提出三项改进：对未剪除的 FFN 权重做误差补偿更新、顺序剪枝 FFN 层以利用前层更新信息、以 attention-head 粒度替代整层 attention 剪枝，并扩展支持 Grouped-Query Attention（GQA）。在多模型、多稀疏率及不同数据集上的实验验证了 Putri 的泛化性，尤其在极端稀疏率场景下仍保持 SOTA 性能。

- 🎯 方法：顺序剪枝 FFN 层并更新残余权重补偿误差，以 attention-head 粒度而非整层剪枝 attention
- 💡 创新：将 attention-head 级剪枝扩展至 GQA 架构，提升对现代 LLM 的适用性
- 📊 结果：在极端稀疏率下仍达 SOTA，优于无权重补偿的现有 PTP 方法
- 🔧 工程：方法结构简单，代码已开源，支持多种模型和稀疏率范围
