**Pruning / 蒸馏** · topic 4/5 · practicality 4/5 · composite 8/10 · 层结构化剪枝加激活对齐恢复是实际部署的标准剪枝+恢复流程。闭形最优解的推导表明算法简洁且校准成本低（小数据集+线性求解），无需梯度更新。相关性评分4而非5是因为摘要未报告具体基准精度（MMLU、GSM8K、困惑度），仅声称"改进"和"一致"——需要完整实验表才能确认精度保留程度。实用性4反映了简洁的无训练设计（有利）、低校准成本（有利）但缺少端到端推理加速数据和最大模型大小的明确说明（中性）。若论文包含标准基准对标和具体加速曲线，可升至5/5。

#### 摘要
Layer pruning 删除整个 Transformer decoder block 后，残存层的输入 hidden state 分布与训练时不匹配，导致显著性能下降。Ghosted Layers 提出一个 training-free 的 boundary activation alignment 模块，通过小规模 calibration set 求解闭合形式的最优线性算子，重建被剪掉层引入的 activation 偏差。该算子对应 alignment objective 的无约束最优解，而现有方法仅限于受限算子子空间内的次优解。在多个 LLM backbone 和剪枝策略上，该方法在 accuracy 和 perplexity 上持续优于已有 training-free baseline，同时保留 layer pruning 的效率收益。

- 🎯 training-free 闭合形式线性算子，修复 layer pruning 后的 activation 分布偏移
- 💡 证明所提算子是 alignment objective 的无约束最优解，优于现有受限子空间方法
- 📊 在多个 LLM backbone 与剪枝策略上，accuracy 和 perplexity 均超越已有 training-free baseline
- 🔧 仅需小规模 calibration set，不引入额外训练开销，保留 layer pruning 的推理效率

#### 🧭 为什么推送这篇
**Survey / 方法论与对比** · topic 3/5 · practicality 2/5 · composite 5/10 · 这项工作是对LLM激活模式的测量和分析研究,属于survey桶。论文系统地测量了六个LLM架构在144个任务-模型组合上的激活值、注意力熵和稀疏性,揭示了编码器和解码器模型的计算特性差异。这种激活特征的统一刻画对低比特量化(需要精确的激活范围估计)和模型选择有直接指导意义。然而,该工作的主要局限是:(1)没有报告模型规模和具体架构名称,无法确认测试模型是否达到1B+参数;(2)完全缺少下游精度评估(MMLU/GSM8K/HumanEval),只报告激活统计,难以判断其对压缩算法部署的实际价值;(3)没有展示这些激活特征如何直接驱动特定量化配置的选择,或与已知的量化方法(GPTQ/AWQ/SmoothQuant)如何交互。因此虽然在scope内,但相关性和实用性都偏中等——这类激活分析工作只有在与具体的压缩部署、量化精度对标或校准成本权衡相结合时,才能达到更高评分。

#### 摘要
对六种 LLM 架构在十二类 cognitive task 上的 neural activation pattern 进行系统性分析，测量指标包括 final activation value、attention entropy 和 sparsity pattern。通过 144 个 task-model 组合的实验，发现数学推理任务在所有架构上产生最高的 attention entropy，而 decoder 模型的 sparsity 显著高于 encoder 模型。结论为模型选型与优化提供参考，但未提出新方法或压缩技术。

- 📊 144 个 task-model 组合覆盖 6 种架构 × 12 类 cognitive task
- 📊 数学推理任务在所有架构中 attention entropy 最高
- 📊 decoder 模型 sparsity pattern 显著高于 encoder 模型
- ⚠️ 纯分析性工作，未提出新的量化/压缩方法
