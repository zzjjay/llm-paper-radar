- 💡 创新：提出 HIF（Hardware-efficient LUT output Format），改善 LUT decode 的性能、能耗与芯片面积

- 🎯 方法：单套 GQLA 权重同时暴露 MQA-absorb 与 GQA 两条解码路径，runtime 按硬件自动切换

- 💡 创新：TransGQLA 将预训练 GQA checkpoint 转换为 GQLA，无需从头预训练
- 🔧 工程：GQA 路径支持最多 8-way zero-redundancy tensor parallelism，H20 可结合 MTP（s_q=2）使用

- TransMLA — 扩展为 TransGQLA，支持 GQA checkpoint 转 GQLA
- GQA — GQLA 的 per-group 路径结构上等价于 GQA，作为 KV cache baseline

#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 4/5 · practicality 4/5 · composite 8/10 · 这是一篇层结构化剪枝论文，针对LLM decoder块移除后的激活分布失配问题。核心贡献是闭形式最优线性算子，通过小校准集解决边界激活对齐。虽然没有报告具体基准（MMLU、GSM8K等），但声称在多个LLM骨干上一致改进准确率/困惑度。训练无关恢复模块和小校准集使其实用性强。然而，缺乏具体的准确率数值、推理加速数据和最大模型规模，阻止了topic_relevance达到5分；practicality因为算法简洁和零训练成本达到4分。

#### 摘要
Layer pruning 将整个 Transformer decoder block 从 LLM 中移除，导致剩余层的输入 hidden state 分布失配，引发显著的性能下降。Ghosted Layers 提出一种 training-free 的恢复模块，通过求解 boundary activation alignment 问题，从少量 calibration 数据推导出闭合解形式的最优线性算子，重建被剪除层引入的 activation 偏差。该解对应 alignment 目标的无约束最优解，而现有方法仅在受限算子子空间内搜索；实验表明，在多种 LLM backbone 和 pruning 策略下，该方法在 accuracy 和 perplexity 上持续优于现有 training-free 基线，同时保留了 layer pruning 的效率收益。

- 🎯 针对 layer pruning 后的 hidden state 分布失配问题，提出 training-free 的闭合解线性算子恢复模块
- 💡 证明所提解为 alignment 目标的无约束最优解，现有方法均为受限子空间内的次优解
- 📊 在多种 LLM backbone 和 pruning 策略下，perplexity 和 accuracy 均优于 training-free 基线
- 🔧 仅需少量 calibration 数据即可推导最优算子，无需任何梯度训练
