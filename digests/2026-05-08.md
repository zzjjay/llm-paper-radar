#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 4/5 · composite 9/10 · RDKV将KV cache压缩建模为rate-distortion问题，统一eviction与quantization为同一bit分配框架，通过reverse water-filling在prefilling后一次性决策每个token/channel的bit宽（0到全精度）。方法简洁有理论依据，无需重训练，在LongBench/RULER/InfiniteBench三个长上下文benchmark上均显著超越baseline，并报告了4.5x decode加速和1.9x内存降低（128K context），具有清晰的实际部署价值。联合优化eviction与quantization是该领域创新点，topic和practicality均评分较高。


#### 🌊 Paper River
[倒读批判链](../paper-river/RDKV-2605.08317.org)

#### 摘要
将 KV cache 压缩建模为率失真问题，把 eviction（0 bit）和 quantization（>0 bit）统一视为同一比特分配方案的两个端点，从而联合优化。RDKV 根据每个 token 或 channel 对 attention 计算造成的失真推导权重，再用 reverse water-filling 在 prefilling 后一次性分配从全精度到 0 bit 的位宽。在 LongBench、RULER、InfiniteBench 上平均超过最佳 baseline 9.1%，LongBench 仅保留 2.48% cache 即可恢复 97.81% 精度。相比 full-cache FlashAttention-2，在 128K 上下文下取得 4.5x decode 加速和 1.9x 峰值显存下降。

- 💡 用率失真框架统一 eviction 与 quantization 的 bit 分配
- 🎯 基于 attention 失真权重 + reverse water-filling 分配位宽
- 📊 LongBench 仅留 2.48% cache 恢复 97.81% 精度
- 📊 128K 上下文下 4.5x decode 加速、1.9x 显存降低

- 🎯 单次 post-training 嵌套 N 个子模型，训练成本省 N 倍
