#### Summary
CONF-KV addresses KV cache dominating GPU memory in long-horizon LLM inference by turning the decoder's next-token distribution into a scalar confidence score that sets a per-step cache budget: more context is retained under uncertainty, aggressive pruning under confidence. Within each budget, tokens are ranked by combined accumulated attention mass and recency, with a protected recent window. The system is paired with blockwise online-softmax attention, mixed FP16/INT8 storage, and a pyramidal per-layer budget. Across four model families and up to 4K generations, memory matches a 512-token sliding window while staying within 1.5–2.1 PPL of full KV; on 32K Needle-in-a-Haystack it reaches 91.4% retrieval vs 53.8% (sliding) and 80.6% (H2O), and on 75 VisualWebArena tasks retains 95.3% of full-KV success at 2.8× lower peak memory.

- 💡 Next-token distribution confidence dynamically sets per-step KV budget
- 🔧 Combines FP16/INT8 mixed-precision storage with pyramidal per-layer budgets
- 📊 91.4% on 32K NIAH vs 53.8% sliding and 80.6% H2O
- 📊 Retains 95.3% of full-KV success on VisualWebArena at 2.8× lower peak memory

#### Summary
Existing LLM serving systems waste large fractions of KV cache memory through fragmentation and redundant duplication, capping batch size. PagedAttention adapts OS-style virtual memory and paging to the attention KV cache, and the vLLM system built on top achieves near-zero KV waste plus flexible KV sharing within and across requests. At matched latency vLLM delivers 2-4× higher throughput than FasterTransformer and Orca, with larger gains for longer sequences, bigger models, and more complex decoding.

- 🎯 OS paging-inspired KV cache management, near-zero fragmentation
- 💡 Flexible KV cache sharing within and across requests
