### 2. The Residual Stream Is All You Need: On the Redundancy of the KV Cache in Transformer Inference (7/10)

👥 Kaleem Ullah Qasim, Jiashu Zhang, Muhammad Kafeel Shaheen... · 🏷 cs.LG, cs.AI
🔗 [arXiv](https://arxiv.org/abs/2603.19664) · [alphaxiv](https://www.alphaxiv.org/abs/2603.19664)
📡 Sources: arxiv
🧪 kv_cache · residual-stream recomputation, KV cache elimination · Gemma 3-4B · cal: data-free，无需标定 · perf: recomputation比KV cache读取快5x（中等batch size）

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · 本文提出KV cache在理论上是完全冗余的（可从residual stream bit-identical重建），并基于此构建KV-Direct方案：只checkpoint residual向量而非KV对，显著压缩峰值内存（42 MB vs 103 MB，Gemma 3-4B上5 KB vs 136 KB per token）。核心洞察新颖，且在6个模型上验证了bit-identical重建。实际部署路径存在：代码已开源，方法conceptually简单。主要局限是最大测试模型仅4B，缺乏标准NLP benchmark（MMLU等）的准确率评测，无A100/H100端到端吞吐数据，且大batch下recomputation的实际延迟影响未充分分析。对kv_cache压缩社区有actionable insight，但规模和评测深度不足以打满分。

#### 摘要
作者证明 transformer 推理中的 KV cache 完全冗余：每层的 key/value 是 residual stream 的确定性投影，从单个 residual 向量重算 K/V 可获得 bit-identical 的零误差重建。在 135M-4B 的六个模型上，跨任务 residual patching 得到 D_KL=0，完全移除 cache 后 greedy decoding 输出 token 级一致。基于此提出 KV-Direct，每 token 仅 checkpoint 5 KB 的 residual（vs 标准 136 KB 的 KV pair），在 Gemma 3-4B 上 20 轮对话峰值显存 42 MB（标准 cache 超 103 MB），与 H2O、StreamingLLM、SnapKV、TOVA、window-only 五种 eviction 基线对比保持 100% token match（基线降至 5-28%），且 per-op 重算延迟在中等 batch 下比读取 cached tensor 快达 5x。

- 💡 证明 K/V 是 residual 的确定性投影，重建 bit-identical 零误差
- 📊 Gemma 3-4B 每 token 5 KB vs 标准 136 KB，20 轮显存 42 MB vs 103 MB
- 📊 对比 5 种 eviction 基线 100% token match，基线仅 5-28%
- 🔧 中等 batch 下重算 K/V 比读 cache 快达 5x
