#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · 本文从数学上证明KV cache是residual stream的确定性投影，因此可以完全消除并按需重算，得到bit-identical输出。KV-Direct方案将每token存储从136KB KV pair压缩到5KB residual vector，峰值内存显著降低，且重算速度在中等batch size下比cache读取快5x。理论新颖，代码公开，实用信号清晰。主要弱点：最大测试模型仅4B，缺少MMLU/HellaSwag等任务精度评估（仅token-match与D_KL），且在大batch或长序列场景下recomputation的计算开销分析不够完整；产品化路径需要更大规模（≥7B）验证及端到端throughput数据。topic_relevance适度降至4（无标准benchmark评估），practicality为3（算法简洁、data-free，但缺乏大模型实测与throughput对比）。

#### 摘要
论文证明 transformer 推理中的 KV cache 完全冗余：每层 key/value 都是 residual stream 的确定性投影，从单个 residual 向量重算 K/V 可获得 bit-identical 结果（D_KL=0），在 135M 到 4B 共六个模型上验证 greedy decoding 输出 token 完全一致。基于此提出 KV-Direct：每 token 只 checkpoint residual 向量（Gemma 3-4B 上 5 KB vs 完整 KV pair 的 136 KB），按需重算 K/V。20 轮对话下 peak memory 保持 42 MB，而标准 KV cache 增长到 103 MB；对比 H2O、StreamingLLM、SnapKV、TOVA 等 eviction baseline，KV-Direct 在所有预算下保持 100% token match，baseline 退化到 5-28%。延迟分析显示中等 batch size 下重算比读 cached tensor 快最多 5x。

- 💡 证明 K/V 由 residual stream 决定，重算 bit-identical（D_KL=0）
