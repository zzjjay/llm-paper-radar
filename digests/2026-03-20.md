### 2. The Residual Stream Is All You Need: On the Redundancy of the KV Cache in Transformer Inference (7/10) 🔁
**arxiv** · `2603.19664` · 2026-03-20
👥 Kaleem Ullah Qasim, Jiashu Zhang, Muhammad Kafeel Shaheen... · 🏷 cs.LG, cs.AI
🔗 [arXiv](https://arxiv.org/abs/2603.19664) · [alphaxiv](https://www.alphaxiv.org/abs/2603.19664)
📡 Sources: arxiv
🧪 kv_cache · KV cache recomputation from residual stream checkpoints · Gemma 3-4B · cal: data-free，无需标定数据 · perf: 每次重计算比读取缓存快最多5x（中等batch size），但端到端吞吐未报告

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · 该论文提出KV-Direct方案：从每token的residual stream向量（5 KB/token）按需重计算KV，取代完整KV cache（136 KB/token），实现有界内存推理。理论证明KV是residual stream的确定性投影（零重构误差），并在135M-4B参数的六个模型上验证。方法无需标定数据，内存节省明显（42 MB vs 103 MB），且单步重计算比读取cache快5x，但尚未提供完整端到端吞吐基准，且最大测试模型仅4B，缺乏在≥7B模型上的端到端serving性能数据。对于production部署，重计算开销在大batch/长序列场景的综合影响还需更多验证，实际可部署性存疑但思路清晰。

#### 摘要
论文证明 transformer 中的 KV cache 在信息论意义上完全冗余：每层 key/value 都是 residual stream 的确定性投影，从每个 token 的单个 residual vector 重算 K/V 可获得 bit-identical 结果。在 6 个模型（135M-4B，4 个架构家族）上验证：跨任务 residual patching 在每层产生 D_KL=0，去掉 cache 完全重算在 greedy decoding 下输出 token 完全一致。基于此提出 KV-Direct，只 checkpoint residual vector（Gemma 3-4B 上 5 KB/token vs 完整 KV 的 136 KB），按需重算 K/V；20 轮对话中峰值显存稳定在 42 MB，而标准 cache 增长到 103 MB。对比 H2O、StreamingLLM、SnapKV、TOVA 等 5 个 eviction baseline，KV-Direct 在所有 cache budget 下保持 100% token match，baseline 降至 5-28%；单 op 延迟分析显示中等 batch 下重算比读 cache 快至多 5x。

- 💡 证明 KV cache 完全冗余：从 residual 重算 K/V bit-identical
- 🔧 KV-Direct: 5 KB/token residual checkpoint 替代 136 KB KV
- 📊 20 轮对话峰值 42 MB vs 标准 cache 103 MB
- 📊 全 budget 下 100% token match，eviction baseline 仅 5-28%
