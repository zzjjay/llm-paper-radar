### 3. HISA: Efficient Hierarchical Indexing for Fine-Grained Sparse Attention (7/10)
**hf_daily** · `2603.28458` · 2026-03-30
👥 Yufei Xu, Fanxu Meng, Fan Jiang... · 🏷 cs.CL

📡 Sources: hf_daily (👍 44, 💬 1)
🧪 other · hierarchical sparse attention indexing · DeepSeek-V3.2 · cal: 无需训练，直接替换indexer模块 · perf: 2倍加速@32K上下文，4倍加速@128K；核心索引阶段性能提升显著

#### 🧭 为什么推送这篇
**其它** · topic 3/5 · practicality 4/5 · composite 7/10 · 稀疏注意力加速引擎，但无量化/剪枝/蒸馏等压缩成分。虽然改进了DSA索引效率，但属推理优化范畴，非核心压缩方法。得分3因为与稀疏性相关且有长文本性能改进；practicality=4因为算法简单、无训练、部署成本低、性能增益明显，但非压缩核心贡献。

#### 摘要
Token-level sparse attention（如 DeepSeek Sparse Attention, DSA）需要对每个 query 扫描全部历史 token 进行打分，indexer 本身存在 O(L²) 的瓶颈。HISA 将 flat token scan 替换为两阶段层次化搜索：先用 block-level coarse filter 对 pooled block representatives 打分剪枝，再在候选 block 内用原始 indexer 做 token-level refinement，完整保留下游 Sparse MLA operator 所需的 exact top-k sparsity pattern 且无需额外训练。在 DeepSeek-V3.2 上直接替换 indexer（无 fine-tuning），kernel-level benchmark 显示 32K context 下加速 2×、128K 下加速 4×，Needle-in-a-Haystack 和 LongBench 质量与原始 DSA 相当，token selection sets 的 mean IoU 超过 99%。

- 🎯 方法：两阶段层次化索引——block-level coarse filter + token-level refinement，替换 DSA 中 O(L²) 的 flat scan indexer
- 📊 结果：32K context 加速 2×，128K context 加速 4×，mean IoU > 99% 几乎无 selection 损失
- 💡 创新：无需训练即可 drop-in 替换 DeepSeek-V3.2 的 indexer，完整保留 exact top-k sparsity pattern
- ⚠️ 局限：显著优于 block-sparse baselines，但后者在质量上与原始 DSA 有差距

#### 📎 相关方法 / 对比基线
- DSA — 直接替换其 indexer，质量对齐且效率显著提升
