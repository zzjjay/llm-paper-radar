#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 4/5 · practicality 4/5 · composite 8/10 · SALR将LoRA低秩适配与稀疏剪枝结合，理论上证明冻结base权重静态剪枝可最小化误差界，并用truncated-SVD低秩适配器恢复残差信息。方法有明确的硬件效率设计（bitmap编码+两阶段pipeline GEMM），在GSM8K和MMLU上验证了50%稀疏度下的性能保持，并报告了1.7x推理加速和2x模型压缩。但最大测试模型规模未明确说明，且50%非结构化稀疏在主流推理栈（vLLM/TensorRT-LLM）中的部署需要自定义kernel支持，部署路径需额外工程投入。整体属于pruning_distill范畴，兼具实用信号。

#### 摘要
针对 LoRA 微调虽减少可训练参数但底层稠密权重仍带来高存储/计算开销、而直接幅值剪枝会显著损害 LoRA 性能的问题，作者在统一的 MSE 框架下提出 SALR：仅对冻结的 base 权重做静态剪枝（证明此举最小化剪枝误差上界），并用 truncated-SVD 低秩 adapter 恢复被丢弃的残差信息，理论上将每元素 MSE 降低 (1 - r/min(d,k)) 倍。系统侧将多个低秩 adapter 融合为单次拼接 GEMM，并采用 bitmap 编码 + 两阶段流水的 decode+GEMM 实现真正的压缩与加速。实验上在多种 LLM 上实现 50% 稀疏度，在 GSM8K 和 MMLU 上匹配 LoRA 精度，模型尺寸 2× 缩减，推理加速最高 1.7×。

- 💡 仅剪 frozen base 权重并用 SVD adapter 补残差
- 📊 50% 稀疏下 GSM8K/MMLU 持平 LoRA
- 🔧 bitmap 编码 + 拼接 GEMM 两阶段流水
- 📊 模型 2× 压缩，推理最高 1.7× 加速
