#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 4/5 · practicality 4/5 · composite 8/10 · SALR将LoRA微调与稀疏剪枝统一在MSE框架下，通过静态剪枝冻结base权重并用truncated-SVD低秩适配器恢复残差信息，从而实现真实模型压缩和推理加速。方法理论基础清晰，GEMM融合和bitmap编码设计使其具备较强的部署可行性，在GSM8K和MMLU上验证了50%稀疏度下的性能保持。算法设计相对简洁，calibration成本低，1.7x加速路径可信。但最大测试模型规模未明确披露，且非结构化稀疏在主流GPU上的真实加速依赖bitmap解码pipeline，略有部署复杂度。整体属于pruning_distill范畴，practicality中等偏上。

#### 摘要
针对 LoRA 仅减少可训练参数但底层 dense 权重仍带来存储与计算开销的问题，提出 SALR：在 MSE 框架下统一 low-rank adaptation 与 sparse pruning。理论证明只对冻结的 base weights 做静态剪枝最小化 pruning error bound，并通过 truncated-SVD low-rank adapter 恢复残差信息，将 per-entry MSE 降低 $(1 - r/\min(d,k))$ 倍。工程上将多个 low-rank adapter 融合为单个 concatenated GEMM，并采用 bitmap 编码 + 两阶段流水 decoding+GEMM 设计实现真实压缩与加速。在多个 LLM 上达成 50% sparsity，GSM8K 与 MMLU 持平 LoRA，模型尺寸缩小 2×，推理加速最高 1.7×。
