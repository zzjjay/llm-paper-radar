#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · CodeComp是一个training-free的KV cache压缩框架，将静态程序分析（Code Property Graph via Joern）集成到LLM推理中，替代纯attention-based eviction策略。在bug localization和code generation基准上优于attention-only压缩基线，并能无缝集成到SGLang agentic coding pipeline。方法创新点在于引入代码结构先验（call sites、branch conditions等）来指导KV token保留，部署成本低（无训练、无模型修改）。主要缺点是评估的模型规模未知，且针对代码任务有特定领域限制，适用场景较窄；缺乏明确的推理speedup数据。整体上属于实用的KV cache压缩工作，具备较好的部署可行性。

#### 摘要
针对 agentic coding 任务中长代码上下文导致 KV cache 成为推理瓶颈的问题，CodeComp 在 training-free 设定下将静态程序分析引入 token 重要性评估。具体做法是利用 Joern 提取 Code Property Graph 先验，识别 call sites、branch conditions、assignments 等结构关键 token，弥补纯 attention-only 方法对代码语义结构的忽略。在 bug localization 与 code generation benchmark 上，同等显存预算下超越 attention-only baseline，激进压缩下仍恢复大部分 full-context 精度，patch 生成质量与 full-context 推理持平，并已集成至 SGLang agentic coding pipeline，无需修改模型。
