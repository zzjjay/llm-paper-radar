#### 摘要

- 🎯 用 block diffusion 模型单次前向传播并行生成所有 draft token，打破 autoregressive drafting 的顺序瓶颈
- 💡 draft 模型以 target model 提取的 context features 为条件，提升 draft 质量与 acceptance rate
- 📊 在多模型多任务上实现超 6x lossless 加速，比 EAGLE-3 最高快 2.5x

#### 🧭 为什么推送这篇
**知识蒸馏** · topic 4/5 · practicality 4/5 · composite 8/10 · 核心贡献是通过RL方法中的控制变量稳定蒸馏训练过程。这是后训练知识蒸馏的改进，主要关注训练稳定性而非推理压缩。虽然论文证明了在推理基准上的改进，但：(1)未报告最大模型规模；(2)无端到端推理加速数据；(3)压缩目标（模型大小/延迟）未明确体现。属于蒸馏范畴但偏向训练稳定性的理论改进，而非直接的推理时间/内存压缩。

#### 摘要
On-Policy Distillation (OPD) 因单样本 Monte Carlo 估计器的高梯度方差而训练不稳定。本文提出 vOPD，将 OPD 重新表述为 policy-gradient RL，并引入 control variate baseline——推导出其 closed form 恰好是每个 token 上 student 与 teacher 之间的 per-token negative reverse KL divergence，无需额外的 critic 或推理开销。该 baseline 以 detached 形式从已有 forward pass 中直接获取，保持梯度无偏的同时降低方差；进一步用 top-k 近似 baseline 可在不损失性能的前提下再降低计算开销。在数学和科学推理 benchmark 上，vOPD 持续优于 vanilla OPD，并达到最昂贵的 full-vocabulary baseline 的性能水平。

- 🎯 将 OPD 梯度方差问题转化为 RL control variate baseline 问题，closed form 为 per-token negative reverse KL divergence
- 💡 baseline 直接从 forward pass 获取，无需额外 critic 或推理，梯度保持无偏
- 📊 vOPD 在数学与科学推理 benchmark 上优于 vanilla OPD，匹配 full-vocabulary baseline 性能
- 🔧 top-k 近似 baseline 进一步降低计算开销且不损失性能

#### 📎 相关方法 / 对比基线
- OPD — 被 vOPD 稳定化，vanilla 版本梯度方差高
