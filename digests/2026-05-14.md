🔗 [arXiv](https://arxiv.org/abs/2605.12825) · [alphaxiv](https://www.alphaxiv.org/abs/2605.12825) · [HF](https://huggingface.co/papers/2605.12825) · [GitHub](https://github.com/chiennv2000/orthrus)

#### 🧭 为什么推送这篇
**Trending / 高热度但未归桶** · topic 3/5 · practicality 4/5 · composite 7/10 · Orthrus提出将autoregressive LLM与diffusion并行生成结合的双视图框架，通过共享KV cache实现并行token生成，声称lossless且7.8x加速。该工作不涉及量化、剪枝、KV压缩等传统压缩算法，属于decoding加速范畴。按规则，pure speculative decoding如有实质性parallel/dual-view新框架可归入trending而非hard_gate。Orthrus的dual-view diffusion drafter设计与Few-shot anchor中Orthrus例子完全一致（直接点名），应归入trending bucket。无标准benchmark评估（MMLU/HumanEval等）是弱点，但加速比显著且memory开销极小，对推理优化团队有一定参考价值。

#### 摘要
Orthrus 提出一种双架构框架，在冻结的 autoregressive LLM 上附加一个轻量可训练 diffusion 模块，使两个 view 共享同一 KV cache：AR head 负责 prefill 构建高保真 KV，diffusion head 负责并行生成 token。通过两个 view 之间的 exact consensus 机制保证无损推理，规避了纯 diffusion LM 性能下降和无收敛保证的问题。最终实现最高 7.8x 的推理加速，KV cache 仅有 O(1) 额外开销，新增参数极少。

- 🎯 冻结 AR LLM + 轻量 diffusion 模块共享同一 KV cache
- 💡 双 view exact consensus 机制保证 lossless 推理
- 📊 最高 7.8x 加速，仅 O(1) memory 开销
- 🔧 可无缝集成进现有 Transformer 架构
