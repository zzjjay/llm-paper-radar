#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 本文是一项benchmark/measurement研究，系统评估多种KV cache eviction策略（H2O、SnapKV及其decoding变体等）在long-reasoning任务（GSM8K、MATH500）上的表现，并揭示了reasoning trace场景下heavy-hitter tracking的优越性，以及低budget下eviction与推理成本之间的tradeoff。属于`survey`/`kv_cache`边界，但核心贡献是新的evaluation发现而非全新算法，且引入了decoding-enabled SnapKV变体，更接近kv_cache实验工作。最大测试模型为Llama-3.1-8B，虽小于7B规模门槛的通常要求，但该门槛仅对ptq bucket生效，kv_cache bucket允许。方法简单可部署，calibration免训练，实用性较强，但仅在8B规模验证略为局限。

#### 摘要

### 2. Adaptive Soft Rolling KV Freeze with Entropy-Guided Recovery: Sublinear Memory Growth for Efficient LLM Inference (7/10) 🔁
**arxiv** · `2512.11221` · 2025-12-12
👥 Adilet Metinov, Gulida M. Kudakeeva, Bolotbek uulu Nursultan... · 🏷 cs.LG, cs.AI, cs.CL
🔗 [arXiv](https://arxiv.org/abs/2512.11221) · [alphaxiv](https://www.alphaxiv.org/abs/2512.11221)
📡 Sources: arxiv
🧪 kv_cache · sliding-window entropy-guided KV freeze with off-GPU storage · LLaMA-3 8B · cal: 无需训练，inference-time，data-free · perf: active KV cache减少55-67%，无端到端延迟数字

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · 该论文提出一种training-free的KV cache压缩框架，通过entropy引导的soft-freeze机制临时挂起低重要性token的KV更新，并将其存入off-GPU存储而非永久丢弃，避免了传统eviction方法的信息损失。方法架构无关、无需fine-tuning，在LLaMA-3 8B上实现55-67% active KV cache减少并通过needle-in-haystack测试。但评估较弱：仅测试8B模型，benchmark仅限于needle-in-haystack，缺乏MMLU/GSM8K等标准benchmark的系统评测，且off-GPU存储的实际端到端延迟影响未被量化。此外，off-GPU swap机制在高吞吐serving场景下实用性存疑。整体属于有价值的kv_cache方向探索，但证据尚初步。

#### 摘要
提出 ASR-KF-EGR，一种 training-free 的推理时 KV cache 压缩框架。核心机制是在 sliding attention window 内识别低重要性 token 并对其 KV 更新进行可逆的 soft-freeze，被冻结的 token 不会被永久 evict，而是转移到 off-GPU 存储并按需通过 entropy-guided recovery 恢复。引入 sublinear freeze scheduling，使冻结时长随重复低重要性检测次数次线性增长，避免过度压缩。在 LLaMA-3 8B 上 active KV cache 减少 55-67%，并通过 needle-in-haystack 检索测试。

- 🎯 Soft-freeze 替代 eviction，token 转 off-GPU 可恢复
- 💡 Sublinear freeze scheduling 防止过度压缩
- 📊 LLaMA-3 8B 上 active KV cache 降低 55-67%
- 🔧 Training-free、架构无关，通过 needle-in-haystack 测试
