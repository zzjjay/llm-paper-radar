#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型压缩工作，引入线性注意力和常数内存 KV cache 两项核心设计用于高效视频生成；NVFP4 量化实现 2.4x 推理加速；训练成本极低且模型可部署在消费级 GPU；虽未报告完整精度基准（如 FVD 数值），但与同级别模型对标且推理加速显著，属于实用高效的扩散压缩方向

#### 摘要
SANA-Video 是一个小型 diffusion 模型，目标是高效生成最高 720x1280 分辨率、分钟级时长的视频。核心设计有两点：用 linear attention 替代 vanilla attention 以应对视频生成中大量 token 的计算开销；以及基于 linear attention 累积性质设计的 block-wise autoregressive 方案，使用固定内存的 constant-memory KV cache，无需传统 KV cache 即可维持全局上下文。训练成本仅需 64 张 H100 GPU 跑 12 天（约为 MovieGen 的 1%），与同规模 SOTA 小模型（Wan 2.1-1.3B、SkyReel-V2-1.3B）相比延迟降低 16x，并可在 RTX 5090 以 NVFP4 精度部署，生成 5 秒 720p 视频从 71s 加速至 29s。

- 🎯 用 linear attention 构建 Linear DiT，并设计 constant-memory block-wise KV cache 支持分钟级长视频生成
- 📊 训练成本仅 64×H100 GPU 跑 12 天，约为 MovieGen 的 1%
- 📊 与 Wan 2.1-1.3B、SkyReel-V2-1.3B 相比延迟降低 16x；NVFP4 部署后 720p 5秒视频推理从 71s 降至 29s（2.4x 加速）
- 🔧 支持在 RTX 5090 GPU 上以 NVFP4 精度部署，实现消费级硬件可用的高质量视频生成
