#### 摘要
自回归 LLM 的 sequential decoding 导致高延迟和低 GPU 利用率，speculative decoding 虽能缓解瓶颈但 draft 阶段仍是自回归的。DFlash 将一个轻量级 block diffusion 模型作为 draft model，在单次 forward pass 内并行生成所有 draft tokens，并通过从 target model 提取的 context features 来条件化 draft model，从而同时提高 drafting 效率和 token 接受率。实验表明，DFlash 在多种模型和任务上实现超过 6x 的无损加速，比 SOTA 方法 EAGLE-3 高出最多 2.5x 的加速比。

- 🎯 用 block diffusion model 替代自回归 draft model，单次 forward pass 并行生成全部 draft tokens
- 💡 Draft model 以 target model 提取的 context features 为条件，兼顾质量与接受率
- 📊 多模型多任务上实现超过 6x lossless acceleration
- 📊 比 SOTA speculative decoding 方法 EAGLE-3 提速最多 2.5x

### 2. SANA-Video: Efficient Video Generation with Block Linear Diffusion Transformer (8/10) 🔁

👥 Junsong Chen, Yuyang Zhao, Jincheng Yu... · 🏷 cs.CV, cs.AI
🔗 [arXiv](https://arxiv.org/abs/2509.24695) · [PDF](https://arxiv.org/pdf/2509.24695v2)

#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型压缩，设计了线性注意力和常数内存KV缓存以实现高效长视频生成；NVFP4量化展示端到端加速；但评估主要为视觉质量（VBENCH）和推理延迟，缺少传统定量指标（如FID）。得分4因为是有效的扩散模型推理优化并有实际部署案例，但评估不如传统扩散压缩全面。

#### 摘要
SANA-Video 是一个小型 diffusion 模型，目标是在有限资源下高效生成 720x1280 分辨率、分钟级时长的视频。核心方法包含两点：以 linear attention 替代 vanilla attention 的 Linear DiT（降低大 token 数下的计算开销），以及基于 linear attention 累积特性设计的 constant-memory KV cache，支持 block-wise autoregressive 长视频生成而无需传统随序列增长的 KV cache。训练成本仅为 MovieGen 的约 1%（64 张 H100 上 12 天），在与 Wan 2.1-1.3B、SkyReel-V2-1.3B 等同量级模型竞争性能的同时，实测延迟快 16x；在 RTX 5090 上以 NVFP4 精度部署，生成 5 秒 720p 视频从 71s 加速至 29s（2.4x speedup）。

- 🎯 用 linear attention 构建 Linear DiT，并设计 constant-memory KV cache 实现分钟级长视频的 block-wise autoregressive 生成
- 📊 实测延迟比 Wan 2.1-1.3B 和 SkyReel-V2-1.3B 快 16x，保持竞争性视频质量
- 📊 RTX 5090 上以 NVFP4 部署，5 秒 720p 视频生成从 71s 降至 29s（2.4x speedup）
- 🔧 训练成本 64×H100 共 12 天，约为 MovieGen 的 1%

- SkyReel-V2-1.3B — 同量级小模型，作为性能对比基线
- MovieGen — 训练成本参照系，SANA-Video 仅需其 1%
