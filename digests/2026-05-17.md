**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · VeriCache是一个推理框架，将有损KV cache压缩（token dropping、quantization）与全KV cache验证结合，通过speculative decoding机制保证输出与full KV cache完全一致。核心贡献在于系统设计：压缩KV解码与full KV swap并行化（HBM-bandwidth vs PCIe/network-bound），并利用压缩KV输出相似性拉长draft horizon来分摊swap开销。支持广泛的压缩算法（通用compressor接口），实现4x吞吐提升且输出无损。该方法具备实际部署价值，特别适合code generation和tool calling等对输出精确性要求高的场景。评估中未明确报告最大模型规模和标准benchmark，略扣topic_relevance分。

#### 摘要
针对 KV cache 压缩方法（token dropping、quantization）在长序列解码时输出逐渐偏离 full-KV 结果、导致代码生成和 tool calling 失败的问题，VeriCache 借鉴 speculative decoding 思路：用压缩 KV cache 起草 token，再用 full KV cache 验证以保证输出与 full-KV 解码完全一致。关键系统设计是将 full KV cache 放在 GPU 外，利用压缩 KV 解码受 HBM 带宽限制、full KV swap 受 PCIe/网络限制的特性并行执行，并通过长 drafting horizon 摊销 swap 开销。框架支持长上下文解码和远程 prefix caching，可统一接入多种压缩算法，并与传统 speculative decoding 组合，吞吐相比 full-KV 推理提升最高 4X。
