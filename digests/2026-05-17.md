#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · VeriCache提出将KV cache压缩方法（token dropping、量化）与speculative decoding结合的推理框架，通过压缩KV draft token后用full KV验证，实现无损输出。核心贡献是系统工程：将compressed-KV HBM解码与full-KV PCIe换入并行化，摊销验证开销。与`kv_cache`桶高度匹配——支持广泛的压缩后端，吞吐提升最高4×，且输出与full KV完全等价，解决了现有lossy KV压缩在长输出场景下的catastrophic failure问题。无需训练或校准，部署成本低；但最大测试模型规模未明确，且系统实现复杂度（CPU/GPU overlap、PCIe swap调度）相对较高，practicality略有折扣。

#### 摘要
针对 KV cache 压缩方法（token dropping、quantization）在长输出场景下逐渐偏离 full-KV 结果、导致代码生成和 tool calling 失败的问题，VeriCache 用压缩 KV cache 进行 draft、再用 full KV cache 进行 verify，从而保证与 full-KV 解码 bit-wise 一致的输出。系统关键在于将 full KV cache 放在 GPU 外，利用 compressed-KV decoding (HBM-bound) 与 full-KV swap (PCIe/网络-bound) 的并行性，以及压缩 KV 高接受率带来的长 drafting horizon 来摊销 swap 开销。框架兼容多种 token-dropping 与 quantization 压缩器，适用于 long-context decoding 和远程 prefix caching，并可与传统 speculative decoding 叠加，相比 full-KV 推理实现最高 4× 吞吐提升。

- 💡 压缩 KV draft + full KV verify，输出与 full-KV 完全一致
- 🔧 full KV 置于 GPU 外，draft 解码与 swap 通过 HBM/PCIe 带宽错峰并行
- 📊 相比 full-KV 推理最高 4× 吞吐
- 🎯 统一 compressor 接口支持 token-dropping 与 quantization，可叠加 speculative decoding
