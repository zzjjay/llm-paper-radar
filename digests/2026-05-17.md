#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 该工作是KV缓存压缩算法，通过轻量级索引器和动态稀疏注意力实现长上下文推理加速。论文基于三个关键观察（只有部分注意头需要全长上下文、长距离检索受低维子空间支配、有用token预算与查询相关）构建RTPurbo方案。优点：仅需数百步训练步数（极低校准成本）、在长上下文基准和推理任务上保持准确率、获得显著推理加速（9.36x预填充、2.01x解码）。缺点：最大模型规模未知、accuracy_benchmarks表述不具体（未列举具体基准名称如RULER/LongEval）、端到端内存开销未充分量化、索引器设计细节相对简洁但实现和调优空间有限。该工作是合理可部署的KV缓存压缩方案，难度中等，适合工程团队在6个月内集成。

#### 摘要
长上下文推理受限于 full attention 的二次复杂度，现有方法在效率、训练成本和精度之间存在取舍。RTPurbo 基于三个观察：仅少量 attention heads 需要完整长上下文处理；long-range retrieval 由低维子空间主导，可用 16 维 indexer 高效检索；有用 token budget 强烈依赖 query，动态 top-p 比固定 top-k 更合适。该方法仅为 retrieval heads 保留完整 KV cache，并引入轻量 token indexer 实现 sparse attention，只需数百步 fine-tuning 即可将标准 full-attention 模型转化为高度稀疏模型，在 1M context 下实现 9.36× prefill 加速和约 2.01× decode 加速，同时保持近无损精度。

- 🎯 方法：识别 retrieval heads，用 16 维 indexer 检索相关 token，动态 top-p 稀疏化替代固定 top-k
- 📊 结果：1M context 下 prefill 加速 9.36×，decode 加速约 2.01×，精度近无损
- 💡 创新：仅需数百步训练步骤将 full-attention 模型转为稀疏模型，无需 native sparse pretraining
- 🔧 工程：非 retrieval heads 使用 sparse attention，retrieval heads 保留完整 KV cache，兼顾效率与精度

#### 📎 相关方法 / 对比基线
- native sparse training — 本文方法无需其昂贵的 pretraining，仅百步适配
- heuristic token eviction — 被替代，本文用动态 top-p indexer 提升准确性

**Diffusion 压缩** · topic 5/5 · practicality 4/5 · composite 9/10 · 核心扩散模型压缩工作，提出旋转感知的W4A4 PTQ框架(PCA低秩子空间保留+旋转对齐)，在PixArt-Σ和FLUX.1上相对SVDQuant显著改进FID/PSNR，引入VLM-as-a-Judge评估协议为扩散模型量化的首创，实现了2.3x端到端推理加速和2.1x内存节省。算法相对简洁(PCA+正交变换+权重融合)，校准成本未明确但应在可接受范围，已有Triton优化内核支持实际部署。

#### 摘要
DiT 模型在 W4A4 PTQ 下面临严重质量退化问题。DiRotQ 通过 PCA 识别激活值的低秩主方差子空间，将该子空间系数保留在更高精度，其余分量量化为 4-bit；推理时将激活旋转到 PCA 基，逆旋转离线融合进权重，并结合 GPTQ 做权重量化。在 PixArt-Σ / MJHQ-30K 上，DiRotQ 取得 FID 15.9、PSNR 19.1 dB，超越 SVDQuant（FID 18.9、PSNR 17.6 dB）；对 12B FLUX.1-dev 实现 2.1x 显存压缩、2.3x 推理加速（相对 BF16 baseline，RTX 4090 24 GB）。

- 🎯 PCA 低秩子空间保留主激活方差，其余分量量化为 4-bit，逆旋转矩阵离线融合进权重
- 📊 W4A4 下 PixArt-Σ FID 15.9 / PSNR 19.1 dB，优于 SVDQuant（FID 18.9 / PSNR 17.6 dB）
- 📊 12B FLUX.1-dev 显存降低 2.1x，推理速度提升 2.3x（vs BF16，RTX 4090 24 GB）
- 💡 首次引入 VLM-as-a-Judge 评估协议用于扩散模型量化的感知质量与 prompt 对齐评估
