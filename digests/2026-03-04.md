#### 🧭 为什么推送这篇
**PTQ（训练后量化）** · topic 5/5 · practicality 4/5 · composite 9/10 · 本文提出了基于SQNR分解的量化误差理论框架，将误差分解为'集中度'和'对齐度'两个维度，并据此设计了CAT变换（covariance估计的块线性变换），在4-bit PTQ场景下同时优化权重与激活的集中度和对齐度。该方法属于function-preserving transform类PTQ改进，与QuaRot/SpinQuant/SmoothQuant属同一技术栈，但提供了更完整的理论解释和新的优化维度。算法简洁，仅需小规模calibration集估计covariance，无需梯度更新，calibration成本低。最大测试模型大小未明确说明，但实验覆盖'多个LLMs'，结合LLM量化领域惯例，预计包含≥7B规模模型。实用性较强，可作为现有rotation-based PTQ流水线的替代或增强组件。

- 🎯 CAT 用 calibration 协方差联合优化两者

- 🔧 轻量线性变换，function-preserving

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · 本文提出利用attention head间的线性可预测性（inter-head linear structure）进行KV cache压缩：只缓存参考头的KV状态，其余头通过轻量线性映射在线重建，实现2x KV cache减少。测试模型包括Qwen3-32B等大模型，具备较强规模说服力。方法本身简洁（线性映射），calibration成本低，与现有推理栈兼容性好。主要不足：精度损失较大（4.5-5.5 pp甚至更高），缺乏端到端延迟/throughput实测数据，部分benchmark名称未披露。综合看是一篇有新颖分析视角（survey角度）和初步压缩应用的论文，偏向kv_cache探索性工作，实用性有待进一步验证。

#### 摘要
研究发现预训练 Transformer 的 attention head 之间存在普遍的线性结构：同层内任一 head 的 QKV 向量可由 2-5 个 peer head 线性重建（C4 上 Key 的 mean R² ≈ 0.76，GSM8K 上常 > 0.85）。该性质是预训练学到的而非架构固有，在随机初始化时几乎不存在，并随 OLMo-2 训练过程快速涌现，与 Key 投影子空间的层内对齐增强相关。基于此冗余，作者提出只缓存 reference head 的 KV，其余 head 通过轻量线性映射在线重建，实现 2x KV cache 压缩。在 Falcon3-10B 和 Qwen3-32B 上五个 benchmark 平均掉 4.5-5.5 个百分点，Llama-3.1-8B 上更大；重建 Key 比重建 Value 损失更小。

- 💡 同层 2-5 个 reference head 可线性重建其他 head 的 QKV
- 📊 C4 Key 重建 mean R²≈0.76，GSM8K 常 >0.85
- 🎯 仅缓存 reference head KV 实现 2x KV cache 压缩
- 📊 Falcon3-10B/Qwen3-32B 平均掉 4.5-5.5pp，Key 比 Value 更易重建
