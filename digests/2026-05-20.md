**PTQ（训练后量化）** · topic 5/5 · practicality 4/5 · composite 9/10 · TORQ directly addresses MXFP4 activation quantization via principled orthogonal transformations (Schur-Horn theorem + maximum-entropy rotation), a core PTQ recipe for next-generation low-bit inference. #### 摘要
MXFP4 格式对 LLM activation 量化存在严重精度损失，根源在于两种结构性失衡：inter-block variance 极端不均衡和 intra-block codebook 利用率不足。TORQ 提出两级正交旋转 PTQ 框架：宏观层面利用 Schur-Horn 定理通过 inter-block 正交旋转重分配 activation 能量，消除高方差 block 对共享 scale 的污染；微观层面用最大熵引导的 intra-block 旋转缓解 codebook collapse。在 Qwen3-32B 上，WikiText perplexity 从直接 RTN 的失效状态降至 8.43（BF16 为 7.61），平均精度从 38.40% 提升至 73.63%（BF16 为 74.82%）。

- 🎯 方法：training-free PTQ，通过两级正交旋转重塑 activation 几何分布以适配 MXFP4 block floating-point 格式
- 💡 创新：宏观用 Schur-Horn 定理做 inter-block 旋转均衡方差；微观用最大熵引导 intra-block 旋转最大化 codebook 信息容量
- 📊 结果：Qwen3-32B 上平均精度从直接 RTN 的 38.40% 提升至 73.63%，与 BF16 的 74.82% 仅差 1.19%
- 📊 结果：Qwen3-32B WikiText perplexity 达 8.43，BF16 基线为 7.61

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 5/5 · composite 10/10 · OScaR通过识别Token Norm Imbalance（TNI）为极端KV缓存量化的关键瓶颈，提出了一个轻量级但有效的解决方案。该方法采用Canalized Rotation加Omni-Token Scaling来缓解序列维度的方差，实现了INT2量化下接近无损的性能。提供了优化的CUDA内核实现、3倍解码加速和5.3倍内存节省，显示高实用性。跨多种LLM类型（文本、多模态、全模态）的广泛评估证实了框架的通用性。代码已开源，部署路径清晰。主要未知点是具体的模型规模和精度基准，但near-lossless声明结合系统加速数据强烈暗示精度保留良好。

#### 摘要
KV cache 的显存占用在长上下文和多模态场景下成为关键瓶颈，现有 per-channel quantization 在极端压缩（INT2）下性能急剧下降。本文从理论和实验两方面指出 Token Norm Imbalance（TNI）是 per-channel 量化保真度的核心制约因素：当 token 组之间 norm 差异悬殊时，共享量化参数会系统性放大误差。OScaR 提出 Canalized Rotation 结合 Omni-Token Scaling 的两阶段变换来消除 TNI 引入的序列维度方差，配合定制 CUDA kernel，在 text-only、multi-modal、omni-modal X-LLMs 上实现 INT2 近无损量化，相比 BF16 FlashDecoding-v2 基线达到最高 3.0x 解码加速、5.3x 显存缩减、4.1x 吞吐提升。

- 🎯 方法：Canalized Rotation + Omni-Token Scaling 两阶段消除 Token Norm Imbalance，推进 per-channel 量化范式
- 📊 结果：相比 BF16 FlashDecoding-v2，解码加速最高 3.0x，显存降低 5.3x，吞吐提升 4.1x
- 📊 结果：INT2 极端量化下仍实现近无损性能，建立新 Pareto front
- 💡 创新：轻量无需复杂量化流水线，统一支持 text-only / multi-modal / omni-modal LLMs
