- 🎯 用 block diffusion 模型替代 autoregressive draft，单次 forward pass 并行生成多个 draft tokens

- 📊 在多类模型和任务上实现超过 6x lossless 加速，较 EAGLE-3 最高提升 2.5x 加速比

#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型压缩工作，结合线性注意力、恒定内存KV cache设计和NVFP4量化。推理性能收益明显（2.4倍NVFP4加速），但缺乏标准图像/视频质量基准（FID、LPIPS、CLIP score）的定量评估。线性注意力设计优雅但需要大规模重新训练（非PTQ）。相比于LLM量化的简洁性，工程复杂度较高。

#### 摘要
SANA-Video 是一个面向高效视频生成的小型扩散模型，支持最高 720x1280 分辨率和分钟级时长视频合成。核心方法包含两点：Linear DiT 以线性 attention 替代标准 attention 以应对视频 token 数量庞大的问题；以及基于线性 attention 累积性质设计的 constant-memory KV cache，通过 block-wise autoregressive 方式在固定显存开销下提供全局上下文，消除传统 KV cache 的线性显存增长。训练仅需 64 张 H100 GPU 运行 12 天（约为 MovieGen 训练成本的 1%），在同级小模型中生成延迟降低 16x，并可在 RTX 5090 上以 NVFP4 精度将 5 秒 720p 视频生成时间从 71s 压缩至 29s（2.4x 加速）。

- 🎯 用 linear attention 构建 Linear DiT，并设计 constant-memory KV cache 实现 block-wise 长视频自回归生成
- 📊 训练成本仅 64×H100×12 天，约为 MovieGen 的 1%
- 📊 与 Wan 2.1-1.3B、SkyReel-V2-1.3B 同级模型相比，生成延迟快 16x
- 📊 RTX 5090 上以 NVFP4 推理，5 秒 720p 视频生成从 71s 降至 29s（2.4x 加速）

#### 📎 相关方法 / 对比基线
