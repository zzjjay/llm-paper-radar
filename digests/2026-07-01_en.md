#### Summary
KV cache memory grows linearly with context length, bottlenecking LLM deployment in long-context settings. Existing Residual Quantization (RQ) methods rely on standard L2 K-means, which suffers from centroid shrinkage in high dimensions—Euclidean centroid averaging weakens angular alignment and degrades directional fidelity. GSRQ introduces Gain-Shape K-means (GSKM), which decouples each vector into a gain (magnitude) and a shape (direction) component and clusters them separately, serving as a drop-in replacement for K-means inside an RQ pipeline. On LLaMA-3-8B at 1-bit, GSRQ raises average LongBench accuracy from 11.34 to 33.54, a gain of 22.20 percentage points over VQLLM.

- 🎯 GSKM decouples gain and shape in K-means clustering to eliminate high-dimensional centroid shrinkage
- 💡 Weighted GSKM is integrated into an RQ pipeline as a drop-in replacement, requiring no model weight modification
- 📊 On LLaMA-3-8B at 1-bit KV cache, LongBench average accuracy improves from 11.34 to 33.54 (+22.20 pp over VQLLM)
- ⚠️ Evaluated only on an 8B model; no end-to-end inference latency or throughput data reported

#### Summary
KV cache grows linearly with context length in long-context LLM serving, exhausting GPU memory and reducing throughput. MosaicKV introduces dynamic two-dimensional KV cache compression, jointly compressing both the sequence and channel dimensions by exploiting non-uniform importance distributions within KV cache elements and selecting compression strategies at the granularity of KV cache segments rather than applying a single global pattern. Compressed KV cache management leverages underutilized GPU and CPU resources to maintain compressed caches and accelerate attention. On an H800 GPU, MosaicKV achieves up to 16x attention speedup, 4.8x lower decode latency, 7.3x higher throughput, and 3x memory reduction over the uncompressed baseline, with only 1.76% average accuracy loss on LongBench and RULER.
