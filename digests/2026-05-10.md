#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 4/5 · composite 9/10 · 本文提出语义感知KV cache四层内存层次结构，将低重要性token的KV offload至CPU DDR而非丢弃，实现零近似误差offloading。核心发现是精度仅由永久eviction比例决定，与HBM占用无关，在7B-32B三种规模和GSM8K/MATH-500四个benchmark上均有验证。方法无需重训或校准，直接在线基于cumulative attention打分，工程实现简洁且与现有推理栈兼容，CPU-GPU传输overhead仅5-7%。与SOTA eviction方法R-KV的对比显示显著精度优势，实用性强。

#### 摘要
针对 reasoning LLM 长 chain-of-thought 场景下 KV cache 占满 HBM 的问题，作者指出永久驱逐低重要性 token 会使精度坍塌至 0-2.5%。方案是基于累积 attention score 将 token 分入 HBM/DDR/compressed/evicted 四层语义感知层级，低重要性 token 卸载到 CPU 内存而非丢弃，attention 计算前以全精度预取回 GPU，实现零近似误差 offloading。理论与实验表明精度只取决于真正被驱逐的比例，而与留在 HBM 的比例无关：仅 3% 驱逐即可在 GSM8K 保留 91%、MATH-500 保留 71% 全 cache 精度；14B 模型在 HBM 减半下匹配未压缩 baseline (90% vs 86%)，传输开销仅 5-7%，HBM 节省 2-48 GB。

- 💡 zero-approximation-error offloading：CPU 暂存+预取，精度只看 eviction ratio
- 📊 3% 驱逐保留 GSM8K 91%、MATH-500 71% 全 cache 精度
- 📊 14B 下 HBM 减半，精度 90% vs baseline 86%，超过 R-KV (0-32%)
- 🔧 真实 GPU-CPU 数据搬运开销仅 5-7%，节省 2-48 GB HBM
