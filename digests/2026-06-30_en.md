#### Summary
KV cache memory grows linearly with sequence length in long-context LLM inference, and existing compression methods—token eviction and semantic grouping—cannot recover token-level detail from compressed spans once they become relevant during generation. SeKV organizes context into entropy-guided semantic spans, storing a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand reconstruction, with a trained zoom-in mechanism that selectively expands query-relevant spans during decoding. The base LLM is kept fully frozen and fewer than 0.05% trainable parameters are added. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context.

- 🎯 Entropy-guided semantic spans stored in a GPU-CPU hierarchy: summary vectors on GPU for routing, low-rank SVD bases on CPU for token-level reconstruction
- 📊 5.9% average improvement over strongest semantic compression baseline; 53.3% GPU memory reduction at 128K context vs. full KV caching
- 💡 Base LLM fully frozen; zoom-in mechanism adds fewer than 0.05% trainable parameters
- ⚠️ CPU-GPU low-rank reconstruction introduces additional latency not quantified in the abstract

#### 📎 Related methods / baselines
- token eviction — discards token information; SeKV avoids loss via hierarchical storage
- semantic grouping — fixes compression at prefill time; cannot recover detail during generation
