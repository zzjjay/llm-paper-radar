#### 摘要
KV cache 的内存开销是 LLM 推理服务的主要成本之一，现有方法主要沿时间轴做压缩或 eviction，而跨层共享（depth 维度）虽然冗余明显但实现困难，常导致吞吐下降或 TTFT 增加。本文提出 stochastic KV routing：训练时每层随机选择 attend 自身 KV 还是某个前层的 KV（random cross-layer attention），使模型适应各种 depth-wise cache sharing 策略，部署时可灵活匹配硬件约束。实验表明，该方案在 pre-training 和 fine-tuning 阶段均有效，对大模型在数据受限场景下甚至呈现类似正则化的效果，在显著降低 KV cache 内存的同时保持乃至提升性能。

- 🎯 训练时随机让每层 attend 自身或前层 KV，使模型天然支持 depth-wise cache sharing
- 💡 dropping 某层 cache 而非压缩，实现零信息损失的内存节省且不降低吞吐
- 📊 大模型数据受限场景下，方法呈正则化效果，性能持平或提升同时显著压缩 KV cache 占用
- 🔧 兼容 pre-training 与 fine-tuning，部署时可灵活适配未知硬件约束
