#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · core KV cache compression work via early exit + KV sharing，实现了理论层削减与实际wall-clock加速的对齐。training-free + 清晰的推理加速是关键优势。但未明确报告具体模型尺寸和基准测试数值（MMLU/HumanEval等），精度评估粒度偏粗

#### 摘要
decoder-only LLM 的 Early Exit 面临 KV Cache Absence 问题：跳过的层无法为后续 token 提供历史状态，现有的 recomputation 或 masking 方案要么引入额外延迟要么精度损失严重。River-LLM 提出一个 training-free 框架，通过轻量级 KV-Shared Exit River 在 exit 过程中自然生成并保留 backbone 缺失的 KV cache，无需昂贵的恢复操作。此外，利用 decoder block 内的 state transition similarity 预测累积 KV 误差，指导精确的 exit 决策。在数学推理和代码生成任务上实现 1.71 到 2.16 倍的实际 wall-clock 加速，同时保持高生成质量。

- 🎯 training-free token-level Early Exit，通过 KV-Shared Exit River 消除 KV Cache Absence 问题
- 💡 利用 decoder block state transition similarity 预测累积 KV 误差，指导 exit 决策
- 📊 数学推理和代码生成任务上实现 1.71×–2.16× 实际 wall-clock 加速
- ⚠️ 仅在数学推理和代码生成任务验证，未覆盖通用语言理解 benchmark

#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型压缩工作，核心贡献是线性注意力DiT和常数内存KV缓存设计用于高效长视频生成，配合NVFP4量化实现2.4倍推理加速。虽然未报告传统视觉质量指标（FID/LPIPS），但对比SkyReel/Wan等竞争模型具有明显速度优势。算法设计简洁，推理性能收益明显（RTX 5090可部署），满足扩散压缩评分标准。

#### 摘要
SANA-Video 是一个小型扩散模型，目标是高效生成最高 720x1280 分辨率、分钟级时长的视频。核心设计有两点：一是用 linear attention 替代 vanilla attention 作为 Linear DiT 的基础操作，以应对视频生成中大量 token 带来的计算压力；二是基于 linear attention 累积性质设计 constant-memory KV cache，以固定显存代价提供全局上下文，支持分钟级长视频的 block-wise autoregressive 生成。模型在 64 张 H100 GPU 上仅训练 12 天（约为 MovieGen 成本的 1%），与 Wan 2.1-1.3B、SkyReel-V2-1.3B 等同量级模型相比推理延迟快 16x，在 RTX 5090 上以 NVFP4 精度推理 5 秒 720p 视频可从 71s 加速至 29s（2.4x speedup）。

- 🎯 Linear DiT + constant-memory KV cache，以固定显存支持分钟级长视频生成
