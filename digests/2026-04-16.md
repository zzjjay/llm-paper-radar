#### 摘要
On-policy knowledge distillation（OPD）中并非所有 token 位置的学习信号同等重要。该工作系统分析了 OPD 中 token 重要性的来源，发现有价值的 token 集中在两类区域：student entropy 高的位置，以及 student entropy 低但 teacher–student divergence 高（即 overconfident 且错误）的位置。基于此提出 TIP 分类框架（两轴：entropy × divergence），实验表明，仅用 entropy-based sampling 保留 50% tokens 即可匹配全 token 训练并降低 peak memory 最多 47%；进一步利用低 entropy 高 divergence token，不足 10% 的 token 即可接近全 token baseline；在 MATH-500、AIME 2024/2025 和 DeepPlanning benchmark 上跨 Qwen3、Llama、Qwen2.5 多对 teacher–student 验证了上述结论。

- 🎯 方法：TIP 用 student entropy 与 teacher–student divergence 两轴构建 token 重要性分类，指导 OPD 中的 type-aware token 选择
- 📊 结果：entropy-based sampling 保留 50% tokens 匹配全 token 训练，peak memory 降低最多 47%
- 📊 结果：单独利用低 entropy 高 divergence tokens，<10% tokens 训练即接近全 token baseline
- 📊 结果：DeepPlanning 上 <20% tokens 的 Q3-only 训练超越完整 OPD

#### 摘要
SANA-Video 针对高分辨率长视频生成的效率瓶颈，提出两项核心设计：Linear DiT 以线性 attention 替代 vanilla attention 处理视频大规模 token，以及基于线性 attention 累积性质的 constant-memory KV cache，以固定显存代价实现块状自回归长视频生成。训练成本压缩至 64 张 H100 上 12 天（仅 MovieGen 的 1%），可在 RTX 5090 以 NVFP4 精度部署，生成 5 秒 720p 视频从 71s 降至 29s（2.4x 加速）。与同规模 SOTA 小模型（Wan 2.1-1.3B、SkyReel-V2-1.3B）相比性能相当，推理延迟快 16x。

- 🎯 Block Linear Attention 使用 constant-memory state 替代传统 KV cache，实现定长显存的分钟级视频生成
- 📊 训练成本 64×H100 GPU 共 12 天，仅为 MovieGen 的 1%
- 📊 RTX 5090 + NVFP4 推理 5 秒 720p 视频耗时从 71s 降至 29s，加速 2.4x；延迟比同级模型快 16x
- 🔧 可部署于单张 RTX 5090 GPU，支持最高 720×1280 分辨率及分钟级时长视频
