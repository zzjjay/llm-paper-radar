#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 5/5 · composite 10/10 · InnerQ是一个针对KV cache的硬件感知量化方案，核心贡献是沿inner dimension做group-wise量化，使去量化操作与向量矩阵乘法对齐，提高GPU数据复用率，从而在降低显存占用的同时实现实际推理加速（1.3× vs prior方法，2.7× vs 非量化baseline）。方法完全tuning-free，calibration成本极低，三项精度维护技巧（hybrid quant、high-precision window、per-channel normalization folded into parameters）均可在现有推理栈上实现。在Llama和Mistral上验证了精度与速度，具备较强的实际部署可行性，topic_relevance和practicality均打高分。

#### 摘要
InnerQ 是一种无需微调的 KV cache 量化方案，针对 decode 阶段 KV cache 主导显存和延迟的问题。其核心是沿 inner dimension 进行 group-wise 量化，使 dequantization 与 vector-matrix 乘法对齐，提升 GPU 计算单元间的数据复用。为在激进压缩下保持精度，结合三项技术：基于局部统计的对称/非对称混合量化、对 recent tokens 和 attention sink tokens 保留高精度窗口、key cache 的 per-channel normalization 在 prefill 时一次性折入模型参数。在 Llama 和 Mistral 上相比已有 KV cache 量化方法平均加速 1.3×，相比未量化基线加速 2.7×，且 few-shot 评测分数更优。
