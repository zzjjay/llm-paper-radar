**PTQ (post-training quantization)** · topic 4/5 · practicality 3/5 · composite 7/10 · ProjQ proposes a new framework constraining PTQ quantization noise to a low-rank manifold, offloading dominant error components to a downstream LoRA adapter via orthogonal subspace projection, validated on LLaMA-2, Qwen2.5 and Qwen3, where 3-bit matches 4-bit baseline performance. The method has theoretical grounding and open-source code, suitable for joint PTQ+LoRA deployment scenarios. However, the main evaluation metrics are PPL and eval loss, lacking standard downstream benchmarks like MMLU/GSM8K/HumanEval, so per the rules topic_relevance is capped at 3~4; additionally, the alternating optimization algorithm adds calibration complexity, and there is no end-to-end inference speedup data, so practicality is moderate. #### Summary
Sequential PTQ + LoRA pipelines suffer because PTQ leaves residual noise spread across directions that LoRA's limited rank cannot correct, wasting adapter capacity. ProjQ constrains quantization noise to the low-rank manifold via orthogonal subspace projection, using an alternating algorithm that shapes dominant error into a low-rank structure offloadable to LoRA while minimizing residuals in the orthogonal "uncorrectable" subspace. Theory shows ProjQ preserves strictly greater downstream plasticity than standard PTQ. Experiments on LLaMA-2, Qwen2.5 and Qwen3 show up to 2× lower evaluation loss during compensation, and 3-bit ProjQ matches standard 4-bit baselines on language modeling.
