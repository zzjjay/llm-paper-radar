#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 3/5 · practicality 4/5 · composite 7/10 · GRAIL提出一种post-hoc blockwise补偿方法，利用Gram矩阵和ridge regression在结构化剪枝后恢复块的输入输出行为，不需要反向传播或标签数据，与selector无关（Magnitude、Wanda等）。方法简洁，校准成本低，适合实际部署。评估覆盖ResNets、ViTs和decoder-only LLMs，但摘要未明确给出最大测试LLM的规模和具体benchmark（如MMLU/HumanEval），仅提及PPL改善，评估证据偏弱。对于pruning_distill bucket，1B门槛适用，LLM部分规模不明确但属于现代decoder-only架构，故不hard_gate。整体属于LLM剪枝后处理补偿工作，具有一定实用价值，但LLM评估细节不足，相关性中等。

#### 摘要
GRAIL 针对结构化压缩后无需微调的精度恢复问题，提出 post-hoc blockwise compensation：用小 calibration set 收集每个 block 的 hidden activation Gram matrix，通过 ridge regression 求出从压缩后表示到原始 hidden representation 的线性重建映射，并把该映射吸收进下游 projection weights，从而无需 gradient 或 label。方法 selector-agnostic，兼容 Magnitude、Wanda、Gram-based selection 与 folding，当 Gram 接近 identity 时退化为经典 pruning/folding。在 ResNets、ViTs 与 decoder-only LLM 上，相对 data-free 和 data-aware 的 pruning/folding baseline 在常用压缩比下均提升 accuracy 或降低 perplexity，开销可控且无 backpropagation。

- 🎯 Gram matrix + ridge regression 重建 block I/O，零微调
- 💡 重建映射融合进下游 projection，selector-agnostic
- 📊 ResNet/ViT/LLM 上超越 data-free 与 data-aware pruning baseline
- 🔧 仅需少量 forward pass，无 gradient 与 label
