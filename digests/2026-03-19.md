#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · EntropyCache 针对 diffusion language models（dLLMs）的 KV caching 问题，提出以解码 token 分布最大熵为常数开销信号，决定是否跳过或重计算 KV cache。方法 training-free，决策开销为 O(V) 与上下文长度无关，在 LLaDA-8B 和 Dream-7B 上实现 15x-26x 加速，代码开源。dLLMs 并非主流 autoregressive LLM，评测 benchmark 名称未具体列出，精度对比细节不足，但整体方法简洁、部署门槛低，具备较强实用性。目标模型属于语言模型范畴，归入 kv_cache bucket 合理。

#### 摘要

- 💡 用 decoded token 最大熵作为 KV cache staleness 的常数开销 proxy
- 🎯 重算最近 k 个解码 token，决策复杂度 O(V) 与上下文/深度无关
- 📊 标准 benchmark 15.2×-26.4×，CoT benchmark 22.4×-24.1× 加速
- 🔧 training-free，决策开销仅占推理时间 0.5%
