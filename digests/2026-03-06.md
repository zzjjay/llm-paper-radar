#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 4/5 · practicality 3/5 · composite 7/10 · ROSE是对SparseGPT的改进，通过两级权重重排序（列级和块级）优化剪枝顺序，显著提升了LLaMA2-70B等主流LLM的剪枝精度，评估包含PPL和多个zero-shot下游任务，实验规模达70B，属于合格的pruning_distill工作。算法在SparseGPT框架上增加了预剪枝和重排序步骤，复杂度略高但仍属one-shot流程，calibration成本可接受。主要缺点是非结构化稀疏在现有主流GPU推理栈上缺乏直接的端到端加速支持，实用性受限，practicality打3分。

#### 摘要
针对 SparseGPT 在权重呈 columnar pattern 时因预设 left-to-right 剪枝顺序导致性能次优的问题，本文提出 ROSE，通过重排剪枝顺序优先剪除潜在误差较大的权重。具体做法：先 pre-pruning 识别候选权重并估计 column loss 和 block loss，然后在 block 内按 column loss 降序重排列、block 之间按 block loss 重排；并以 block loss 的相对 range 作为指标识别 columnar layers 实现自适应重排。在 LLaMA2-7B/13B/70B、LLaMA3-8B、Mistral-7B 上均优于 SparseGPT 等基线。

- 💡 发现 SparseGPT 的 left-to-right 顺序在 columnar 权重上次优
- 🎯 两级重排：block 内按 column loss、block 间按 block loss
- 🔧 用 block loss 相对 range 自适应识别 columnar layers
- 📊 在 LLaMA2/3、Mistral 上一致超过 SparseGPT
