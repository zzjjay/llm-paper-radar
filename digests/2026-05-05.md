🧪 kv_cache · 2-bit KV-cache quantization with low-rank residual side code + logit correction · cal: 需要校准学习query basis和低秩残差side code，成本未明确 · perf: 无端到端速度数据报告

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 2/5 · composite 6/10 · 本文针对KV-cache量化提出HeadQ，核心创新是在模型可见坐标系（score空间/A²加权）下衡量量化误差，而非传统的存储空间MSE。理论分析扎实，通过null-space干预、反例和query-PCA控制验证了方法，对2-bit KV量化有明显PPL提升。然而实际部署存在较大障碍：需要低秩残差side code的校准学习query basis，额外引入logit correction步骤，系统复杂度明显高于KIVI/KVQuant等方法；评测仅用WikiText-103 PPL，未涉及MMLU/GSM8K等下游任务；最大测试模型规模不明确；无端到端推理速度数据。综合来看，理论价值较高，但practicality受限于方法复杂性和评估不完整。

#### 摘要
本文指出 KV-cache 量化的传统目标——最小化存储空间的重构误差（如 key 的 raw MSE）——与 attention 实际使用方式不匹配：key 通过 logits 读取、value 通过 attention 加权读取，因此误差应在 model-visible 坐标下衡量。作者提出 HeadQ：对 key 端，在 calibration 学到的 query basis 中存储一个 low-rank residual side code，作为 additive logit correction（score-space 修正，模常数偏移）；对 value 端，使用 fixed-attention readout 推出 $A^2$ 加权的 token-distortion surrogate。在 6 个模型的实验中，Fisher/score-space 误差预测 attention KL 远优于 raw key MSE；K-only WikiText-103 2-bit decode 上 HeadQ 消除了最强 2-bit 行约 84–94% 的 excess perplexity，full-KV 2-bit 组合下 HeadQ + $A^2$ value policy 改善了全部 6 个模型。

- 💡 KV-cache 误差应在 model-visible 坐标下度量，而非 storage MSE
- 🎯 key 端 low-rank residual 残差码做 additive logit correction
- 📊 2-bit K-only decode 消除 84–94% 的 excess perplexity
- 🔧 value 端用 $A^2$ 加权 token-distortion 作为量化目标

#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 4/5 · practicality 3/5 · composite 7/10 · 该论文提出Budgeted LoRA，一种在计算预算约束下通过结构化计算分配实现模型蒸馏压缩的框架，最终产出推理时结构高效的学生模型。方法结合dense retention系数、自适应低秩分配和post-training压缩，通过单一budget参数控制压缩强度，可实现1.74x至4.05x的模块加速。该工作属于pruning_distill范畴，具有一定创新性。然而评估较弱：仅报告PPL和in-context learning probes，未在MMLU、GSM8K、HumanEval等标准benchmark上验证；最大测试模型规模未明确说明，存在是否达到7B+的疑问；蒸馏训练成本尚不清晰；方法涉及多个组件（dense系数、低秩分配、post-training选择性压缩），实现复杂度中等。综合来看，方向正确但评估不充分，工程可部署性尚需验证。

#### 摘要
针对LoRA等参数高效蒸馏方法仅降低训练成本、保留dense backbone而无法带来推理加速的问题，提出Budgeted LoRA框架，将模型压缩建模为结构化算力分配问题。通过单一全局compute budget控制最终dense计算保留比例，结合模块级dense保留系数、自适应low-rank分配以及后训练压缩（选择性移除/近似/保留dense组件）三种机制在dense和low-rank路径间重新分配容量。中等预算下匹配标准LoRA的perplexity并获得1.74x compressed-module加速；激进预算下达到4.05x加速且perplexity仅适度退化，同时在function-style in-context learning探针上保持更高准确率。

- 💡 用单一budget dial控制dense到low-rank的算力分配
- 📊 中等预算匹配LoRA PPL并获1.74x加速
