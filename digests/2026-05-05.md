#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型压缩论文，采用线性注意力和常数内存KV缓存设计实现高效长视频生成。包含NVFP4量化部署和明确的推理加速指标（2.4倍），但未报告标准图像/视频质量基准（如FID或LPIPS）。算法设计新颖（线性DiT + block-wise KV缓存），部署实用（支持RTX 5090），但评估主要依赖定性对比和延迟而缺乏量

#### 摘要
SANA-Video 针对高分辨率、长时视频生成效率低的问题，提出两项核心设计：Linear DiT 用线性 attention 替代 vanilla attention 以降低大 token 数下的计算开销；基于线性 attention 累积特性设计 Block Linear Attention 的 constant-memory KV cache，以固定显存代价提供全局上下文，支持分钟级视频生成。训练仅需 64 张 H100 GPU 跑 12 天（约为 MovieGen 的 1%），生成质量与 Wan 2.1-1.3B、SkyReel-V2-1.3B 持平，但实测延迟快 16x；在 RTX 5090 上以 NVFP4 精度推理 5 秒 720p 视频从 71s 降至 29s（2.4x 加速）。

- 🎯 Block Linear Attention 的 constant-memory KV cache，以固定显存实现分钟级长视频 autoregressive 生成
- 📊 与 Wan 2.1-1.3B / SkyReel-V2-1.3B 质量相当，实测延迟快 16x
- 📊 RTX 5090 + NVFP4 推理 5s 720p 视频：71s → 29s（2.4x speedup）
- 🔧 训练成本 64×H100 GPU × 12 天，仅为 MovieGen 的 ~1%
