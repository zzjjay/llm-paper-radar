#### 摘要
针对 KV-cache 量化通常以存储重构 MSE 为目标、却忽略 attention 实际通过 logits 读 key、通过 attention-weighted readout 读 value 的问题，作者主张在 model-visible 坐标系下衡量持久 cache 误差。对 key 端，提出 HeadQ：在 calibration 学到的 query basis 中存储 low-rank residual side code，作为 additive logit correction；对 value 端，用 fixed-attention readout 导出 $A^2$ 加权的 token-distortion surrogate。实验证明 Fisher/score-space 误差比 raw key MSE 更能预测 attention KL，并通过 null-space、query-PCA、wrong-sign 等 ablation 证伪 storage-MSE 替代方案。在 WikiText-103 K-only 2-bit decode 中，HeadQ 消除最强 2-bit 行约 84–94% 的 excess perplexity；全 KV 2-bit 组合下，HeadQ+$A^2$ value policy 在 6 个模型上均有改进。

- 💡 用 score-space 而非 storage MSE 衡量 KV 量化误差
- 🎯 key 侧低秩 residual 在 query basis 中作 logit 校正
- 📊 2-bit K-only 下消除 84–94% excess PPL
- 📊 full-KV 2-bit 组合在 6 个模型上全部改进

### 2. Budgeted LoRA: Distillation as Structured Compute Allocation for Efficient Inference (6/10)

🧪 pruning_distill · LoRA-based structured distillation with compute budget allocation · cal: 需要蒸馏训练，具体成本未说明 · perf: 中等budget下压缩模块1.74x加速，激进budget下4.05x加速

#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 3/5 · practicality 3/5 · composite 6/10 · 该论文提出Budgeted LoRA，一个将模型压缩视为结构化计算分配问题的蒸馏框架，通过全局计算budget控制dense与low-rank路径的容量分配，并进行后训练压缩。与纯LoRA适配不同，该方法确实能在推理时产生结构化压缩收益（最高4.05x模块加速），属于pruning_distill范畴。但评估较弱：仅使用PPL和自定义in-context learning探针，没有MMLU/GSM8K/HumanEval等标准benchmark；最大测试模型规模未明确说明；蒸馏训练成本不明，且需要dense retention系数、adaptive rank分配、后训练压缩等多个组件，算法中等复杂度。整体来看是有意义的压缩探索，但实证评估不充分，实用性中等。

#### 摘要
针对 LoRA 类参数高效蒸馏只降训练成本、不改 dense backbone 导致推理无加速的问题，提出 Budgeted LoRA，把蒸馏建模为受全局 compute budget 控制的结构化算力分配问题。方法包含三部分：模块级 dense 保留系数、自适应 low-rank 分配、以及在训练后选择性删除/近似/保留 dense 组件的压缩流程，由单一 budget 旋钮控制 student 家族。在中等 budget 下 perplexity 与标准 LoRA 持平、compressed-module 加速 1.74x；激进 budget 下加速 4.05x，perplexity 仅中度下降，并在 function-style in-context learning 探针上保留更高准确率。

- 🎯 用全局 compute budget 控制 dense→low-rank 算力再分配
- 📊 中等 budget 持平 LoRA PPL，模块加速 1.74x
- 📊 激进 budget 加速 4.05x，PPL 仅中度退化
- 💡 蒸馏关键在控制 dense 计算如何转移到 low-rank 路径
