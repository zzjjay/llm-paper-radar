#### 摘要
长上下文 LLM 推理中，KV cache 随序列长度线性增长，成为显存瓶颈。现有 token eviction 方法丢失信息，语义分组在 prefill 阶段固定压缩决策，均无法在生成时按需恢复细节。SeKV 将上下文按 entropy 引导划分为语义 span，在 GPU 保留轻量 summary vector 用于粗粒度路由，在 CPU 存储 low-rank SVD basis 用于按需 token 级重建，并训练一个 zoom-in 机制在解码时选择性展开相关 span。在 128K context 下，SeKV 相比完整 KV caching 节省 53.3% GPU 显存，在四个 benchmark 上平均超越最强语义压缩基线 5.9%，base LLM 完全冻结，可训练参数不足 0.05%。

- 🎯 用 entropy 引导的语义 span 分组 + GPU-CPU 分层存储，GPU 存 summary vector，CPU 存 low-rank SVD basis，按需重建 token 级 KV，不丢弃任何信息
- 📊 128K context 下 GPU 显存减少 53.3%，四个 benchmark 上平均超越最强语义压缩基线 5.9%
- 💡 训练 zoom-in 机制实现查询相关 span 的选择性展开，base LLM 完全冻结，可训练参数 <0.05%
- ⚠️ CPU↔GPU 重建引入的延迟开销未报告端到端 speedup，生产 serving 栈集成复杂度较高
