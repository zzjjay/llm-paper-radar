#### Summary
Consistency distillation for video diffusion fails to scale with more inference steps because it replaces the ODE trajectory with a consistency-sampling trajectory, breaking test-time scaling. AnyFlow addresses this by distilling flow maps over arbitrary time intervals (z_t→z_r) rather than fixed endpoint mappings (z_t→z_0), and introduces Flow Map Backward Simulation to decompose full Euler rollouts into shortcut transitions for on-policy distillation. This eliminates both discretization error in few-step sampling and exposure bias in causal generation, achieving results that match or surpass consistency-based methods in the few-step regime while also scaling with additional sampling steps, validated across 1.3B–14B parameter models.

- 🎯 Method: Flow map distillation over arbitrary intervals (z_t→z_r) replaces fixed endpoint consistency mapping to preserve ODE trajectory quality
- 💡 Innovation: Flow Map Backward Simulation enables on-policy distillation by decomposing Euler rollouts into shortcut flow-map transitions
- 📊 Result: Matches or surpasses consistency distillation in few-step regime while gaining test-time scaling with more steps, across 1.3B–14B models
- ⚠️ Limitation: Consistency-distilled baselines degrade with more sampling steps; AnyFlow is the first framework to recover this scaling behavior for video

#### 🧭 Why this paper
**KV cache compression** · topic 4/5 · practicality 4/5 · composite 8/10 · KV cache compression via low-rank latent design with dual decoding paths and hardware adaptation. No accuracy eval weakens topic_relevance to 4. Simple algorith

#### Summary
MLA in DeepSeek-V2/V3 compresses KV cache via low-rank latents but exposes only a single MQA-absorb decoding path optimized for H100 compute-bandwidth ratios, breaking tensor parallelism and MTP gains on H20-class hardware. GQLA introduces a minimal weight modification that exposes two algebraically equivalent decoding paths over identical parameters: MQA-absorb (H100-optimal) and GQA with per-group expanded cache (H20+MTP-optimal), selected at runtime without retraining or custom kernels. TransGQLA converts pretrained GQA checkpoints to GQLA; on LLaMA-3-8B, the MQA-absorb path reduces per-token KV cache to 28.125% of the GQA baseline while the GQA path supports up to 8-way zero-redundancy tensor parallelism.

- 🎯 Method: Dual algebraically equivalent decoding paths (MQA-absorb / GQA) from single weight set, runtime-selected per hardware
- 📊 Result: KV cache compressed to 28.125% of GQA baseline on LLaMA-3-8B MQA-absorb path via TransGQLA conversion
- 💡 Innovation: Pins rooflines of both H100 (s_q=1) and H20 (GQA+MTP, s_q=2) with no retraining or custom kernels
- 🔧 Engineering: Supports up to 8-way zero-redundancy tensor parallelism on GQA path, absent in original MLA

#### 📎 Related / compared methods
- MLA — directly extends; exposes second decoding path over same weights
- TransMLA — extended into TransGQLA for GQA-to-GQLA checkpoint conversion

#### Summary
Distilling Long-CoT reasoning from large reasoning models (LRMs) is impractical at scale due to computational cost; existing trace-selection approaches do post-hoc curation of complete traces without inter-teacher collaboration or dynamic exploration. CoRD performs step-wise reasoning synthesis via collaborative multi-teacher decoding, using predictive perplexity-based scoring and beam search to let heterogeneous LRMs jointly construct coherent reasoning trajectories while retaining diverse high-potential hypotheses. CoRD produces higher-quality distillation data, achieves near teacher-level student performance with fewer structured supervision signals, and generalizes to out-of-domain and open-ended settings without substantial efficiency overhead.

- 🎯 Method: Step-wise multi-teacher beam search decoding guided by predictive perplexity scoring for collaborative reasoning trace construction
- 💡 Innovation: Heterogeneous LRMs jointly build reasoning trajectories step-by-step, enabling complementary reasoning beyond post-hoc trace selection
- 📊 Result: Near teacher-level student performance achieved with fewer, structured supervision signals and no substantial efficiency overhead
- 🔧 Engineering: Generalizes to out-of-domain and open-ended settings; dataset and models publicly released
