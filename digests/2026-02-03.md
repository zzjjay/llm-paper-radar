**Pruning / 蒸馏** · topic 4/5 · practicality 4/5 · composite 8/10 · POP提出stage-aware结构化剪枝策略，仅在prefill阶段跳过深层，decode阶段保留完整模型，从而实现精度与效率的平衡。方法属于pruning_distill范畴，核心思路清晰（virtual gate重要性分析 + 独立KV projection），在Llama-3.1/Qwen3-VL/Gemma-3上验证，覆盖LLM和VLM。实用性较好：算法相对简洁，硬件友好（标准layer跳过），prefill加速1.37x有实际意义；但decode阶段不压缩意味着内存占用与原模型相同，且对decode-heavy workload无帮助，限制了部署场景。整体是一篇有实际价值的inference优化工作，但不是compression-in-the-strict-sense。

#### 摘要
针对结构化剪枝在 LLM/VLM 上精度损失严重的问题，作者通过 virtual gate 机制分析发现：深层对 decode（next-token prediction）至关重要，但在 prefill（context encoding）阶段大量冗余。基于此提出 POP（Prefill-Only Pruning），在 prefill 阶段跳过深层、decode 阶段保留完整模型，并通过独立的 KV projection 维持 cache 一致性、用 boundary handling 保证首 token 准确。在 Llama-3.1、Qwen3-VL、Gemma-3 上 prefill 延迟最高加速 1.37×，性能损失极小。

- 💡 揭示深层对 prefill 冗余、对 decode 关键的非对称性
- 🎯 stage-aware 剪枝：prefill 跳深层，decode 用完整模型
- 🔧 独立 KV projection + boundary handling 保 cache 一致
- 📊 Llama-3.1/Qwen3-VL/Gemma-3 上 prefill 加速 1.37×
