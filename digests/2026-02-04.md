**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · KV缓存压缩核心工作，通过细粒度混合头机制动态选择关键token，在长上下文理解和复杂推理任务上有充分基准评估，展示明显推理加速。缺少详细的内存消耗和校准成本信息，故practicality为4而非5。

#### 摘要
长上下文 LLM decoding 阶段的 KV cache 快速膨胀带来显存和延迟瓶颈。现有方法在层间共享同一组关键 token，忽视了 attention head 功能多样性，导致模型性能下降。LycheeDecode 提出细粒度 hybrid-head sparse decoding：基于 HardKuma 机制将 attention heads 分为少量 retrieval heads（动态识别关键 token）和多数 sparse heads（复用这些 token），并采用 hardware-efficient top-k 选择策略。在 Llama3、Qwen3 上的 LongBench、RULER、AIME24、OlympiadBench 等基准测试中，于 128K context length 下实现最高 2.7x 加速，同时保持与 full-attention 相当甚至更优的生成质量。

- 🎯 用 HardKuma 机制将 attention heads 分为 retrieval heads 和 sparse heads，实现细粒度 KV cache 稀疏化
- 📊 128K context length 下推理速度最高提升 2.7x，同时不低于 full-attention baseline 质量
- 💡 相比层间共享同一关键 token 集合的粗粒度方法，保留 attention head 功能多样性以避免性能损失
- 🔧 采用 hardware-efficient top-k 选择策略，便于实际部署加速

#### 📎 相关方法 / 对比基线

#### 🧭 为什么推送这篇
**Pruning / 稀疏化** · topic 4/5 · practicality 4/5 · composite 8/10 · 核心剪枝贡献：阶段感知策略（前缀剪枝+解码保留），突破已有结构化剪枝的精度瓶颈。端到端加速可衡量，精度保留在标准基准上评估。KV投影与边界处理为实现细节，增加复杂度但可部署。未报告完整QAT成本，推测为PTQ变体，实用性较好。

#### 摘要
现有 structured pruning 方法对 prefill 和 decode 阶段一视同仁，导致显著精度损失。通过引入 virtual gate mechanism 分析各层重要性，发现深层对 next-token prediction（decode）关键但在 context encoding（prefill）中冗余。据此提出 Prefill-Only Pruning（POP）：prefill 阶段跳过深层以降低计算开销，decode 阶段保留完整模型；同时引入独立 KV projection 保证 cache 完整性，以及 boundary handling strategy 确保首个生成 token 的准确性。在 Llama-3.1、Qwen3-VL、Gemma-3 上实验表明，POP 可实现最高 1.37× prefill latency 加速，且性能损失极小。

- 💡 virtual gate mechanism 揭示深层在 prefill 阶段冗余、在 decode 阶段关键的非对称角色
- 🎯 POP 仅在 prefill 阶段剪枝深层，decode 阶段保留完整模型，实现阶段感知推理
- 📊 在 Llama-3.1、Qwen3-VL、Gemma-3 上最高达 1.37× prefill latency 加速，精度损失极小
- 🔧 独立 KV projection 维护 KV cache 完整性，boundary handling strategy 保障首 token 精度
