#### 🧭 为什么推送这篇
**Low-bit（≤ 2-bit）** · topic 4/5 · practicality 4/5 · composite 8/10 · RSR-core 为 binary/ternary LLM 推理提供了优化的低层 CPU 和 CUDA kernel，实现 RSR 算法的实际部署。该工作与 low_bits 桶（1-bit/1.58-bit 权重）高度相关，且具备 HuggingFace 集成和开源代码，部署门槛低。CUDA token generation 最高 1.9x 加速对 ternary LLM（如 BitNet b1.58）有实用价值。主要扣分点：未报告任何 accuracy benchmark（MMLU、PPL 等），无法判断加速是否保持模型质量；CUDA 加速幅度（1.9x）相对有限；测试的最大模型规模未明确说明。整体来看，作为一个 inference engine / kernel 贡献，实用性尚可，但缺乏压缩角度的原创算法贡献和完整评估。

#### 摘要
针对 1-bit / 1.58-bit 量化权重矩阵-向量乘法，已有 Redundant Segment Reduction (RSR) 算法仅有应用层实现，无法融入硬件 kernel。RSR-core 将 RSR 实现为 CPU 与 CUDA 低层优化 kernel，支持 binary 和 ternary 权重对一般 activation 向量的乘法，并提供 HuggingFace 集成用于预处理低位模型和加速推理。在主流 ternary LLM 上，相比 HuggingFace PyTorch baseline 在 CPU 上获得最高 62x 加速，CUDA 上 token generation 最高 1.9x 加速。

- 🔧 将 RSR 算法落到 CPU/CUDA 低层 kernel
- 📊 CPU 上 matvec 最高 62x 加速 vs PyTorch
- 📊 CUDA ternary LLM token 生成最高 1.9x 加速
- 🎯 支持 1-bit / 1.58-bit 权重 × FP activation
