#### 🧭 为什么推送这篇
**Low-bit（≤ 2-bit）** · topic 4/5 · practicality 4/5 · composite 8/10 · 本文提出RSR-core，一个针对binary（1-bit）和ternary（1.58-bit）权重矩阵-向量乘法的高性能CPU/CUDA内核引擎，实现了RSR算法的底层优化，并提供HuggingFace集成以方便部署主流ternary LLM（如BitNet等）。这属于low_bits范畴，实用性较强：算法基于已有RSR理论、代码开源、HuggingFace集成降低工程门槛、CPU和CUDA均有实测加速数据。不足之处在于：CUDA加速仅1.9x（相对baseline HuggingFace PyTorch而非优化的内核），最大测试模型未明确说明，且无任何下游准确率评估（MMLU/GSM8K等）。整体而言是一个有实际价值的内核工程贡献，但缺乏模型规模和精度评估削弱了其直接部署信心。

#### 摘要
针对二值（1-bit）和三值（1.58-bit）权重矩阵与高精度向量的低比特 matrix-vector multiplication，作者将原本停留在应用层的 Redundant Segment Reduction (RSR) 算法下沉为 CPU 和 CUDA 的底层 kernel，构建了 RSR-core 引擎。该引擎集成 HuggingFace，可对低比特模型做预处理并在推理 pipeline 中直接调用。相较于 HuggingFace PyTorch 基线，在 CPU 上对流行的 ternary LLM 实现最高 62x 加速，CUDA 上 token generation 最高 1.9x 加速。

- 🎯 将 RSR 算法实现为 CPU/CUDA 低层 kernel
- 📊 ternary LLM 上 CPU 最高 62x 加速
- 📊 CUDA token generation 最高 1.9x 加速
- 🔧 HuggingFace 集成，production-ready 引擎
