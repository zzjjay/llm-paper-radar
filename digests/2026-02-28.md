#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · RelayCaching是一种training-free的KV cache复用方法，专门针对multi-agent LLM协作场景中的prefill冗余问题。核心思想是将前一个agent的decoding阶段KV cache直接复用于后续agent的prefill阶段，通过稀疏局部重计算修正prefix偏差，在保持精度的同时大幅减少TTFT（最高4.7×）和KV cache内存开销。方法不需要训练，算法相对简洁，可集成到现有inference pipeline。不足之处是目标场景较为特定（multi-agent协作），最大测试模型未明确说明，且benchmark描述较为宽泛，缺乏具体模型规模和标准LLM benchmark（如MMLU/HumanEval）的精确数据，因此topic_relevance评为4而非5。整体practicality较好：training-free、复用率高、TTFT提升明显，工程可行性较强。

#### 摘要
针对多 agent LLM 系统中下游 agent 对上游生成内容重复 prefill 计算导致的 KV cache 显存膨胀和 TTFT 增加问题，提出 RelayCaching：直接将前序 agent 的 decoding 阶段 KV cache 复用到后续 agent 的 prefill 阶段。基于观察——相同内容的 KV cache 在不同阶段高度一致，prefix 引起的偏差仅稀疏分布在少量 layer 和 token 位置——对这些位置选择性重算。在数学推理、通用知识、代码生成等协作任务上实现 >80% KV cache 复用率，TTFT 较标准 pipeline 加速最高 4.7×，精度损失可忽略。
