#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · RelayCaching提出了一种training-free的KV cache复用方法，专为多智能体LLM协作系统设计。核心思路是将前一个agent的decoding阶段KV cache直接复用于后续agent的prefill阶段，通过识别稀疏且局部化的prefix偏差位置进行选择性重计算来保持精度。该方法属于kv_cache bucket，且有明确的效率收益（TTFT最高降低4.7x，KV cache复用率>80%）。不足之处在于最大测试模型规模未明确说明，benchmark名称较模糊（未列明具体基准如MMLU/HumanEval），评估相对宽泛。算法设计简洁（无需重新训练，选择性重计算），部署复杂度适中，与vLLM/SGLang等主流框架的集成路径合理，practicality评4分。

#### 摘要
针对多 agent LLM 协作场景下重复 prefill 共享内容导致的 KV cache 显存和 TTFT 开销问题，作者提出 RelayCaching，将前一 agent 的 decoding KV cache 直接复用到后续 agent 的 prefill 阶段。核心观察是同内容跨阶段的 KV cache 高度一致，prefix 引起的偏差稀疏且局限于少数 layer 和 token 位置，因此仅对这些位置选择性重算即可保精度。在数学推理、通用知识、代码生成等任务上达到 80%+ KV cache 复用率，TTFT 最多降低 4.7×，精度几乎无损。
