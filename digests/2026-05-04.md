#### 摘要
论文提出三个层次的 LLM 量化无损性概念：task-lossless（zero-shot 精度在采样方差内）、distribution-lossless（next-token 分布几乎不可区分，用 Expected Acceptance Rate (EAR) 衡量，EAR≥0.99 表示 99% token 一致）、以及理论上的 γ² variance law（证明对称量化噪声方差比非对称大 γ² 倍，故 distribution-lossless 必须用非对称量化）。基于此提出 SLQ：layer-wise 非均匀量化方法，结合非对称量化与宽 bitwidth search。结果上 task-lossless 可压到 3.3 bits/param，distribution-lossless 需 5-6 bits/param，配合优化 kernel 相对 FP16 加速 1.7-3.6x。

- 💡 提出 EAR 指标：最优 coupling 下 token 一致概率，EAR≥0.99 即 99% 一致
- 💡 证明 γ² variance law：对称量化噪声方差比非对称高 γ² 倍
- 📊 task-lossless 低至 3.3 bits/param，distribution-lossless 需 5-6 bits

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · WindowQuant针对视频语言模型（VLMs）的KV cache提出混合精度量化方案，通过window-level相似度评分快速确定最优bit-width配置，并通过重排KV cache窗口提升hardware执行效率，属于kv_cache压缩范畴。方法思路较清晰，具有一定实用性。但摘要未明确最大测试模型规模、具体benchmark名称及量化精度格式，也未给出推理加速的具体数值，评估信息不足。整体算法设计合理但摘要过于笼统，实用性受限于缺乏具体部署细节和性能数字。

#### 摘要
针对 VLM 推理中视觉 token 序列过长导致 KV cache 显存与延迟问题，提出基于 window 粒度的混合精度 KV cache 量化方法 WindowQuant。该方法包含两个模块：window-level quantization search 通过计算 visual token window 与 text prompt 的相似度快速决定每个 window 的 bit-width，避免 token 粒度搜索的高耗时；window-level KV cache computation 在量化前重排 KV cache window，消除混合精度在硬件上的低效问题。实验表明其在多数据集上优于现有 VLM KV cache 量化方法。

- 🎯 以 window 而非 token 为粒度做混合精度 KV cache 量化
- 💡 用 visual-text 相似度得分代替昂贵的 bit-width 搜索
- 🔧 量化前重排 KV cache window 解决混合精度硬件低效
- 📊 在多数据集上超过 SOTA VLM 与 KV cache 量化方法
