### 1. Statistically-Lossless Quantization of Large Language Models (9/10) 🔁

👥 Michael Helcig, Eldar Kurtic, Dan Alistarh · 🏷 cs.LG

📡 Sources: arxiv
🧪 ptq · non-uniform asymmetric PTQ, layer-wise wide bitwidth search (3.3–6 bits) · cal: layer-wise PTQ，无需梯度更新，校准成本较低 · perf: 优化kernel下相对FP16实现1.7–3.6x加速

#### 🧭 为什么推送这篇
**PTQ（训练后量化）** · topic 5/5 · practicality 4/5 · composite 9/10 · 本文提出SLQ，一种基于非均匀非对称量化的PTQ方法，系统探讨了'统计无损'压缩的三层含义（task-lossless、distribution-lossless和EAR指标），并在低于4 bits实现task-lossless、5–6 bits实现distribution-lossless，同时报告了1.7–3.6x的推理加速。方法核心是layer-wise宽比特搜索配合非对称量化，有代码开源，整体pipeline贴近PTQ实际部署流程。主要不确定性在于最大测试模型规模未在摘要中明确说明，可能影响scale-up置信度；但方法本身对GPTQ/AWQ有明确的benchmark和理论对比，属于核心PTQ贡献。实用性评4分：算法简洁、有kernel优化加速、校准成本低，但非均匀量化的kernel部署复杂度略高于标准均匀量化方案。

#### 摘要
提出统计无损量化的三层概念体系：task-lossless（零样本精度在采样方差内）、distribution-lossless（下一token分布与原模型不可区分，用 Expected Acceptance Rate (EAR) 衡量，如 EAR>=0.99 表示99%一致）和基于 gamma 平方方差定律的理论分析——证明对称量化相比非对称量化将噪声方差放大 gamma 平方倍，因此 distribution-lossless 必须用 asymmetric quantization。基于此提出 SLQ，一种采用 asymmetric quantization + 宽 bitwidth 搜索的 layer-wise 非均匀量化方法。实验表明 SLQ 可在低至 3.3 bits 实现 task-lossless，在 5-6 bits 平均实现 distribution-lossless，并通过优化 kernel 相对 FP16 获得 1.7-3.6x 推理加速。

- 💡 提出 EAR 指标量化 distribution-lossless：EAR>=0.99 即 99% token 一致
- 🎯 证明 gamma² 方差定律：对称量化噪声方差是非对称的 gamma² 倍
- 📊 task-lossless 低至 3.3 bits，distribution-lossless 平均 5-6 bits
- 📊 相比 FP16 实现 1.7-3.6x 推理加速

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · WindowQuant针对视频语言模型（VLM）提出窗口级混合精度KV cache量化方法，属于kv_cache压缩范畴。方法通过窗口级相似度快速确定最优bit-width配置，并通过重排序解决混合精度计算的硬件低效问题，思路清晰有部署潜力。但abstract未明确最大测试模型规模及具体benchmark名称，评估数据不够透明；目标域为多模态VLM而非纯LLM，适用范围有限。整体算法简洁、有硬件效率考量，实用性中等。

#### 摘要
针对 VLM 中 visual token 序列过长导致 KV cache 显存和延迟过大的问题，提出基于 window 粒度的混合精度 KV cache 量化方法 WindowQuant。包含两个模块：window-level quantization search 通过计算 visual token window 与 text prompt 的相似度分数，快速确定每个 KV cache window 的最优 bit-width，避免 token 级搜索的高开销；window-level KV cache computation 在量化前对 KV cache window 重排序，缓解混合精度在推理时的硬件低效问题。实验表明在多个数据集上优于 SOTA VLM 和 KV cache 量化方法。

- 🎯 以 window 为粒度做 KV cache 混合精度量化
- 💡 用 visual-text 相似度替代逐 token 搜索 bit-width
- 🔧 量化前 window 重排，提升混合精度硬件效率
