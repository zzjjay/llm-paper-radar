#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型压缩工作，采用 NVFP4 量化和线性注意力实现高效长视频生成。优势：推理加速明显（2.4x），部署友好（RTX 5090）。劣势：无定量精度评估（如 FID、LPIPS）。属于扩散压缩但缺乏标准基准评估。

#### 摘要
SANA-Video 是一个面向高效视频生成的小型 diffusion 模型，支持最高 720x1280 分辨率和分钟级时长。核心设计包含两点：以 linear attention 替代 vanilla attention 构建 Linear DiT，显著降低大 token 数场景下的计算开销；以及基于 linear attention 累积特性设计 constant-memory KV cache 的 block-wise autoregressive 方案，以固定显存开销实现全局上下文感知。训练仅需 64 张 H100 GPU 跑 12 天（约为 MovieGen 的 1%），在 latency 上比同量级模型（Wan 2.1-1.3B、SkyReel-V2-1.3B）快 16x，部署于 RTX 5090 GPU 并采用 NVFP4 精度推理可将 5 秒 720p 视频生成时间从 71s 压缩至 29s。

- 🎯 用 linear attention 构建 Linear DiT，配合 constant-memory KV cache 实现 block-wise autoregressive 长视频生成
- 📊 latency 比 Wan 2.1-1.3B 和 SkyReel-V2-1.3B 快 16x，性能与其竞争
- 📊 RTX 5090 + NVFP4 推理 5s 720p 视频：71s→29s，2.4x 加速
- 🔧 训练成本仅为 MovieGen 的 1%（64 H100 GPU，12 天）
