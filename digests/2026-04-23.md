- 🎯 方法：将训练好的E-expert MoE通过expert duplication + router extension扩展为mE-expert，保持per-token计算量不变
- 💡 创新：utility-based expert selection利用梯度重要性分数指导非均匀复制，CPT受限时gap closure提升超3倍
- 📊 结果：7B–13B参数规模下，upcycled模型匹配固定规模baseline的validation loss，节省32% GPU hours
- 🔧 工程：理论框架将quality gap分解为capacity term与initialization term，提供跨模型规模与训练预算的实用配方

#### 摘要
SANA-Video 是一个面向高效视频生成的小型扩散模型，支持最高 720x1280 分辨率和分钟级时长。核心设计包括两点：用 linear attention 替代 vanilla attention 以降低大量 token 处理的计算开销，以及为 block-wise autoregressive 长视频生成设计基于 linear attention 累积性质的 constant-memory KV cache，以固定显存代价提供全局上下文。训练仅需 64 张 H100 GPU 跑 12 天（约为 MovieGen 的 1%），在与 Wan 2.1-1.3B、SkyReel-V2-1.3B 等同量级模型保持竞争力的同时实现 16x 延迟加速，并在 RTX 5090 以 NVFP4 精度将 5 秒 720p 视频的生成时间从 71s 压缩到 29s。

- 🎯 用 linear attention 构建 Linear DiT，并设计 constant-memory KV cache 支持分钟级长视频生成
- 📊 与 Wan 2.1-1.3B / SkyReel-V2-1.3B 同量级模型相比，延迟降低 16x
- 📊 RTX 5090 + NVFP4 推理 5 秒 720p 视频从 71s 降至 29s（2.4x speedup）
