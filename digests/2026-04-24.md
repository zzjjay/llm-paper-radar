**投机解码** · topic 4/5 · practicality 3/5 · composite 7/10 · 投机解码框架与块扩散并行草稿生成结合，展现显著推理加速。然而：(1) 无报告的精度基准测试（MMLU/GSM8K等），"无损"说法未量化验证；(2) 具体模型尺寸/校准成本未披露；(3) 算法复杂性中等（扩散 drafter 需定制）。作为投机解码与压缩相关工作得分4，但缺乏量化精度评估和完整部署细节限制实用性至3。

#### 摘要

- 🎯 用 block diffusion model 替代 autoregressive draft model，单次 forward pass 并行生成 draft tokens
- 💡 draft model 以 target model 提取的 context features 为条件，提升 draft 质量与 acceptance rate
- 📊 跨模型和任务实现 >6x lossless 加速，比 EAGLE-3 高出最多 2.5x speedup

**知识蒸馏** · topic 4/5 · practicality 3/5 · composite 7/10 · 核心知识蒸馏工作，设计统一框架并提出混合KL方法。但缺点：（1）未明确报告最大模型规模；（2）仅报告性能改进的定性描述，未提供MMLU等标准基准的定量精度数值；（3）未提供端到端推理加速数据；（4）在线采样成本与实际校准时间不清楚。这些缺陷降低了学术价值评分和可部署性评分。

#### 摘要

- 💡 将现有 KD 方法统一重构为 token 级 reweighted log-likelihood 目标，揭示各方法间内在联系
- 🎯 HPD 融合 forward KL 与 reverse KL，结合 off-policy 数据与近似 on-policy sampling
- 📊 在数学推理、对话、代码等多类任务及多种模型规模上均提升性能与训练稳定性
- 🔧 近似 on-policy sampling 保持计算效率，避免完整 on-policy rollout 的高开销

**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型压缩工作，通过线性注意力和常数内存KV cache设计实现高效长视频生成。NVFP4量化部署在消费级GPU上并展现显著推理加速。虽无标准扩散评估指标（FID/IS），但相对于同规模模型的延迟改进（16倍）和量化加速（2.4倍）明确可衡量。算法简洁（线性注意力 + 块化KV设计）、推理性能清晰、内存高效，具有实用

#### 摘要
SANA-Video 是一个小型扩散模型，目标是在 RTX 5090 GPU 上高效生成最高 720x1280 分辨率、分钟级时长的视频。核心设计有两点：用 linear attention 替代 vanilla attention 以降低大量 token 处理的计算开销；以及基于 linear attention 累积性质设计 constant-memory KV cache 的 block-wise autoregressive 生成方案，以固定显存代价提供全局上下文。训练仅需 64 张 H100 GPU 运行 12 天（约为 MovieGen 成本的 1%），在与 Wan 2.1-1.3B、SkyReel-V2-1.3B 等同量级模型性能相当的前提下，实测延迟快 16x；部署于 RTX 5090 时以 NVFP4 精度将 5 秒 720p 视频生成从 71s 压缩至 29s（2.4x 加速）。

- 🎯 用 linear attention 构建 Block Linear DiT，constant-memory KV cache 以固定显存支持分钟级视频生成
- 📊 与 Wan 2.1-1.3B / SkyReel-V2-1.3B 性能相当，实测延迟快 16x
- 📊 RTX 5090 上以 NVFP4 推理，5 秒 720p 视频生成从 71s 降至 29s（2.4x 加速）
- 🔧 训练成本仅 64 H100 × 12 天，约为 MovieGen 的 1%
