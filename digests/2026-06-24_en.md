#### Summary
Existing Post-Training Sparsity (PTS) methods are confined to single-sparsity optimization, requiring a separate, costly optimization run for each target sparsity level, which severely limits flexible deployment. EPTS introduces an elastic multi-sparsity framework that produces a single model supporting diverse sparsity configurations via one-shot optimization. Two core mechanisms are proposed: MS-HiLoRA enables knowledge inheritance from low- to high-sparsity groups to reduce parameter reconstruction conflicts, and MSFM dynamically fuses feature representations across sparsity granularities to improve robustness to pruning perturbations. Experiments on LLaMA and OPT families show EPTS achieves competitive performance against state-of-the-art methods SparseGPT and Wanda, while substantially reducing optimization overhead for multi-scenario deployment.

#### Summary
Post-training quantization (PTQ) of reasoning models cannot be fully evaluated by final-answer accuracy or per-token latency alone. INT4/INT3 quantization preserves accuracy but causes quantized models to generate longer chains of thought, introducing a hidden test-time compute cost in real serving. The paper introduces the CoT Token Inflation Ratio to systematically measure this effect across mathematical reasoning, code generation, scientific QA, and agentic tool-use benchmarks, finding that token inflation is accompanied by more intermediate steps and greater semantic repetition, translating into measurable end-to-end serving penalties. Three mitigation strategies are evaluated: prompting and decoding-time sampling yield inconsistent accuracy-length trade-offs, while quantization-aware training (QAT) shows more promise in reducing both accuracy degradation and token inflation.

- 🎯 Introduces CoT Token Inflation Ratio to measure reasoning-length growth in quantized models across multiple benchmarks
- 📊 INT4/INT3 PTQ preserves final-answer accuracy but inflates reasoning-token usage, offsetting the per-token speedup
- 💡 Token inflation correlates with more intermediate steps and greater semantic repetition, causing measurable end-to-end serving latency penalties
- 🔧 Prompting and decoding-time sampling show inconsistent trade-offs; QAT is more effective at reducing both accuracy loss and token inflation
