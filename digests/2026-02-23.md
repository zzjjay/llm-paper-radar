#### 🧭 为什么推送这篇
**PTQ（训练后量化）** · topic 4/5 · practicality 4/5 · composite 8/10 · R&Q是针对SMoE推理阶段负载不均衡问题的training-free方案，通过复制heavy-hitter专家并对低重要性专家及副本进行量化来保持内存预算。方法简洁，无需修改路由器或重训练，仅需小型校准集，具有良好的部署可行性。量化是方案的核心组成部分（用于offset复制带来的内存开销），因此归入ptq桶。评估覆盖多个SMoE模型和基准，精度保持良好，推理负载均衡显著改善。主要不确定性在于最大测试模型规模未在摘要中明确说明，以及端到端吞吐量提升数据不明确，略微拉低了实用性评分。

#### 摘要
针对 SMoE LLM 推理阶段专家负载严重不均衡的问题，作者通过路由行为分析发现：负载不均随 batch size 增大而加剧、选择频率不能可靠反映专家重要性、且整体专家负载与重要性可用小规模 calibration set 估计。基于此提出 Replicate-and-Quantize (R&Q)，对 heavy-hitter 专家做副本复制以提升并行容量，对低重要性专家及副本进行量化以保持原显存预算，无需重训或修改 router。同时定义 Load-Imbalance Score (LIS) 衡量路由偏斜。实验显示负载不均最高降低 1.4x，精度波动控制在 ±0.6% 以内。

- 💡 推理时负载不均随 batch size 增大而加剧

- 📊 imbalance 降低 1.4x，精度波动 ±0.6%
- 🔧 training-free，不改 router，即插即用
