#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 核心是扩散模型压缩，通过学习的自适应分块机制在推理时动态压缩token序列。FID/Inception Score明确改进，FLOP降低4-16×。主要优点：(1)算法简洁（编码器-路由-解码器scaffold），(2)推理FLOP显著降低，(3)可从预训练检查点快速蒸馏（训练步减8×），(4)内存占用未报告但toke

#### 摘要
固定长度 patch token 序列让 Diffusion Transformer 在低信息区域浪费等量计算。DC-DiT 在 DiT backbone 上引入可学习的 encoder-router-decoder scaffold，通过端到端训练的 chunking 机制将 2D 输入自适应压缩为更短的 token 序列：背景区域合并为少量 token，细节丰富区域保留更多 token，且在 diffusion 早期（噪声大）使用更少 token、后期（细节涌现）使用更多 token。在 ImageNet 256×256 class-conditional 生成上，DC-DiT 在 4× 和 16× 压缩比下均优于参数匹配和 FLOP 匹配的 DiT baseline（FID 和 Inception Score 均有提升），且可从预训练 DiT checkpoint upcycle，训练步数最多减少 8×。

- 🎯 方法：端到端学习 chunking 机制，按图像内容和 diffusion timestep 动态分配 token 数量
- 📊 结果：4× 和 16× 压缩比下 FID/IS 均超越参数匹配与 FLOP 匹配的 DiT baseline
- 💡 创新：无需显式监督，自动涌现有意义的视觉语义分割；跨 timestep 自适应压缩率
- 🔧 工程：可从预训练 DiT checkpoint upcycle，最多节省 8× post-training 训练步数
