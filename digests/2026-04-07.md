**KV cache 压缩** · topic 3/5 · practicality 2/5 · composite 5/10 · 该论文提出一种partial-KV解码框架，通过残差质量估计将精确检索token与未检索prefill token的softmax贡献分离合并，属于KV cache压缩范畴。然而该方法存在明显局限：1) 使用exhaustive Top-K作为oracle选择器，明确声明不可部署为实际检索系统，缺乏实际可用的token选择机制；2) 需要学习正特征映射φ，增加训练复杂度；3) 最大测试模型仅为3B（Llama-3.2-3B），偏小；4) 无端到端延迟或内存节省数据，仅有准确率对比。整体实用性较低，但作为KV cache压缩的理论/分析工作有一定参考价值，topic relevance为3分。

#### 摘要
针对长上下文 partial-KV decoding 场景，在仅保留 sink/tail 锚点和检索到的 token 子集做精确 softmax 计算时，提出一种残差质量记账规则：用学习到的正特征映射 φ 构建固定大小的 summary state (S,u) 表示未检索 token 的残差贡献，并显式减去已检索 token 的特征以保证精确集与残差集不重叠，最后在单一归一化下融合两个分支的分子分母。该方法不改动 backbone LM 和 exact branch KV 张量，且使用 exhaustive Top-K 仅作 oracle selector。在 1% exact-support 预算下，于 frozen Llama-3.2-Instruct 1B/3B 上 RULER 和 BABILong 全部 context length 均优于 selection-only Top-K baseline；0.5-4% 预算扫描中趋势基本保持；LongBench summarization 多数占优，multi-doc QA 表现不一。诊断显示主要剩余误差来自学习 φ 对未检索残差质量的近似不足。

- 💡 用 (S,u) summary state + 检索 token 特征减除做残差记账
- 📊 1% exact-support 下 RULER/BABILong 全长度超 Top-K baseline
- 🎯 backbone 与 exact-branch KV 不变，仅改 softmax 归一化
- ⚠️ 主要误差源于学习 φ 对残差质量的近似不准
