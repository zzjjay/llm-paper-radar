#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 4/5 · composite 9/10 · 核心KV缓存压缩方法，利用pre-RoPE空间的Q/K向量浓聚特性实现高效关键位置估计。在AIME25长序列推理(32K token)上保持满注意力精度同时实现2.5x吞吐量提升和10.7x显存降低，相比baseline方法显著超越。算法简洁(三角级数距离加权)，推理速度提升明显，内存收益巨大，无需昂贵校准，实用性强。

#### 摘要
Extended reasoning 场景下 LLM 的 KV cache 内存瓶颈严重，现有方法用 post-RoPE query 的 attention score 估计 KV 重要性，因 query 随位置旋转导致代表性不足、top-key 选取不准。TriAttention 转向 pre-RoPE 空间，发现 Q/K 向量在非零中心附近高度集中且位置稳定（Q/K concentration），并通过三角级数分析揭示这种集中导致 query 对特定距离的 key 产生偏好，中心向量决定距离偏好模式。基于此，TriAttention 用这些中心向量对 key 按位置打分，同时结合 Q/K norm 作为补充信号估计重要性。在 AIME25 的 32K-token 生成任务上，TriAttention 保持与 Full Attention 相当的推理精度，同时实现 2.5x 吞吐提升或 10.7x KV 内存压缩，而主流 baseline 在同等效率下精度仅约一半。

- 💡 post-RoPE query 因位置旋转难以代表性选 key，转而在 pre-RoPE 空间利用 Q/K concentration 建模重要性
- 🎯 用 pre-RoPE 中心向量构造三角级数，按 key 位置打分，并结合 Q/K norm 作双信号估计
- 📊 AIME25 上实现 Full Attention 同等精度下 2.5x 吞吐提升或 10.7x KV 内存压缩，baseline 同效率下精度降至约一半
- 🔧 支持 OpenClaw 在单张消费级 GPU 上完成原本 Full Attention OOM 的长上下文部署

#### 摘要
SANA-Video 是一个小型扩散模型，目标是高效生成 720x1280 分辨率、分钟级时长的视频。核心设计有两点：用 linear attention 替代 vanilla attention 以处理视频 token 数量大的问题（Linear DiT），以及为 Block Linear Attention 设计基于 linear attention 累积性质的 constant-memory KV cache，以固定显存代价提供全局上下文，支持分钟级长视频生成。训练成本仅为 64 张 H100 GPU 上 12 天（MovieGen 的约 1%），在对比 Wan 2.1-1.3B 和 SkyReel-V2-1.3B 等同规模模型时实测延迟快 16x，并可在 RTX 5090 上以 NVFP4 精度将 5 秒 720p 视频生成时间从 71s 压缩到 29s（2.4x 加速）。

- 🎯 linear attention 替代 vanilla attention，设计 constant-memory KV cache，以固定显存实现分钟级长视频生成
- 📊 与同规模 SOTA 模型（Wan 2.1-1.3B、SkyReel-V2-1.3B）相比，实测延迟快 16x
