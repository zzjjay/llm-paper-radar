#### 摘要
针对长推理场景下 KV cache 内存瓶颈，作者指出现有基于 post-RoPE attention score 估计 KV 重要性的方法因 query 随位置旋转而仅有少量代表性 query、导致 top-key 选择不稳。论文转向 pre-RoPE 空间，发现 Q/K 向量在固定非零中心附近高度集中且跨位置稳定（Q/K concentration），并证明该集中性通过三角级数决定 query 对不同距离 key 的偏好。基于此，TriAttention 用这些中心通过三角级数按位置打分，并结合 Q/K norm 作为辅助信号。在 AIME25 32K-token 生成上达到 Full Attention 同等精度，吞吐提升 2.5x 或 KV 内存降低 10.7x，主流 baseline 在同等效率下精度仅约一半。

- 💡 利用 pre-RoPE 空间 Q/K concentration 估计 key 重要性
- 📊 AIME25 32K 生成下匹配 Full Attention 精度
- 📊 吞吐 2.5x 或 KV 内存压缩 10.7x，baseline 同效率精度减半
- 🔧 让 OpenClaw 可在单张消费级 GPU 上长上下文部署
