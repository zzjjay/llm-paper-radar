#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 3/5 · practicality 4/5 · composite 7/10 · CONF-KV提出基于next-token分布置信度的动态KV cache eviction策略，结合FP16/INT8混合精度存储和pyramidal per-layer budget，属于kv_cache压缩核心方向。方法本身部署简单——置信度计算是每步decoding的副产品，eviction逻辑为attention mass+recency排名，无需训练或校准。但评测基准偏弱：Needle-in-a-Haystack是retrieval型合成任务，VisualWebArena是多模态agent任务，均非MMLU/GSM8K/HellaSwag等标准LLM下游benchmark；PPL也只是perplexity指标。按规则，未报告任何标准下游benchmark时topic_relevance上限为3。实用性方面，算法简单（信号在线获取）、低校准成本、有明确内存节省数据，practicality=4。总体是一篇实用性强但验证范围偏窄的KV cache压缩工作。

#### 摘要
针对长上下文 LLM 推理中 KV cache 占用过大的问题，CONF-KV 利用每步 decoding 已有的 next-token 分布计算 confidence 标量，动态调整每步 cache 预算——不确定时多保留 context，自信时激进剪枝。预算内 token 按累计 attention mass 与 recency 的组合排序，并保留 protected recent window，配合 blockwise online-softmax attention、FP16/INT8 混合精度存储和 pyramidal per-layer budget。在 4 个模型族、生成长度 4K 下，footprint 接近 512-token sliding window，PPL 仅比 full KV 高 1.5--2.1；32K Needle-in-a-Haystack 检索准确率 91.4%（vs sliding window 53.8%、H2O 80.6%），75 个 VisualWebArena 任务保留 full-KV 95.3% 成功率，峰值显存降低 2.8 倍。

- 💡 用 next-token 分布的 confidence 动态决定每步 KV 预算
- 📊 32K NIAH 准确率 91.4%，远超 H2O 80.6% 与 sliding 53.8%
- 📊 VisualWebArena 保留 95.3% 成功率，峰值显存降 2.8×
- 🔧 结合 FP16/INT8 混合存储与 pyramidal per-layer 预算
