### 1. CONF-KV: Confidence-Aware KV Cache Eviction with Mixed-Precision Storage for Long-Horizon LLM (7/10) 🔁

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 3/5 · practicality 4/5 · composite 7/10 · CONF-KV提出了一种基于模型置信度的动态KV cache驱逐策略，结合FP16/INT8混合精度存储和金字塔式per-layer预算，属于kv_cache压缩核心工作。方法本身相对简洁，推理时在线计算next-token distribution的confidence score，无需离线校准，实用性较强。但评估基准较弱：主要使用PPL（WikiText等）、Needle-in-a-Haystack和VisualWebArena，均不属于MMLU/GSM8K/HumanEval等主流下游基准，NIAH本身对KV压缩质量的预测能力有限，VisualWebArena是多模态任务。根据规则，缺乏主流下游基准时topic_relevance上限为3。practicality方面：算法简单（online softmax + 置信度分数）、data-free、内存收益明显，但对推理引擎有一定集成要求。整体是值得跟踪的kv_cache工作。

#### 🌊 Paper River
[倒读批判链](../paper-river/Conf-KV-2605.24786.org)

#### 摘要
针对长序列 LLM 推理中 KV cache 主导 GPU 内存的问题，CONF-KV 利用解码时 next-token 分布的 confidence score 动态调整每步 cache 预算：模型不确定时保留更多 context，确定时激进剪枝。预算内 token 按累计 attention mass 与 recency 组合排序，并设保护性 recent window；同时结合 blockwise online-softmax attention、FP16/INT8 混合存储和金字塔式 per-layer 预算。在四个模型家族、最长 4K 生成上，内存接近 512-token sliding window，PPL 仅比 full KV 高 1.5–2.1；32K NIAH 检索准确率 91.4%（sliding 53.8%、H2O 80.6%），VisualWebArena 75 任务以 2.8× 更低峰值内存保留 95.3% full-KV 成功率。

- 💡 用 next-token 分布的 confidence score 动态决定每步 KV 预算
- 🔧 结合 FP16/INT8 混合存储与金字塔式 per-layer 预算
- 📊 32K NIAH 达 91.4%，远超 sliding 53.8% 与 H2O 80.6%
- 📊 VisualWebArena 保留 95.3% full-KV 成功率，峰值内存降 2.8×
