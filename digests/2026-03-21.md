#### 摘要
针对 KV cache 内存随上下文线性增长的问题，作者将 token eviction 重新理解为粗粒度的维度分配（每个 token 要么 0 维要么全维），并提出 MixedDimKV，在 token 级别按更细的粒度分配维度数量。进一步的 MixedDimKV-H 融合 head-level importance 信息。在 LongBench 上仅用 6.25% 的 KV cache 即可逼近 full attention；Needle-in-a-Haystack 测试中在 50K 上下文下仅用 0.26% 的 cache 仍保持 100% 准确率。
