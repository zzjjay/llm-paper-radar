### 2. Image-Free Timestep Distillation via Continuous-Time Consistency with Trajectory-Sampled Pairs (8/10)
**hf_daily** · `2511.20410` · 2025-11-25
👥 Bao Tang, Shuai Zhang, Yueting Zhu... · 🏷 cs.CL
🔗 [arXiv](https://arxiv.org/abs/2511.20410) · [PDF](https://arxiv.org/pdf/2511.20410.pdf)
📡 Sources: hf_daily (👍 3, 💬 2)
🧪 diffusion_compression · Trajectory-Backward Consistency Model (TBCM), continuous-time consistency distillation · cal: 无外部数据依赖，相比 Sana-Sprint 降低约 40% 训练时间 · perf: 一步生成，无端到端延迟数据

#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · 扩散模型步数蒸馏，移除数据依赖提升效率；FID/CLIP 指标评估；缺少最大模型规模、端到端性能数据；数据无关蒸馏的简洁性和内存节省支撑实用性但信息不完整，故评分 4/4。

#### 摘要
当前连续时间 consistency distillation 方法依赖大规模训练数据和计算资源，限制了其在资源受限场景的部署。TBCM（Trajectory-Backward Consistency Model）通过直接从 teacher model 的生成 trajectory 中提取 latent representations，完全消除对外部训练数据的依赖，无需 VAE encoding 和大规模数据集。trajectory 提取的样本天然弥合了训练与推理的分布差距，从而实现更有效的知识迁移。在 MJHQ-30k 单步生成任务上，TBCM 达到 6.52 FID 和 28.08 CLIP score，训练时间相比 Sana-Sprint 减少约 40%，同时节省大量 GPU memory。

- 💡 image-free 蒸馏范式：从 teacher model 生成 trajectory 直接提取样本，无需外部数据集或 VAE encoding
- 📊 单步生成在 MJHQ-30k 上达到 6.52 FID / 28.08 CLIP score
- 🎯 训练时间比 Sana-Sprint 减少约 40%，并显著节省 GPU memory
- 💡 揭示连续时间 consistency distillation 中的 diffusion-generation space discrepancy，分析 sampling 策略对蒸馏性能的影响

#### 📎 相关方法 / 对比基线
- Sana-Sprint — 训练时间减少约 40%，GPU memory 消耗更低
- [CM](https://arxiv.org/abs/2303.01469) — 基于 Consistency Model trajectory 框架扩展，去除外部数据依赖
