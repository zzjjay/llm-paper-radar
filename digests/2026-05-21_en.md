**KV cache compression** · topic 3/5 · practicality 4/5 · composite 7/10 · OCTOPUS提出了一种基于八面体参数化的KV cache量化方法，对旋转坐标三元组联合量化，使用Lloyd-Max标量量化器，无需校准数据，有fused Triton实现。方法简洁，data-oblivious且online，具有较强工程可行性。然而，论文未报告任何标准LLM下游benchmark（MMLU、GSM8K、HellaSwag等），仅在PPL/metric层面比较rotation codec，且最大测试模型规模未知。按规则，无recognized downstream benchmark时topic_relevance上限为3。practicality较好：data-free、fused Triton kernel、无额外latency，得4分。整体值得追踪但评估深度有限。

#### Summary
OCTOPUS targets the KV cache memory and bandwidth bottleneck in long-context autoregressive inference. Building on rotation-preconditioned codecs, it jointly quantizes triplets of rotated coordinates by mapping each triplet's direction onto a square via an octahedral parameterization, then Lloyd-Max quantizing the two angular coordinates and the triplet norm against implementation-matched marginals. Minimizing per-triplet squared error yields a non-uniform bit allocation that depends only on key dimensionality and is empirically constant across tested decoders. Across text, video, and audio, OCTOPUS matches or beats every prior rotation codec at every reported bit width, with a growing lead at extreme compression, while a fused Triton kernel reconstructs keys on the fly with no added decode-time bandwidth or latency.

#### Summary
Long-context decoding is bottlenecked by KV-cache memory traffic, and existing sparse decoders target softmax attention whose dense tails make any truncation lossy. EntmaxKV reframes sparse decoding for α-entmax attention as support recovery: if selected candidates cover the entmax support, decoding is exact. It combines query-aware page scoring, support-aware candidate selection, sparse entmax attention, and a Gaussian-aware selector that estimates the entmax threshold from lightweight page statistics. Output error is bounded by the dropped probability mass δ and vanishes upon support recovery. At 1M context, it matches full-cache entmax with a small fraction of the KV cache and delivers up to 3.36× (softmax) and 5.43× (entmax) speedup over full attention.

- 💡 Exploits α-entmax exact zeros, recasting sparse decoding as support recovery
- 🎯 Query-aware page scoring with Gaussian-based entmax threshold estimation
- 📊 Up to 3.36× (softmax) / 5.43× (entmax) speedup at 1M context
- 📊 Lower dropped mass and output error than softmax sparse decoding at matched KV budget

#### Summary
Addresses the lack of runtime failure detection in KV cache quantization with a tiered architecture: INT8 keys and INT4 values on GPU, FP16 originals retained in system RAM for deterministic fallback. A two-term error decomposition produces per-head, per-step online bounds on attention distribution distortion (from key quantization) and value reconstruction error, driving adaptive precision selection and a multi-stage fallback ladder that recovers exact dense attention when required. On LLaMA 3.1-8B with up to 128K context across PG-19, NIAH, and RULER, the system matches FP16 KV quality within noise and repairs catastrophic failures of naive INT8/INT4 baselines. Short-context value-sensitive tasks show a controlled compression–fidelity trade-off, removable via tighter value tolerances or FP16-value fallback.

- 📊 LLaMA 3.1-8B at 128K matches FP16 on PG-19/NIAH/RULER within noise
- ⚠️ Certification is local per-head/per-step, not end-to-end model correctness
