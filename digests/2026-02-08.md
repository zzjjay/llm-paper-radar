#### 🧭 为什么推送这篇
**Pruning / 蒸馏** · topic 4/5 · practicality 2/5 · composite 6/10 · 该论文将LLM层剪枝建模为合作博弈，用Shapley值估计每层的边际贡献，并引入轻量surrogate网络替代直接计算。思路新颖，能捕捉层间依赖关系。但方法引入了额外的surrogate网络训练和Monte Carlo采样，整体calibration流程复杂度较高，难以简单叠加到GPTQ/AWQ等现有pipeline上；最大测试模型规模未明确，zero-shot评估具体benchmarks不详，落地工程路径不够清晰，实用性有限。

#### 摘要
针对 layer-wise pruning 中静态启发式规则忽视层间依赖的问题，将层剪枝建模为合作博弈：每层为 player，模型性能为 utility，用 Shapley value 衡量层贡献。由于 LLM 上精确计算 Shapley 不可行，训练一个轻量 surrogate network 来预测任意层组合下的 LLM 性能，并结合 stratified Monte Carlo mask sampling 降低估计成本。在 perplexity 和 zero-shot accuracy 上一致优于现有 layer pruning 方法。

- 💡 把 layer pruning 形式化为 cooperative game 求 Shapley value
- 🎯 用 surrogate network 预测任意层组合下的 LLM 性能
- 🔧 stratified Monte Carlo mask sampling 降低 Shapley 估计成本
- 📊 perplexity 与 zero-shot accuracy 均超过 layer pruning baselines
