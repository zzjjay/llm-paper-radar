📡 Sources: arxiv
🧪 kv_cache · KV cache eviction, learned importance scoring · cal: 1.7M参数scorer需训练，但结论表明无需此开销 · perf: 无end-to-end性能数据，结论偏向简单启发式方法

#### 🧭 为什么推送这篇
**Survey / 方法论与对比** · topic 4/5 · practicality 4/5 · composite 8/10 · 本文不提出新的压缩算法，而是通过系统实验评估learned KV cache importance scoring的上限，属于`survey`桶的测量/方法论工作。核心发现具有直接的压缩工程价值：(1) position-based启发式（保留前4+最后N个token）与复杂learned scorer持平甚至更优；(2) prefill attention信号已足够，无需专门的1.7M参数scorer；(3) KV表示中超出位置和prefill attention的边际信息极为有限。这些结论对工程师选择KV cache压缩方案有直接指导意义，建议放弃昂贵的learned scorer而采用简单启发式。不过，论文未报告标准LLM基准（MMLU、GSM8K等），最大测试模型规模不明，略微削弱其普适性，但作为负面结果的方法论研究仍有较高价值。

#### 摘要
本文研究了基于学习的 KV cache 压缩方法，构建了 Speculative Importance Prediction (SIP)——一个 1.7M 参数的 non-query-aware 重要性打分器，仅从 KV 表示预测 token 重要性，并引入了 multi-horizon lookahead 与 cross-attention 等机制。在 5 个 seed、4 个保留率、3 个任务的实验下，SIP 未能超过包括随机选择在内的简单 baseline。位置启发式（保留前 4 + 后 N 个 token）与 prefill attention 信号已能达到甚至超过复杂学习打分器的效果，说明 KV 表示中除位置和 prefill attention 之外可用于重要性预测的边际信息有限。作者推测未来 query 与生成轨迹的循环依赖是学习困难的根源。

- 📊 1.7M 参数 SIP 在 5 seeds×4 保留率×3 任务下不敌随机选择
- 💡 保留前 4 + 后 N token 的位置启发式即可匹敌学习方法
- 🎯 prefill attention 提供等价于复杂学习打分器的信号
- ⚠️ 未来 query 与生成轨迹的循环依赖阻碍学习可行性
