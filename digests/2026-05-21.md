#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 4/5 · composite 9/10 · OCTOPUS提出了一种针对KV cache的新型量化方法，通过将旋转后的坐标三元组联合量化（octahedral参数化+Lloyd-Max量化），在bit效率上严格优于TurboQuant/PolarQuant等前沿rotation codec。方法data-oblivious、在线、确定性，且有fused Triton实现支持无额外延迟的on-the-fly解码，部署路径清晰。缺点是abstract未明确最大测试模型规模及具体benchmark名称，实际LLM规模和下游任务精度尚需从正文确认，略降practicality评分。总体而言，这是一个算法简洁、无需标定数据、有kernel实现支撑的KV cache压缩方案，对团队高度相关。

#### 摘要
OCTOPUS 针对长上下文推理中 KV cache 的带宽与显存瓶颈，在 rotation-preconditioned 量化范式（TurboQuant、PolarQuant）基础上提出对旋转后坐标三元组的联合量化：通过八面体参数化将三元组方向映射到方形上，并对得到的两个坐标及三元组范数用 Lloyd-Max 量化器匹配各自的解析边缘分布。基于 squared error 最优化推出仅依赖 key 总维度的非均匀 bit allocation，得到的最优配置在所测的真实 decoder 上恒定。该 codec 数据无关、在线、给定 seed 后确定性，在 text、video、audio 上以所有 bit width 匹配或超过此前 rotation codec，bit 越低优势越大；fused Triton 实现在线重建 key 而不物化未压缩 key，相对现有 dequantization 不增加 decode 延迟。

- 💡 旋转后三元组联合量化 + 八面体参数化映射到方形
- 🎯 Lloyd-Max 匹配边缘分布，bit allocation 仅依 key 维度
- 📊 text/video/audio 上全 bit width 超越 rotation codec，极低 bit 优势更大
- 🔧 fused Triton kernel 在线重建 key，零额外 decode 带宽与延迟

#### 📎 相关方法 / 对比基线
- TurboQuant — 同属 rotation codec，被全 bit width 超越
- PolarQuant — 前序 rotation codec，被联合三元组量化超越
