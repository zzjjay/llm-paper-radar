### 3. OCTOPUS: Optimized KV Cache for Transformers via Octahedral Parametrization Under optimal Squared error quantization (9/10) 🔁

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 4/5 · composite 9/10 · OCTOPUS 是一个专门针对 KV cache 压缩的新型量化 codec，核心贡献是将旋转后的坐标三元组进行联合量化，通过八面体参数化将方向映射到正方形，并对两个坐标和三元组 norm 应用 Lloyd-Max 量化，实现严格非均匀 bit 分配。方法完全 data-oblivious（无需校准数据），online，且有 fused Triton 实现可 on-the-fly 重建 key 而无额外延迟，这对生产部署非常有利。与 TurboQuant、PolarQuant 等 rotation codec 基线相比，在所有 bit width 和 metric 上均优，且极低 bit 时优势更大。唯一扣分点是 abstract 未明确最大测试模型规模及具体 benchmark 名称，实际 speedup 数字也未量化，但 Triton kernel 的存在暗示有可信的部署路径。

#### 摘要
OCTOPUS针对长上下文推理中KV cache的内存瓶颈，延续rotation-preconditioned标量量化的思路（如TurboQuant、PolarQuant），但改为对rotated坐标三元组联合量化。每个triplet的方向通过octahedral参数化映射到正方形上得到两个坐标，加上triplet范数共三个量，按匹配的marginal分布做Lloyd-Max量化，并依据squared error最优推导出仅依赖key总维度的非均匀bit分配。该codec为data-oblivious、在线、给定seed确定性的方案，在文本、视频、音频任务上于所有bit宽度均匹配或超过既有rotation codec，且bit越低优势越大；融合的Triton kernel可在不实例化完整key的情况下dequant，不引入额外decode延迟。

- 💡 三元组方向经octahedral参数化映射到方形再Lloyd-Max量化
- 🎯 仅依赖key维度的非均匀bit allocation，data-oblivious
- 📊 在text/video/audio各bit宽度上均超越已有rotation codec
- 🔧 Triton fused kernel在线重建keys，无额外decode延迟

#### 📎 相关方法 / 对比基线
- TurboQuant — 同属rotation+标量量化范式，被超越
- PolarQuant — 前序rotation codec，每bit宽度被超越
