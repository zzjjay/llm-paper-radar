**KV cache 压缩** · topic 5/5 · practicality 5/5 · composite 10/10 · OScaR是针对极端KV缓存压缩（INT2量化）的轻量级、可部署方案。核心贡献是识别Token Norm Imbalance（TNI）瓶颈并通过Canalized Rotation + Omni-Token Scaling有效缓解，避免复杂的量化管道。提供了优化的CUDA内核实现和系统设计，在文本、多模态和全模态LLM上展示一致的改进。实现了3.0x解码加速、5.3x内存缩减和4.1x吞吐量提升，且代码已开源，具有强实用部署价值。虽然具体模型规模和基准精度数据未在摘要中明确，但KV缓存量化的核心地位和系统级优化设计使其具有高度的话题相关性和可行性。

#### 摘要
KV cache 的显存占用是长上下文和多模态 LLM 部署的主要瓶颈，现有 per-channel quantization 在极端压缩下（如 INT2）性能显著下降。本文通过理论和实验分析，将失效根因归结为 Token Norm Imbalance（TNI）：当同一 quantization 参数需要覆盖 norm 差异悬殊的 token 组时，误差被系统性放大。为此提出 OScaR（Omni-Scaled Canalized Rotation），依次应用 Canalized Rotation 和 Omni-Token Scaling 来消除序列维度的方差不均，并配套优化 CUDA kernel，适用于纯文本、多模态及全模态 LLM（X-LLMs）。相比 BF16 FlashDecoding-v2 基线，OScaR 在 INT2 下实现近无损精度，解码加速最高 3.0x，显存降低 5.3x，吞吐提升 4.1x。

- 🎯 方法：Canalized Rotation + Omni-Token Scaling 两步消除 KV cache 中的 Token Norm Imbalance
- 📊 结果：INT2 量化下近无损精度，解码最高加速 3.0x，显存减少 5.3x，吞吐提升 4.1x
- 💡 创新：从理论角度明确识别 TNI 为 per-channel quantization 在极端压缩下失效的根本原因
- 🔧 工程：适配 X-LLMs（文本、多模态、全模态），配套定制 CUDA kernel，无需复杂 pipeline

#### 📎 相关方法 / 对比基线

- FlashDecoding-v2 — BF16 解码基线，OScaR 相比其加速最高 3.0x

### 3. OCTOPUS: Optimized KV Cache for Transformers via Octahedral Parametrization Under optimal Squared error quantization (10/10) 🔁

👥 Mark Boss, Vikram Voleti, Simon Donné... · 🏷 cs.CL
🔗 [arXiv](https://arxiv.org/abs/2605.21226) · [alphaxiv](https://www.alphaxiv.org/abs/2605.21226) · [HF](https://huggingface.co/papers/2605.21226)

🧪 kv_cache · Octahedral parametrization + Lloyd-Max quantization for KV triplets · cal: 数据无关、在线、确定性(仅需种子)，无校准成本 · perf: Fused Triton实现无额外decode时间带宽或延迟；在线重构无需非压缩KV物化

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 5/5 · practicality 5/5 · composite 10/10 · 该工作是对KV cache压缩范例的有意义推进。贡献包括：(1)通过联合量化rotated坐标三元组引入octahedral参数化，使逐三元组误差最优化导致严格非均匀比特分配；(2)数据无关且在线的编码器，仅需种子；(3)fused Triton实现在decode时即时重构KV，零额外延迟。在文本、视频、音频上相比TurboQuant/PolarQuant等先前方法全面超越，极端压缩(低比特)优势明显。虽然抽象中未报告具体模型规模和MMLU/HumanEval数值，但该工作遵循rotation-preconditioned codec范式(已被验证为KV压缩近最优)，实现了严格的LSE优化，且具备立即可部署的高效kernel。算法简洁(octahedral映射+Lloyd-Max)、校准成本零、推理零开销、内存节省显著，完全符合部署标准。

#### 摘要
KV cache 在长上下文 autoregressive inference 中占主导内存带宽与占用。OCTOPUS 在 TurboQuant、PolarQuant 等 rotation-preconditioned codec 基础上，对旋转后坐标三元组（triplet）进行联合量化：将每个 triplet 的方向用八面体参数化映射到正方形，再对两个坐标及 triplet norm 分别做 Lloyd-Max 量化，并按最优 per-triplet squared error 推导出严格非均匀的比特分配。该 codec 无需校准数据，完全 data-oblivious、在线且确定性；配合 fused Triton 实现可在不物化未压缩 key 的情况下飞速重建，decode 时无额外带宽或延迟开销。在文本、视频、音频场景下，OCTOPUS 在所有测试 bit width 和指标上均持平或超越此前全部 rotation codec，低 bit 极端压缩场景优势更显著。

- 🎯 方法：对旋转后 key triplet 做八面体参数化 + Lloyd-Max 联合量化，推导出仅依赖总维度的非均匀比特分配
- 💡 创新：codec 完全 data-oblivious、无需 calibration 数据，给定 seed 即确定性运行
- 🔧 工程：fused Triton kernel 直接重建 key，不物化未压缩 key，decode 时零额外带宽与延迟
- 📊 结果：在文本、视频、音频全部 bit width 指标上匹配或超越所有先前 rotation codec，极端低 bit 场景领先更大

#### 📎 相关方法 / 对比基线
- TurboQuant — OCTOPUS 在其 rotation-preconditioned 范式上扩展，所有 bit width 下均超越
- PolarQuant — 同为 rotation codec 对比基线，OCTOPUS 在所有指标上持平或超越
