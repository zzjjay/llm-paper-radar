- 🎯 方法：Canalized Rotation + Omni-Token Scaling 组合，专门压制 TNI 在 sequence 维度引发的量化误差
- 📊 结果：相较 BF16 FlashDecoding-v2 基线，decoding 加速最高 3.0x，内存降低 5.3x，吞吐提升 4.1x
- 📊 结果：INT2 极端压缩下仍达到 near-lossless 性能，覆盖 text-only / multi-modal / omni-modal LLMs
- 💡 创新：理论与实验揭示 Token Norm Imbalance（TNI）是 per-channel quantization 在极端压缩下失效的根本原因

### 3. OCTOPUS: Optimized KV Cache for Transformers via Octahedral Parametrization Under optimal Squared error quantization (9/10)

**KV cache 压缩** · topic 5/5 · practicality 4/5 · composite 9/10 · 该论文提出了针对KV缓存的新型无旋转编码方案OCTOPUS，通过八面体参数化联合量化旋转坐标三元组，实现严格非均匀比特分配。方法简洁优雅（结构化随机旋转+Lloyd-Max量化），数据无关且在线适配。融合Triton内核可在线重建KV而无需物化未压缩张量，解决了实际部署中的带宽瓶颈。论文在文本、视频、音频跨域测试表现优于TurboQuant/PolarQuant等先前编解码方案，特别是极限压缩下优势明显。主要缺陷是摘要未报告MMLU/PPL等标准LLM基准的准确度保留情况，仅提及压缩指标对比；模型规模信息缺失。若论文包含完整的精度基准评估，topic_relevance应升至满分。

#### 摘要
LLM 长上下文推理中 KV cache 占用大量显存和内存带宽，现有 rotation-preconditioned 编解码方案（TurboQuant、PolarQuant）对单坐标标量量化。OCTOPUS 将旋转后的坐标三元组联合量化：用八面体参数化将三元组方向映射到正方形平面，对两个方向坐标和三元组范数分别做 Lloyd-Max 量化，并按最小化 per-triplet 均方误差的方式分配比特数，所得最优比特分配仅依赖 key 的总维度。该方案无需校准数据、在线确定性运行，在文本、视频、音频三类任务上于所有测试 bit width 均匹配或超越已有 rotation codec，极低比特时优势更显著；配套 fused Triton 实现可在不物化未压缩 key 的前提下 on-the-fly 重建，无额外解码带宽或延迟开销。

- 🎯 方法：对旋转 KV cache 的坐标三元组做联合量化，八面体参数化方向 + Lloyd-Max 量化范数与方向坐标。
- 💡 创新：per-triplet 最优比特分配仅依赖 key 总维度，在所测真实 decoder 上实验发现该最优值为常数。
- 📊 结果：文本、视频、音频多任务上全面匹配或超越 TurboQuant、PolarQuant，低 bit width 下优势更大。
- 🔧 工程：fused Triton kernel 实现 on-the-fly key 重建，零额外解码带宽与延迟。

- PolarQuant — 被 OCTOPUS 在所有 bit width 和指标上匹配或超越
