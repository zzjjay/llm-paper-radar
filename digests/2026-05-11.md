### 1. Shallow Prefill, Deep Decoding: Efficient Long-Context Inference via Layer-Asymmetric KV Visibility (8/10) 🔁
**hf_daily** · `2605.06105` · 2026-05-07
👥 Jungsuk Oh, Hyeseo Jeon, Hyunjune Ji... · 🏷 cs.CL
🔗 [arXiv](https://arxiv.org/abs/2605.06105) · [PDF](https://arxiv.org/pdf/2605.06105.pdf)
📡 Sources: hf_daily (👍 3, 💬 2)
🧪 kv_cache · layer-asymmetric KV visibility, shallow prefill deep decode · Llama-3.1-8B · cal: data-free, inference-time structural change only · perf: 33% TTFT improvement, 22% TPOT improvement at 128K context

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · KV缓存压缩的核心工作：通过非对称层可见性策略（Prefill仅在低层缓存，Decode全深度）在长上下文推理中大幅降低成本。简洁的算法设计、明确的推理性能提升（TTFT/TPOT）、内存减少量化（25%），以及OLMES基准保持精度（<0.4pt drop）。缺点：仅在8B模型上验证，未测更大规模模型；精度评估仍相对

#### 摘要
长上下文推理中，prefill 阶段产生的 KV cache 在每一层都被缓存并在 decode 阶段反复访问，代价高昂。SPEED（Shallow Prefill, dEEp Decode）提出一种 phase-asymmetric KV-visibility 策略：非 anchor prompt token 的 KV states 只在低层 layer 中物化，而 decode 阶段的 token 保持全深度；这与以往降低上层 KV 存储代价的方法不同，SPEED 直接将 prefill token 从上层 decode 的 visibility set 中移除。在 Llama-3.1-8B 的 128K 上下文实验中，使用 75% 层处理 prefill token 时，OLMES 评分从 51.4 降至 51.2，同时 TTFT 提升 33%、TPOT 提升 22%、active KV memory 减少 25.0%。逐层诊断表明该截止点保留了模型的主要 prompt 选择与表征稳定区域。

- 🎯 方法：prefill token 仅在浅层（75% 层）生成 KV cache，decode token 保持全深度 full-depth attention
- 📊 结果：128K 上下文下 TTFT 提升 33%、TPOT 提升 22%、active KV memory 减少 25.0%
- 📊 结果：OLMES 评分 51.2 vs 全深度基线 51.4，精度损失极小
- 💡 创新：通过 BoS anchor token 保持全层可见性，绕过直接删除 prefill KV 带来的信息断层问题

#### 🧭 为什么推送这篇
**Diffusion 压缩** · topic 4/5 · practicality 3/5 · composite 7/10 · 扩散模型压缩论文，核心贡献是用标准化流和自蒸馏实现少步高质量生成；精确likelihood框架和4步采样是优点，但缺乏明确的端到端推理速度数据和与现有少步方法（SDXL Turbo、LCM）的定量对比；实用性受限于训练复杂度和内存需求未知。

#### 摘要
Diffusion 模型在极少步数采样时高斯去噪假设失效，现有 few-step 方法（蒸馏、consistency training、对抗训练）均放弃了 likelihood 框架。NTM 将每个 reverse step 建模为带精确 likelihood 训练的 conditional normalizing flow，架构上在每步内使用 shallow invertible blocks、跨轨迹使用 deep parallel predictor，可从头训练或从预训练 flow-matching 模型初始化。利用精确轨迹 likelihood，NTM 进一步支持 self-distillation：用模型自身 score 训练轻量 denoiser，仅 4 步即可生成高质量图像，在 text-to-image benchmark 上达到或超越强基线，同时保留精确 likelihood。

- 🎯 方法：每个 reverse step 用 conditional normalizing flow 建模，保留精确 trajectory likelihood
- 💡 创新：shallow invertible blocks（步内）+ deep parallel predictor（跨轨迹）端到端架构，支持从预训练 flow-matching 初始化
- 📊 结果：仅 4 步采样即匹配或超越 text-to-image 强基线，且唯一保留精确 generative trajectory likelihood
- 🎯 方法：基于精确 likelihood 的 self-distillation，无需外部教师模型即可训练轻量 denoiser

#### 📎 相关方法 / 对比基线
- flow-matching — NTM 可从预训练 flow-matching 模型初始化权重
- consistency training — 现有 few-step 基线之一，NTM 在保留 likelihood 的前提下替代

#### 摘要
DeepSeek Sparse Attention (DSA) 在长上下文推理中使用多头 indexer 对所有 prefix token 打分，但多头设计导致 indexer 成为主要计算瓶颈。MISA 将 DSA indexer heads 视为 mixture-of-experts 池，通过轻量 router 基于 block-level 统计量为每个 query 选取少量 active heads，仅用这些 heads 执行 token-level 打分，同时引入层级变体先扩大候选集再用原始 DSA indexer 重排以精确恢复选中 token。在无额外训练、仅 8 个 active heads 的条件下，MISA 在 DeepSeek-V3.2 和 GLM-5 上的 LongBench 与 dense DSA indexer 持平，indexer heads 数量分别减少 8 倍和 4 倍，TileLang kernel 在单张 NVIDIA H200 GPU 上相比 DSA 原始 indexer kernel 实现约 3.82 倍加速。

- 🎯 方法：用 MoE router 从 DSA indexer head 池中为每个 query 动态选取少量 active heads，跳过其余 heads 的 token-level 打分
- 📊 结果：8 active heads 下 LongBench 与 dense DSA 持平，indexer heads 分别减少 8×（DeepSeek-V3.2）和 4×（GLM-5）
- 📊 结果：128K token 上 Needle-in-a-Haystack 全绿，每层恢复 DSA 所选 token 的 92% 以上，TileLang kernel 加速 3.82×
- 💡 创新：层级变体先用路由 heads 扩大候选集，再由原始 DSA indexer 重排，以极低代价精确逼近全头选token结果

#### 📎 相关方法 / 对比基线
- DSA — MISA 作为其 indexer 的 drop-in 替代，减少 active heads 数量
- HISA — MISA 在 LongBench 平均表现优于 HISA
