#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 3/5 · practicality 4/5 · composite 7/10 · KV缓存压缩的创意方法：在prefill阶段仅在下层保留提示token的KV，decode阶段保持全深度。推理加速显著（TTFT +33%、TPOT +22%），精度损失极小（<0.4pt），内存节省可观。但最大模型仅8B，基准评估相对有限（OLMES而非MMLU/GSM8K等标准）。算法简单（无需额外训练），部署友好

#### 摘要
长上下文推理中，prefill 阶段处理大量 prompt token 并在每层缓存 KV state，导致显存和延迟开销极大。SPEED（Shallow Prefill, dEEp Decode）提出一种 phase-asymmetric KV-visibility 策略：非 anchor 的 prompt token KV state 仅在低层 layer 中物化，而 decode 阶段的 token 保持全深度可见；同时使用最小化的 BoS anchor 保证信息传递。在 Llama-3.1-8B 的 128K context 场景下，使用 75% 层处理 prefill token 时，OLMES benchmark 平均分从 51.4 降至 51.2（-0.2），同时 TTFT 提升 33%，TPOT 提升 22%，active KV memory 减少 25.0%。

- 🎯 方法：prompt token KV state 仅在下层 layer 物化，upper layer 的 decode visibility set 中直接移除 prefill token
- 📊 结果：128K context 下，TTFT -33%、TPOT -22%、active KV memory -25.0%，benchmark 分数仅降 0.2（51.4→51.2）
- 💡 创新：phase-asymmetric 设计，prefill 与 decode 使用不对称的 layer 深度，无需改变 decode token 的全深度表示
- 🔧 工程：最小化 BoS anchor 机制保持 prompt 信息传递，layer-wise 诊断验证保留了 prompt-selection 和 representation-stabilization 区域

- 📊 结果：TileLang kernel 在单张 NVIDIA H200 上比 DSA 原始 indexer kernel 加速约 3.82×，128K token 上 Needle-in-a-Haystack 保持全绿
