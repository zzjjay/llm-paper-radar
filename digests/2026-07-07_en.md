#### Summary
KV cache memory grows linearly with sequence length during long-context LLM inference, creating a dominant GPU memory bottleneck. Token eviction discards information irreversibly, while semantic grouping fixes compression at prefill time and cannot recover token-level detail on demand. SeKV organizes context into entropy-guided semantic spans stored across a GPU-CPU hierarchy: a lightweight summary vector resides on GPU for coarse routing, and a low-rank SVD basis resides on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding without materializing the full KV cache on GPU. SeKV reduces GPU memory by 53.3% at 128K context versus full KV caching, improves over the strongest semantic compression baseline by 5.9% on average across four benchmarks, keeps the base LLM fully frozen, and adds fewer than 0.05% trainable parameters.

- 🎯 Entropy-guided semantic span grouping with GPU-CPU hierarchical storage: summary vectors on GPU for routing, low-rank SVD bases on CPU for token-level reconstruction without discarding any information
- 📊 53.3% GPU memory reduction at 128K context vs. full KV caching; 5.9% average improvement over the strongest semantic compression baseline across four benchmarks
- 💡 Trained zoom-in mechanism enables selective span expansion during decoding; base LLM fully frozen with <0.05% trainable parameters added
- ⚠️ End-to-end latency/speedup impact of CPU↔GPU reconstruction is not reported; integration into production serving stacks requires additional engineering effort
