**Low-bit（≤ 2-bit）** · topic 5/5 · practicality 5/5 · composite 10/10 · CodeGEMM针对2-bit codebook量化提出了全新的GEMM内核，用预计算的Psumbook（centroid与activation的内积）替代传统dequantization流程，直接通过code索引gather partial sum，消除逐元素查表开销并降低cache压力。在Llama-3-8B和70B上分别实现1.83x和8.93x加速，精度与state-of-the-art codebook方法相当。算法思路简洁（codebook index gather + precomputed inner product），针对CUDA商用GPU，属于可直接部署的CUDA kernel优化，对low_bits（≤2bit）量化推理效率有直接工程价值，属于低比特量化领域高实用性工作。

#### 摘要
针对 codebook-based 极低比特量化（如 2-bit）推理中 dequantization kernel 反复获取 centroid 并重建权重导致的高延迟与 cache 压力问题，CodeGEMM 提出 codebook-centric GEMM：将 centroid 与 activation 的内积预计算并存入轻量级 Psumbook，推理时 code indices 直接 gather 部分和，避免逐元素查表。该 kernel 在统一实现下支持 latency-memory-accuracy 权衡探索。在 Llama-3 上，2-bit 配置下相对 SOTA codebook 量化分别取得 1.83x (8B) 和 8.93x (70B) 加速，精度相当。

- 💡 用 Psumbook 预计算 centroid·activation 内积替代 dequantization

- 🔧 code indices 直接 gather 部分和，减小 on-chip footprint

#### 🧭 为什么推送这篇
**KV cache 压缩** · topic 4/5 · practicality 4/5 · composite 8/10 · WG-KV提出了一种KV cache准入机制（Admission），通过轻量级门控网络在写入前预测token utility，从而维持更紧凑的全局cache。与H2O/StreamingLLM等事后驱逐方法不同，该方法从根源减少cache写入。在Llama和Qwen上报告了显著的prefill和decode加速，与FlashAttention和Paged-KV兼容，代码已开源，具有较好的部署可行性。主要不确定性在于：摘要未明确测试模型的具体大小（可能 < 7B），以及下游benchmark准确率未披露，仅作为kv_cache方法评估practicality扣分。总体而言属于有实用价值的KV cache压缩工作。

#### 摘要
针对长上下文 LLM 推理中 KV cache 线性增长和 attention 二次复杂度的瓶颈，作者将 KV cache 管理形式化为 Admission、Selection、Eviction 三个因果原语，指出已有 selection/eviction 方案忽视了根源——无差别写入。提出 Write-Gated KV (WG-KV)：在 token 写入 cache 前用轻量机制预测其 utility，过滤低价值状态，仅维护一个紧凑的 global cache 加 sliding local cache。在 Llama 和 Qwen 上实现 46-68% 显存削减、3.03-3.70x prefill 加速、1.85-2.56x decode 加速，且兼容 FlashAttention 与 Paged-KV。
